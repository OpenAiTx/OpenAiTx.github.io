unused-task-warning

Al usar inyección de dependencias y el patrón async-await, es posible terminar con una interfaz que tiene un método que devuelve un Task. Si este método de la interfaz se usa en un método sincrónico, existe la probabilidad de que se ejecute erróneamente como un método fire and forget (lo cual no activará la advertencia incorporada CS4014). En esta situación, este analizador genera una advertencia.

Ambos pueden usarse como una extensión de Visual Studio o preferiblemente como un analizador de proyecto disponible desde NuGet.

Ejemplo:

using System.Threading.Tasks;

namespace AsyncAwaitProblem { public interface ICallee { bool ProblemSolved { get; } Task SolveProblemAsync(); }

public class Callee : ICallee { public bool ProblemSolved { get; set; }

public async Task SolveProblemAsync() { await Task.Delay(10); ProblemSolved = true; } } public class Caller { public bool DoCall() { ICallee xxx = new Callee(); // Este analizador mostrará una advertencia en la siguiente línea xxx.SolveProblemAsync(); // Esto probablemente sea un fire and forget no deseado. return xxx.ProblemSolved; // Retornará false - esperábamos que retornara true } } }

Tenga en cuenta que este analizador actualmente solo verifica los tipos awaitables conocidos (por ejemplo, tipos Task, ValueTask y ConfiguredTaskAwaitable (el tipo devuelto al usar el método ConfigureAwait)). Si se devuelve otro tipo 'Awaitable', este analizador no emitirá la advertencia. Esto podría corregirse en una versión futura.

Disponible en NuGet aquí: https://www.nuget.org/packages/Lindhart.Analyser.MissingAwaitWarning/

Disponible en el marketplace de Visual Studio aquí: https://marketplace.visualstudio.com/items?itemName=Lindhart.missingAwaitWarning#overview

Para información sobre cómo usar analizadores consulte aquí: https://docs.microsoft.com/en-us/visualstudio/code-quality/roslyn-analyzers-overview?view=vs-2019
