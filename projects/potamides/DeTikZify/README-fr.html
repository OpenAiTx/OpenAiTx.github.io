DeTikZify
Synthétiser des programmes graphiques pour figures scientifiques et croquis avec TikZ

Créer des figures scientifiques de haute qualité peut être chronophage et difficile, même si esquisser des idées sur papier est relativement facile. De plus, recréer des figures existantes qui ne sont pas stockées dans des formats préservant les informations sémantiques est tout aussi complexe. Pour relever ce défi, nous présentons DeTikZify, un nouveau modèle de langage multimodal qui synthétise automatiquement des figures scientifiques sous forme de programmes graphiques TikZ préservant la sémantique, à partir de croquis et de figures existantes. Nous introduisons également un algorithme d'inférence basé sur MCTS qui permet à DeTikZify d'affiner itérativement ses résultats sans nécessiter d'entraînement supplémentaire.

Installation

[!TIP]
Si vous rencontrez des difficultés avec l'installation ou l'inférence sur votre propre
matériel, pensez à visiter notre Hugging Face
Space (notez que
le redémarrage de l'espace peut prendre jusqu'à 30 minutes). En cas de longues
files d'attente, vous avez la possibilité de
dupliquer cet espace
avec un runtime GPU privé payant ou de le faire
tourner localement avec
Docker. De plus, vous pouvez essayer notre démo sur Google
Colab.
Cependant, la configuration de l'environnement peut prendre un certain temps, et
le niveau gratuit ne supporte l'inférence que pour les modèles 1b.

Le paquet Python de DeTikZify peut être facilement installé avec pip :

pip install 'detikzify[legacy] @ git+https://github.com/potamides/DeTikZify'
L’extra [legacy] est uniquement nécessaire si vous prévoyez d’utiliser les modèles DeTikZifyv1. Si vous prévoyez uniquement d’utiliser DeTikZifyv2, vous pouvez le supprimer. Si votre objectif est d’exécuter les exemples inclus, il est plus simple de cloner le dépôt et de l’installer en mode modifiable comme ceci : 
git clone https://github.com/potamides/DeTikZify
pip install -e DeTikZify[examples]
 ``
De plus, DeTikZify nécessite une installation complète
de TeX Live 2023,
ghostscript, et
poppler que vous devez installer via
votre gestionnaire de paquets ou par d’autres moyens.
Utilisation
[!CONSEIL]


Pour une utilisation interactive et des conseils d’utilisation,


nous recommandons de consulter notre interface web, qui peut être lancée


directement depuis la ligne de commande (utilisez --help pour une liste de toutes les options) :


`sh


python -m detikzify.webui --light


`
Si toutes les dépendances requises sont installées, l’ensemble des fonctionnalités de DeTikZify
telles que la compilation, le rendu, la sauvegarde des graphiques TikZ, et
l’inférence basée sur MCTS sont accessibles via son interface de programmation :

Exemple DeTikZify
python from operator import itemgetter

from detikzify.model import load from detikzify.infer import DetikzifyPipeline

image = "https://w.wiki/A7Cc" pipeline = DetikzifyPipeline(*load( model_name_or_path="nllg/detikzify-v2.5-8b", device_map="auto", torch_dtype="bfloat16", ))

generate a single TikZ program

fig = pipeline.sample(image=image)

if it compiles, rasterize it and show it

if fig.is_rasterizable: fig.rasterize().show()

run MCTS for 10 minutes and generate multiple TikZ programs

figs = set() for score, fig in pipeline.simulate(image=image, timeout=600): figs.add((score, fig))

save the best TikZ program

best = sorted(figs, key=itemgetter(0))[-1][1] best.save("fig.tex") 
Grâce aux adaptateurs TikZero et
TikZero+, il est également possible
de synthétiser des programmes graphiques conditionnés par du texte (cf. notre
article pour
les détails). Notez que cela est actuellement uniquement pris en charge via l'interface
de programmation :
Exemple TikZero+
python from detikzify.model import load from detikzify.infer import DetikzifyPipeline

caption = "A multi-layer perceptron with two hidden layers." pipeline = DetikzifyPipeline(*load( model_name_or_path="nllg/tikzero-plus-10b", device_map="auto", torch_dtype="bfloat16", ))

generate a single TikZ program

fig = pipeline.sample(text=caption)

if it compiles, rasterize it and show it

if fig.is_rasterizable: fig.rasterize().show() 


Exemple TikZero
python from detikzify.model import load, load_adapter from detikzify.infer import DetikzifyPipeline

caption = "A multi-layer perceptron with two hidden layers." pipeline = DetikzifyPipeline( *load_adapter( *load( model_name_or_path="nllg/detikzify-v2-8b", device_map="auto", torch_dtype="bfloat16", ), adapter_name_or_path="nllg/tikzero-adapter", ) )

generate a single TikZ program

fig = pipeline.sample(text=caption)

if it compiles, rasterize it and show it

if fig.is_rasterizable: fig.rasterize().show() 
Des exemples plus complexes, par exemple pour l'évaluation et l'entraînement, peuvent être trouvés

Remerciements

L'implémentation de l'architecture du modèle DeTikZify est basée sur LLaVA et AutomaTikZ (v1), ainsi que Idefics 3 (v2). Notre implémentation MCTS est basée sur VerMCTS. L'architecture TikZero s'inspire de Flamingo et LLaMA 3.2-Vision.
