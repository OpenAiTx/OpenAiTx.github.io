attribute-events

⭐ 329 stars Korean by jpkleemans 

class Order extends Model
{
    protected $dispatchesEvents = [
        'status:shipped' => OrderShipped::class,
        'note:*' => OrderNoteChanged::class,
    ];
}
Eloquent 모델은 created 및 deleted와 같은 여러 유용한 이벤트를 라이프사이클 동안 발생시킵니다. 그러나 모델의 생애 동안 일반적으로 더 많은 비즈니스 의미 있는 이벤트가 발생합니다. 이 라이브러리를 사용하면 속성 변경을 자체 이벤트 클래스에 매핑하여 이를 캡처할 수 있습니다.

설치

composer require jpkleemans/attribute-events

사용 방법

모델에서 Kleemans\AttributeEvents 트레이트를 사용하고 속성을 $dispatchesEvents 속성에 추가하세요:

class Order extends Model
{
    use AttributeEvents;
    protected $dispatchesEvents = [
        'created'         => OrderPlaced::class,
        'status:canceled' => OrderCanceled::class,
        'note:*'          => OrderNoteChanged::class,
    ];
}
속성 이벤트는 업데이트된 모델이 저장된 후에 디스패치됩니다. 각 이벤트는 생성자를 통해 모델의 인스턴스를 받습니다.

모델 이벤트 및 $dispatchesEvents 속성에 대한 자세한 내용은 Laravel 문서를 참조하세요.

리스닝

이제 EventServiceProvider의 $listen 배열을 통해 이벤트를 구독하거나, 클로저 기반 리스너로 수동 구독할 수 있습니다:

Event::listen(function (OrderCanceled $event) {
    // Restock inventory
});

또는 Laravel의 브로드캐스팅 기능을 사용하여 사용자에게 실시간 업데이트를 푸시할 수 있습니다:

Echo.channel('orders')
    .listen('OrderShipped', (event) => {
        // Display a notification
    })

JSON 속성

JSON으로 저장된 속성의 경우 -> 연산자를 사용할 수 있습니다:

protected $dispatchesEvents = [
    'payment->status:completed' => PaymentCompleted::class,
];

접근자

더 복잡한 상태 변경을 위해 접근자로 정의된 속성을 사용할 수 있습니다:

class Product extends Model
{
    protected $dispatchesEvents = [
        'low_stock:true' => ProductReachedLowStock::class,
    ];
    public function getLowStockAttribute(): bool
    {
        return $this->stock <= 3;
    }
}
Laravel 9에서 도입된 새로운 접근자 정의 방식도 사용할 수 있습니다.
