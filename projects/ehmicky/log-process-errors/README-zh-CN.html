log-process-errors

⭐ 469 stars Simplified Chinese by ehmicky

📰 Medium 文章。

为 Node.js 进程错误献上 ❤️。

这提升了进程错误的处理： 未捕获 异常， 未处理 的 Promise，过晚 处理的 Promise 以及 警告。

特性

  • 警告和
rejectionHandled 的堆栈追踪
  • 针对所有进程错误的单一事件处理器
  • 忽略重复的进程错误
  • 规范化无效错误
  • 进程退出#exit优雅且可被阻止

安装

生产代码（例如服务器）可以将此作为生产依赖或 开发依赖安装： 

npm install log-process-errors
然而，库应该将此作为开发依赖安装：

npm install -D log-process-errors

这是因为日志记录是全局修改的，库的使用者可能不会 预期这种副作用。此外，这可能导致库之间的冲突。

此包适用于 Node.js >=18.18.0。

这是一个 ES 模块。必须使用 import 或 import() 语句 加载，而非 require()。如果使用 TypeScript，必须配置为 输出 ES 模块， 而非 CommonJS。

API

logProcessErrors(options?)

options 对象?\ _返回值_: () => void

开始处理进程错误。

import logProcessErrors from 'log-process-errors'
logProcessErrors(options)
返回值恢复 Node.js 的默认行为。

const restore = logProcessErrors(options)
restore()

选项

exit

_类型_: boolean

是否在 未捕获的异常 或 未处理的 promise 时退出进程。

如果其他库正在监听这些事件，则默认值为 false，这样它们可以执行退出操作。否则，默认值为 true。

如果仍有任务在进行中，退出会等待这些任务最多完成 3 秒。

onError

_类型_: (error, event) => Promise | void\ _默认值_: console.error(error)

每当进程错误发生时调用的函数。重复的进程错误会被忽略。

// Log process errors with Winston instead
logProcessErrors({
  onError: (error, event) => {
    winstonLogger.error(error.stack)
  },
})
#### error

_Type_: Error

进程错误。保证这是一个 标准化的错误实例。 其消息中还附加了event的简要描述。

#### event

_Type_: Event

进程事件名称包括： 'uncaughtException'， 'unhandledRejection'， 'rejectionHandled'， 'warning'。
