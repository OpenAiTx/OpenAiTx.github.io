Refined GitHub

ส่วนขยายเบราว์เซอร์ที่ทำให้หน้าตา GitHub เรียบง่ายขึ้นและเพิ่มฟีเจอร์ที่มีประโยชน์

พวกเราใช้ GitHub เป็นประจำและพบว่ามีจุดที่น่ารำคาญหลายอย่างที่อยากจะแก้ไข ดังนั้นนี่คือจุดเริ่มต้นของเรา

เราหวังว่า GitHub จะเห็นและนำการปรับปรุงเหล่านี้ไปใช้ หากคุณชอบการปรับปรุงเหล่านี้ โปรดเปิดการสนทนาใน GitHub feedback หรือ ติดต่อ GitHub support เพื่อแจ้งให้ทราบ

GitHub Enterprise ก็รองรับเช่นกัน: วิธีเปิดใช้งาน

เครื่องหมายการค้า GITHUB และ REFINED GITHUB เป็นของ GitHub, Inc. และได้รับอนุญาตให้ใช้

ให้คุณซ่อนทุก event ยกเว้นคอมเมนต์หรือคอมเมนต์ที่ยังไม่ได้แก้ใน issues และ PRs

เพิ่มลิงก์ไปที่ default branch ในหน้ารายการไดเรกทอรีและไฟล์

  • [](# "new-repo-disable-projects-and-wikis") ปิดการใช้งาน projects และ wikis อัตโนมัติเมื่อสร้าง repo ใหม่
  • [](# "sticky-sidebar") ทำให้ sidebar ใน repo, issue, และ PR ติดขอบหน้าจอถ้าแสดงผลได้

  • [](# "github-actions-indicators") ใน sidebar workflow แสดงตัวบ่งชี้ว่าทำงานด้วยตนเองได้ และเวลานัดถัดไปถ้ามี
  • [](# "quick-repo-deletion") ลบ repo ได้ง่ายขึ้น เพิ่มปุ่ม "Delete fork" ใน repo ที่ไม่มีดาว และใช้ shift alt ลบ repo ใน 2 คลิก

  • [](# "clean-repo-tabs") ย้าย "Security" และ "Insights" ไปไว้ในเมนูนำทางดรอปดาวน์ของ repo รวมถึง "Projects", "Actions" และ "Wiki" หากว่าง/ไม่ใช้
  • [](# "repo-avatars") เพิ่มรูปโปรไฟล์ที่หัวของ repo สาธารณะ
  • [](# "quick-new-issue") เพิ่มลิงก์สร้าง issue จากทุกที่ใน repo
  • [](# "small-user-avatars") แสดง avatar เล็กข้างชื่อผู้ใช้ในรายการ issue/PR และการ mention
  • [](# "unreleased-commits") 🔥 แจ้งว่าคุณกำลังดู repo เวอร์ชันล่าสุดหรือมี commit ที่ยังไม่ release
  • [](# "action-pr-link") 🔥 เพิ่มลิงก์กลับไปยัง PR ที่รัน workflow
  • [](# "mobile-tabs") ทำให้แท็บกะทัดรัดขึ้นในมือถือเพื่อให้เห็นหลายแท็บมากขึ้น
  • [](# "repo-header-info") แสดงว่า repo นี้เป็น fork และเพิ่มจำนวนดาวที่หัวข้อ
  • [](# "visit-tag") เมื่อเรียกดูไฟล์ใน repo บน tag เฉพาะ จะเพิ่มลิงก์ไปยัง release/tag นั้น

การสนทนา

  • [](# "open-all-conversations") ให้คุณเปิด issue/PR ที่แสดงอยู่ทั้งหมดในครั้งเดียว
  • [](# "sticky-conversation-list-toolbar") ทำให้แถบเครื่องมือฟิลเตอร์ของรายการ issue/PR ติดขอบหน้าจอ
  • [](# "conversation-authors") ไฮไลต์ issue/PR ที่เปิดโดยคุณหรือผู้ร่วมงาน repo ปัจจุบัน
  • [](# "align-issue-labels") ในรายการ issue/PR จัดตำแหน่ง label ไปทางซ้ายใต้แต่ละหัวข้อ
  • [](# "sort-conversations-by-update-time") 🔥 เปลี่ยนการเรียงลำดับเริ่มต้นของ issue/PR เป็น Recently updated
  • [](# "more-conversation-filters") เพิ่ม filter Everything you’re involved in และ Everything you subscribed to ในเมนูค้นหาแบบดรอปดาวน์
  • [](# "global-conversation-list-filters") เพิ่ม filter สำหรับ issue/PR _ใน repo ของคุณ_ และ _ที่คุณคอมเมนต์_ ในการค้นหา issue/PR ทั่วโลก
  • [](# "clean-conversation-sidebar") 🔥 ซ่อนส่วนว่าง (หรือเฉพาะ label "ว่าง") ใน sidebar ของ issue/PR
  • [](# "clean-conversation-filters") ซ่อนตัวกรอง Projects และ Milestones ในรายการ issue/PR หากว่างเปล่า
  • [](# "toggle-everything-with-alt") เพิ่มคีย์ลัดเพื่อสลับรายการที่คล้ายกันทั้งหมด (คอมเมนต์ที่ย่อไว้, diff ที่ถูกเลื่อน ฯลฯ) ในครั้งเดียว: alt คลิก ที่แต่ละปุ่มหรือเช็คบ็อกซ์
  • [](# "extend-conversation-status-filters") ให้คุณสลับระหว่างตัวกรอง is:open/is:closed/is:merged ในการค้นหา
  • [](# "bugs-tab") เพิ่มแท็บ "Bugs" ให้กับ repo หากมี issue ที่เปิดอยู่พร้อม label "bug"
  • [](# "pinned-issues-update-time") แทนที่เวลาที่ "opened" ด้วยเวลาที่ "updated" บน pinned issues
  • [](# "clean-pinned-issues") เปลี่ยนเลย์เอาต์ของ pinned issues จากแบบข้างกันเป็นแบบรายการมาตรฐาน
  • [](# "quick-label-removal") เพิ่มปุ่มคลิกเดียวเพื่อลบ label ใน issues/PRs
  • [](# "clean-conversation-headers") ลบข้อมูลซ้ำใน header ของ issues และ PRs ("User wants to merge X commits from Y into Z")
  • [](# "dim-bots") ทำให้ commit และ PR จากบอทจางลงเพื่อลดเสียงรบกวน
  • [](# "esc-to-cancel") เพิ่มคีย์ลัดเพื่อยกเลิกการแก้ไขชื่อ issue/PR: esc
  • [](# "no-duplicate-list-update-time") ซ่อนเวลาการอัปเดตของ issues/PRs ในรายการเมื่อมันตรงกับเวลาเปิด/ปิด/merged
  • [](# "linkify-user-labels") ลิงก์ label "Contributor" และ "Member" ในคอมเมนต์ไปยัง commit ของผู้เขียนใน repo
  • [](# "jump-to-conversation-close-event") เพิ่มลิงก์เพื่อข้ามไปยัง event ปิดล่าสุดของ issue/PR
  • [](# "close-as-unplanned") ให้คุณ "ปิด issue เป็น unplanned" ได้ด้วยคลิกเดียวแทนที่จะต้องสามคลิก
  • [](# "locked-issue") แสดง label บน issue และ PR ที่ล็อกแล้ว

  • [](# "batch-mark-files-as-viewed") ทำเครื่องหมาย/ยกเลิกเครื่องหมายไฟล์หลายไฟล์ว่า “Viewed” ในแท็บ Files ของ PR โดยคลิกเช็คบ็อกซ์แรกที่ต้องการ จากนั้น shift-คลิกอีกอัน; ไฟล์ทั้งหมดระหว่างสองเช็คบ็อกซ์จะถูกทำเครื่องหมาย/ยกเลิก
  • [](# "closing-remarks") 🔥 แสดง Git tag แรกที่ PR ที่ merge แล้วถูกใส่ไว้ หรือแนะนำให้สร้าง release หากยังไม่มี release
  • [](# "pr-jump-to-first-non-viewed-file") ข้ามไปยังไฟล์ที่ยังไม่ได้ดูไฟล์แรกใน PR เมื่อคลิกที่ progress bar
  • [](# "jump-to-change-requested-comment") เพิ่มลิงก์เพื่อข้ามไปยังคอมเมนต์ change request ล่าสุด
  • [](# "view-last-pr-deployment") เพิ่มลิงก์เพื่อเปิด deployment ล่าสุดจาก header ของ PR
  • [](# "no-unnecessary-split-diff-view") ใช้ unified diff เสมอในไฟล์ที่ split diff ไม่เป็นประโยชน์
  • [](# "emphasize-draft-pr-label") ทำให้แยกแยะ draft PR ในรายการได้ง่ายขึ้น

การแก้ไข pull requests

  • [](# "sync-pr-commit-title") 🔥 ใช้ชื่อ PR เป็นชื่อ commit squash โดยดีฟอลต์ และ อัปเดตชื่อ PR ให้ตรงกับชื่อ commit หากมีการเปลี่ยนแปลง
  • [](# "update-pr-from-base-branch") เพิ่มปุ่ม "Update branch" ให้กับทุก PR GitHub มีฟีเจอร์เดียวกันแต่ต้องตั้งค่าด้วย branch ที่ได้รับการปกป้อง
  • [](# "one-click-review-submission") ทำให้ฟอร์มรีวิว PR ง่ายขึ้น: อนุมัติหรือปฏิเสธรีวิวได้เร็วขึ้นด้วยปุ่มรีวิวแบบคลิกเดียว
  • [](# "pull-request-hotkeys") เพิ่มคีย์ลัดสำหรับเปลี่ยนแท็บ PR: g ← และ g → หรือ g 1, g 2, g 3, และ g 4
  • [](# "pr-branch-auto-delete") ลบ branch อัตโนมัติหลังจาก merge PR ถ้าเป็นไปได้ โดยชื่อ branch ที่เป็นที่รู้จักว่าใช้งานระยะยาวจะไม่ถูกลบ (develop, release/* เป็นต้น)
  • [](# "one-click-pr-or-gist") ให้คุณสร้าง draft PR และ public gist ได้ในคลิกเดียว
  • [](# "clear-pr-merge-commit-message") ลบข้อความ commit merge ของ PR ที่รกออก เหลือแต่ co-author ที่ไม่ซ้ำกัน
  • [](# "quick-review") เพิ่มปุ่ม quick-review ใน sidebar ของ PR, โฟกัส textarea รีวิวโดยอัตโนมัติ, และเพิ่มคีย์ลัดเพื่อเปิด popup รีวิว: v
  • [](# "pr-first-commit-title") ใช้ commit แรกเป็นชื่อและคำอธิบายของ PR ใหม่

  • [](# "one-click-diff-options") เพิ่มปุ่มคลิกเดียวเพื่อเปลี่ยนรูปแบบ diff และข้ามช่องว่าง พร้อมคีย์ลัด: d w

  • [](# "tags-on-commits-list") แสดงแท็กที่เกี่ยวข้องถัดจาก commit

  • [](# "deep-reblame") เมื่อดู blame, Alt-คลิกที่ปุ่ม “Reblame” จะดึง commit ที่เกี่ยวข้องกับ PR ก่อน แทนที่จะมอง commit นั้นเป็นการเปลี่ยนแปลงเดียว
  • [](# "new-or-deleted-file") แสดงไอคอนว่ากำลังเพิ่มหรือลบไฟล์ใน commit และ PR
  • [](# "easy-toggle-files") สลับการแสดงผล diff ของไฟล์โดยคลิกที่แถบหัวเรื่องของไฟล์
  • [](# "same-branch-author-commits") คง branch และ path ปัจจุบันไว้เมื่อดู commit ทั้งหมดของผู้เขียน
  • [](# "easy-toggle-commit-messages") สลับแสดง/ซ่อนข้อความ commit โดยคลิกที่กล่อง commit
  • [](# "link-to-compare-diff") ลิงก์ข้อความ "X files changed" บนหน้า compare เพื่อข้ามไปยัง diff ได้
  • [](# "conventional-commits") แสดงประเภท conventional commit เป็น label หน้าข้อความ commit

  • [](# "linkify-user-edit-history-popup") ลิงก์ชื่อผู้ใช้ใน popup ประวัติการแก้ไข

เคยเป็นส่วนหนึ่งของ Refined GitHub

GitHub ได้แรงบันดาลใจจาก Refined GitHub และได้เพิ่มฟีเจอร์ของมันเข้าไปในระบบเองหลายสิบรายการ 🎉 ดังนั้นจึงได้ นำออกจากส่วนขยายนี้แล้ว คุณยังสามารถดู ฟีเจอร์เก่าทั้งหมด ของ Refined GitHub ได้ในรายการเดียว

การปรับแต่ง

ฟีเจอร์ส่วนใหญ่สามารถปิดการใช้งานได้หากเป็น JavaScript-based และคุณสามารถ override CSS ของเราได้เองในตัวเลือกของส่วนขยาย

เรายินดีรับข้อเสนอแนะและการมีส่วนร่วม แต่โปรดทราบว่านี่เป็นโปรเจกต์ที่มีความคิดเห็นเฉพาะทางสูง มีเกณฑ์สูงสำหรับการเพิ่มฟีเจอร์ ผู้ใช้จะมีความเห็นไม่ตรงกันเสมอ อย่างไรก็ตาม เรายินดีที่จะพูดคุย
