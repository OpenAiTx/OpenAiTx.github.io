protobuf-net.Grpc

protobuf-net.Grpc پشتیبانی از خدمات مبتنی بر کد را برای سرویس‌ها بر بستر gRPC با استفاده از API بومی Grpc.Core یا API کاملاً مدیریت‌شده Grpc.Net.Client / Grpc.AspNetCore.Server اضافه می‌کند.

این ابزار باید بر روی همه زبان‌های .NET که می‌توانند چیزی حتی کمی مشابه مدل نوع معمولی .NET تولید کنند، کار کند.

  • شروع کار
  • تمام مستندات
  • ابزار ساخت/استفاده در دسترس از طریق protobuf-net.BuildTools

استفاده از آن به سادگی اعلام یک اینترفیس برای قرارداد سرویس شماست:

`` c# [ServiceContract] public interface IMyAmazingService { ValueTask SearchAsync(SearchRequest request); // ... } 


سپس یا آن اینترفیس را برای سرور پیاده‌سازی می‌کنید:
c# public class MyServer : IMyAmazingService { // ... } 

یا از سیستم یک کلاینت می‌خواهید:
c# var client = http.CreateGrpcService(); var results = await client.SearchAsync(request); 

این معادل سرویس در .proto خواهد بود:
proto service MyAmazingService { rpc Search (SearchRequest) returns (SearchResponse) {} // ... } `

بدیهی است که باید uri و غیره را به آن بدهید - به شروع کار مراجعه کنید. معمولاً پیکربندی مبتنی بر قرارداد است، اما در صورت تمایل: گزینه‌های پیکربندی مختلفی وجود دارد.

نحوه تهیه

همه چیز به صورت بسته‌های از پیش ساخته شده در nuget موجود است؛ به طور خاص، احتمالاً یکی از این‌ها را می‌خواهید:

  • protobuf-net.Grpc.AspNetCore برای سرورهایی که از ASP.NET Core 3.1 استفاده می‌کنند
  • protobuf-net.Grpc.Native برای کلاینت‌ها یا سرورهایی که از API بومی/مدیریت‌نشده استفاده می‌کنند
  • protobuf-net.Grpc و Grpc.Net.Client برای کلاینت‌هایی که از HttpClient روی .NET Core 3.1 استفاده می‌کنند

نمونه‌های استفاده در C#، VB و F# در دسترس است.
