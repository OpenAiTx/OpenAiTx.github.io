TensorZero

TensorZero tworzy pętlę sprzężenia zwrotnego dla optymalizacji aplikacji LLM — zamieniając dane produkcyjne w inteligentniejsze, szybsze i tańsze modele.

Czym jest TensorZero? TensorZero to open-source'owy framework do budowy aplikacji LLM klasy produkcyjnej. Ujednolica bramkę LLM, obserwowalność, optymalizację, ewaluacje i eksperymentowanie.
Czym TensorZero różni się od innych frameworków LLM?
  • TensorZero umożliwia optymalizację złożonych aplikacji LLM na podstawie metryk produkcyjnych i feedbacku od ludzi.
  • TensorZero wspiera potrzeby aplikacji LLM na skalę przemysłową: niskie opóźnienia, wysoką przepustowość, bezpieczeństwo typów, self-hosting, GitOps, możliwość dostosowania, itd.
  • TensorZero jednoczy cały stack LLMOps, tworząc efekt kuli śnieżnej. Na przykład ewaluacje LLM mogą być wykorzystywane do fine-tuningu modeli razem z AI judge'ami.
Czy mogę używać TensorZero z ___? Tak. Wspierane są wszystkie główne języki programowania. Możesz używać TensorZero z naszym klientem Pythona, dowolnym SDK OpenAI lub przez nasze HTTP API.
Czy TensorZero jest gotowy na produkcję? Tak. Przykład wdrożenia: Automatyzacja changelogów w dużym banku za pomocą LLM-ów
Ile kosztuje TensorZero? Nic. TensorZero jest w 100% self-hosted i open-source. Nie ma płatnych funkcji.
Kto rozwija TensorZero? Nasz zespół techniczny to m.in. były maintainer kompilatora Rust, badacze machine learning (Stanford, CMU, Oxford, Columbia) z tysiącami cytowań i CPO startupu typu decacorn. Wspierają nas ci sami inwestorzy co wiodące projekty open-source (np. ClickHouse, CockroachDB) i laboratoria AI (np. OpenAI, Anthropic).
Jak zacząć? Możesz wdrażać TensorZero stopniowo. Nasz Szybki start prowadzi od zwykłego wrappera OpenAI do aplikacji LLM gotowej na produkcję z obserwowalnością i fine-tuningiem w 5 minut.

Użycie: Python — Klient TensorZero (zalecane)

Możesz uzyskać dostęp do dowolnego dostawcy za pomocą klienta TensorZero dla Pythona.

  • pip install tensorzero
  • Opcjonalnie: Skonfiguruj TensorZero.
  • Uruchom inferencję:

from tensorzero import TensorZeroGateway  # lub AsyncTensorZeroGateway

with TensorZeroGateway.build_embedded(clickhouse_url="...", config_file="...") as client:
    response = client.inference(
        model_name="openai::gpt-4o-mini",
        # Łatwo przetestuj innych dostawców: "anthropic::claude-3-7-sonnet-20250219"
        input={
            "messages": [
                {
                    "role": "user",
                    "content": "Napisz haiku o sztucznej inteligencji.",
                }
            ]
        },
    )

Użycie: Python — Klient OpenAI

Możesz uzyskać dostęp do dowolnego dostawcy przez klienta OpenAI dla Pythona z TensorZero.

  • pip install tensorzero
  • Opcjonalnie: Skonfiguruj TensorZero.
  • Uruchom inferencję:

from openai import OpenAI  # lub AsyncOpenAI
from tensorzero import patch_openai_client
client = OpenAI()
patch_openai_client( client, clickhouse_url="http://chuser:chpassword@localhost:8123/tensorzero", config_file="config/tensorzero.toml", async_setup=False, )

response = client.chat.completions.create( model="tensorzero::model_name::openai::gpt-4o-mini", # Wypróbuj innych dostawców łatwo: "tensorzero::model_name::anthropic::claude-3-7-sonnet-20250219" messages=[ { "role": "user", "content": "Napisz haiku o sztucznej inteligencji.", } ], )

Inżynieria LLM z TensorZero



  • TensorZero Gateway to wysokowydajna bramka modelowa napisana w Rust 🦀, która zapewnia zunifikowany interfejs API dla wszystkich głównych dostawców LLM, umożliwiając bezproblemową integrację międzyplatformową i mechanizmy awaryjne.
  • Obsługuje inferencję opartą na strukturalnych schematach z opóźnieniem P99 poniżej 1 ms (zobacz Benchmarki) oraz wbudowaną obserwowalność, eksperymentowanie i optymalizacje w czasie inferencji.
  • Zbiera także metryki i informacje zwrotne związane z tymi inferencjami, z natywnym wsparciem dla wieloetapowych systemów LLM.
  • Wszystko jest przechowywane w hurtowni danych ClickHouse, którą kontrolujesz — dla analityki w czasie rzeczywistym, skalowalnej i przyjaznej deweloperom.
  • Z czasem Receptury TensorZero wykorzystują ten uporządkowany zbiór danych do optymalizacji promptów i modeli: uruchamiaj gotowe receptury dla typowych zadań takich jak fine-tuning lub twórz własne z pełną elastycznością, używając dowolnego języka i platformy.
  • Na koniec funkcje eksperymentowania bramki i orkiestracja GitOps pozwalają Ci iterować i wdrażać z pewnością — niezależnie czy to pojedynczy LLM, czy tysiące modeli.

Naszym celem jest pomoc inżynierom w budowaniu, zarządzaniu i optymalizacji nowej generacji aplikacji LLM: systemów uczących się na podstawie rzeczywistych doświadczeń. Przeczytaj więcej o naszej Wizji i Planie Rozwoju.
