AutoPlug-Client

⭐ 67 stars Simplified Chinese by Osiris-Team

AutoPlug-Client 是一个独立的、可执行的 Java 程序，具有多种执行模式：服务器包装器、命令行工具、后台服务。它的主要目的是自动更新与服务器相关的任何软件，并简化服务器维护。虽然所有服务器类型均兼容（包括 Steam 游戏服务器），但其大部分功能专门针对 Minecraft，该游戏被誉为全球最畅销游戏，并拥有极其庞大的模组社区。

它能做什么？

  • 自动更新自身、服务器软件、插件、模组和 JRE（Java 运行环境）
  • 定时服务器重启
  • 自动化、模块化备份创建
  • 快速、异步、多线程任务执行
  • 支持同步（共享）文件夹
  • Web 面板用于启动、停止、重启服务器及查看最近更新结果摘要

高级版 AutoPlug-Web 功能：

  • 在线控制台
  • 在线文件管理器
  • 员工访问权限
  • 插件详情数据库

稳定且功能完整，因此采用社区驱动开发，意味着主开发者将不再开发新功能，依赖他人提交的拉取请求，除非涉及安全问题和关键错误。

开发者：

你也可以通过将 AutoPlug 作为依赖项添加到你的项目中来使用它， 通过 Maven/Gradle/Sbt/Leinigen。一些有用的类例如 AutoPlugConsole，它允许你以编程方式执行 AutoPlug 命令： 

AutoPlugConsole.executeCommand(".check plugins");
AutoPlugConsole.executeCommand(".check server");
// etc...

详情：

  • 使用 Java 编写，基于 JDK 8，在 IntelliJ IDEA 中开发
  • 使用 1JPM 构建，命令：java ./src/main/java/JPM.java

库

请注意此列表可能过时。请查看 build.gradle 文件以获取我们当前的依赖项。

  • AutoPlug-Core 包含所有 AutoPlug
项目中频繁使用的代码，打包在一个 jar 中。
  • Dream-Yaml 用于处理 YAML 文件。
  • Better-Thread 支持显示“实时任务”。
  • 控制台着色和任务显示使用了 Jansi 和 Jline。
  • 使用 Zip4j 解压和处理 jar/zip 文件。
  • 服务器重启由 Quartz 调度。
