🎬 Narzędzie do masowego pobierania wideo yt-dlp (obsługa Windows / Linux)

Jednoetapowy skrypt do pobierania wideo oparty na yt-dlp, inspirowany wpisem użytkownika NodeSeek.
Obsługuje pobieranie pojedynczych i wielu wideo, kompatybilny z Windows oraz głównymi dystrybucjami Linux (Debian / Ubuntu / Alpine / CentOS).

✨ Funkcje

  • 📥 Jedno kliknięcie uruchamia program: Bez ręcznej konfiguracji, wystarczy podać dane według wskazówek, aby rozpocząć pobieranie
  • 🍪 Obsługa własnych ciasteczek Cookie: Przydatne dla filmów wymagających logowania (⚠️ Funkcja nie została jeszcze zweryfikowana, mogą występować błędy)
  • 📂 Możliwość wyboru katalogu wyjściowego: Łatwo zapisuj do wybranego folderu
  • 📃 Obsługa pobierania wsadowego: Automatyczne czytanie linków z pliku urls.txt i pobieranie
  • ⚙️ Automatyczna instalacja zależności: W środowisku Linux automatycznie instaluje yt-dlp i ffmpeg

🪟 Instrukcja obsługi dla Windows

  • Pobierz następujące pliki:

  • yt-dlp.exe
  • ffmpeg wersja na Windows

  • Po rozpakowaniu umieść następujące pliki w tym samym folderze (np. w folderze yt-dlp):

  • yt-dlp.exe
  • ffmpeg.exe
  • ffprobe.exe
  • yt-dlp.bat (plik skryptu)

  • Kliknij dwukrotnie yt-dlp.bat, postępuj zgodnie z instrukcjami, aby pobrać wideo.
  • Edytuj config/config.ini, aby zmienić ustawienia domyślne

---

🐧 Instrukcja obsługi dla Linux (Skrypt najnowszej wersji od technologicznego lion-a także zawiera te same funkcje)

Systemy obsługiwane: Debian / Ubuntu / Alpine / CentOS

Jedno kliknięcie do wdrożenia, aktualizacji i uruchomienia:

bash -c "$(curl -fsSL https://raw.githubusercontent.com/nodeloc666/yt-dlp-script/main/install.sh)"

Aby odinstalować:

bash -c "$(curl -fsSL https://raw.githubusercontent.com/nodeloc666/yt-dlp-script/main/uninstall.sh)"

📱 Rekomendacje dla użytkowników Androida

  • Skorzystaj z aplikacji Seal, aby korzystać z funkcji yt-dlp na Androidzie.
  • Instalacja Debiana przez termux również umożliwia pobieranie, ale nie ma takiej potrzeby, Seal jest bardzo wygodny.

⚠️ Uwagi

  • Tryb pobierania wsadowego: Skrypt automatycznie odczyta plik config\urls.txt w bieżącym katalogu, jeden link do wideo na linię. Jeśli plik nie istnieje, zostanie automatycznie utworzony.
  • Wskazówka dotycząca zasobów: Na urządzeniach z małą ilością pamięci RAM nie zaleca się pobierania wielu dużych filmów jednocześnie ani ustawiania zbyt dużej liczby wątków równoległych.
  • Szczególna uwaga dla CentOS: Ze względu na specyfikę środowiska nie został jeszcze w pełni przetestowany, zaleca się używanie w środowiskach nieprodukcyjnych. Pozostałe popularne systemy zostały przetestowane i działają poprawnie.
  • Mały bug w wersji Linux: Skrypt po wejściu na stronę główną sprawdza zależności, kierując się zasadą "działa, to nie ruszaj", nie chciało mi się poprawiać.
  • Mały bug w wersji Windows: Po powrocie z innych stron na stronę główną i ponownym wyborze opcja domyślna może nie działać, przez co następuje zapętlenie; nie udało się tego naprawić.
Dwa rozwiązania:
  • Wyjdź i uruchom ponownie
  • Nie wybieraj domyślnej opcji, tylko wybierz numer
