Comment ça marche

Supabase est une combinaison d'outils open source. Nous construisons les fonctionnalités de Firebase en utilisant des produits open source de niveau entreprise. Si les outils et communautés existent, avec une licence open source MIT, Apache 2 ou équivalente, nous utiliserons et soutiendrons cet outil. Si l'outil n'existe pas, nous le développons et le publions en open source nous-mêmes. Supabase n'est pas une réplique exacte de Firebase. Notre objectif est d'offrir aux développeurs une expérience similaire à Firebase en utilisant des outils open source.

Architecture

Supabase est une plateforme hébergée. Vous pouvez vous inscrire et commencer à utiliser Supabase sans rien installer. Vous pouvez également auto-héberger et développer localement.

  • Postgres est un système de base de données relationnelle objet avec plus de 30 ans de développement actif, ce qui lui a valu une solide réputation en termes de fiabilité, de robustesse des fonctionnalités et de performance.
  • Realtime est un serveur Elixir qui vous permet d'écouter les insertions, mises à jour et suppressions PostgreSQL via websockets. Realtime interroge la fonctionnalité de réplication intégrée de Postgres pour les changements de la base de données, convertit les changements en JSON, puis diffuse le JSON via websockets aux clients autorisés.
  • PostgREST est un serveur web qui transforme directement votre base de données PostgreSQL en une API RESTful.
  • GoTrue est une API d'authentification basée sur JWT qui simplifie l'inscription des utilisateurs, les connexions et la gestion des sessions dans vos applications.
  • Storage une API RESTful pour gérer les fichiers dans S3, avec Postgres gérant les permissions.
  • pg_graphql une extension PostgreSQL qui expose une API GraphQL.
  • postgres-meta est une API RESTful pour gérer votre Postgres, permettant de récupérer des tables, d'ajouter des rôles, d'exécuter des requêtes, etc.
  • Kong est une passerelle d'API cloud-native.

#### Bibliothèques clientes
