Como funciona

O Supabase é uma combinação de ferramentas open source. Estamos construindo as funcionalidades do Firebase usando produtos open source de nível empresarial. Se as ferramentas e comunidades existirem, com uma licença aberta MIT, Apache 2 ou equivalente, iremos utilizar e apoiar essa ferramenta. Se a ferramenta não existir, nós a construímos e a disponibilizamos como open source. O Supabase não é um mapeamento 1-para-1 do Firebase. Nosso objetivo é oferecer aos desenvolvedores uma experiência semelhante à do Firebase, utilizando ferramentas open source.

Arquitetura

O Supabase é uma plataforma hospedada. Você pode se cadastrar e começar a usar o Supabase sem instalar nada. Você também pode hospedar por conta própria e desenvolver localmente.

  • Postgres é um sistema de banco de dados objeto-relacional com mais de 30 anos de desenvolvimento ativo, conquistando uma forte reputação por confiabilidade, robustez de recursos e desempenho.
  • Realtime é um servidor Elixir que permite escutar inserts, updates e deletes no PostgreSQL usando websockets. O Realtime consulta a funcionalidade de replicação interna do Postgres para alterações no banco de dados, converte as alterações para JSON e transmite o JSON por websockets para clientes autorizados.
  • PostgREST é um servidor web que transforma seu banco de dados PostgreSQL diretamente em uma API RESTful.
  • GoTrue é uma API de autenticação baseada em JWT que simplifica cadastros, logins e gerenciamento de sessões em suas aplicações.
  • Storage é uma API RESTful para gerenciamento de arquivos no S3, com o Postgres controlando permissões.
  • pg_graphql é uma extensão do PostgreSQL que expõe uma API GraphQL.
  • postgres-meta é uma API RESTful para gerenciar seu Postgres, permitindo buscar tabelas, adicionar roles, executar queries, etc.
  • Kong é um gateway de API nativo para a nuvem.
