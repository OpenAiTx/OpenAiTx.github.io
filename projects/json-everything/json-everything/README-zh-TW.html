什麼是 json-everything？

json-everything 背後的主要理念，是確保在 _System.Text.Json_ 領域中，常見的 JSON 功能能夠獲得良好支援。

第一步是檢查是否已經有其他專案支援某個特定功能。

如果有，這個專案通常不會重複該功能。在下方你會找到一些其他專案，它們在 JSON 領域做了一些很酷的事。

否則，則會考慮納入本專案。

盒子裡有什麼？

實際上有多個盒子。每個功能都被拆分為獨立的函式庫，因此你可以根據需要選擇使用。

所有函式庫皆支援 .Net Standard 2.0 及 .Net 8。

JsonSchema.Net
 JSON Schema，支援第6版及以上草案
JsonSchema.Net.Generation
 從 .Net 類型產生 Schema
JsonSchema.Net.Generation.DataAnnotations
 支援 System.ComponentModel.DataAnnotations 的 Schema 產生
JsonSchema.Net.DataGeneration
 隨機實例資料產生（由 Bogus 提供）
JsonSchema.Net.Data
 一組用於存取實例及外部資料的詞彙
JsonSchema.Net.ArrayExt
 擴充陣列驗證的詞彙
JsonSchema.Net.OpenApi
 OpenApi 3.1 詞彙擴充（由 Graeae 使用）
JsonPath.Net
 JSON 路徑 (RFC 9535)
JsonPatch.Net
 JSON 補丁 (RFC 6902)
JsonPointer.Net
 JSON 指標 (RFC 6901) 及相對 JSON 指標 (規格)
JsonLogic
 JsonLogic (官方網站)
JsonE.Net
 JSON-e (官方網站)
Json.More.Net
 一般用途的擴充，理應應該包含於 System.Text.Json[.Nodes] 但尚未被納入
Yaml2JsonNode
 於 YamlDotNet 的 YAML 文件模型與 JsonNode 之間的轉換（雙向）。

以上所有函式庫皆完全支援其相關規範。

_JsonSchema.Net_ 的錯誤訊息翻譯支援以下語言：

  • 義大利語 it
  • 挪威語 nb-NO
  • 波蘭語 pl-PL
  • 俄語 ru
  • 西班牙語 es
  • 瑞典語 sv-SE
  • 土耳其語 tr-TR

注意 每個語言包都由其專屬的 Nuget 套件提供。

你也可以在 Bowtie 上查看 JSON Schema 測試套件的結果，該網站會在不同平台的多個實作上運行測試套件。

免責聲明 我的函式庫 _JsonSchema.Net_ 與網站 https://jsonschema.net 無任何關聯或連結，僅有兩者都是優秀的 JSON Schema 工具。

已棄用的套件

這些套件曾經是本套件組的一部分，但現已不再維護。

JsonSchema.Net.CodeGeneration
 從 schema 產生 C# 程式碼 已棄用，建議改用 Corvus.JsonSchema

其他專案對 _System.Text.Json_ 的支援

如果你在這裡沒找到想要的功能，請試試以下這些優秀的專案：

  • JsonCons.Net 作者 @danielaparker
  • JSON 指標 (Pointer)
  • JSON 修補 (Patch)
  • JSON 合併修補 (Merge Patch)
  • JSON 路徑 (Path)
  • JMES 路徑 (JMES Path)
  • Corvus.JsonSchema 作者 @mwadams
  • 專注於 C# 程式碼生成的 JSON Schema 驗證
  • （更多即將推出）

如果你用 JSON 做了某些本套件組或上述專案未涵蓋的功能，歡迎建立功能需求 issue。

如果你維護或知道其他擴充 _System.Text.Json_ 的專案，能做很酷的事，我希望在這裡列出，請建立一般 issue告知我。
