json-everythingとは？

json-everythingの主な理念は、一般的なJSON機能が_System.Text.Json_領域で十分にサポートされることを保証することです。 最初のステップは、指定された機能をサポートする他のプロジェクトが存在するかどうかを確認することです。

もし存在する場合、その作業はここで重複して行われることはありません。少し下に進むと、JSONを使って面白いことをしている他のプロジェクトのリストが見つかります。

そうでなければ、このプロジェクトで検討の余地があります。

箱の中身は？

実際には複数の箱があります。各機能は独自のライブラリとして分割されているため、必要なものだけを選択して使用できます。

すべてのライブラリは .Net Standard 2.0 および .Net 8 でビルドされています。

JsonSchema.Net
 JSON Schema、ドラフト6以降
JsonSchema.Net.Generation
 .Net型からのスキーマ生成
JsonSchema.Net.Generation.DataAnnotations
 System.ComponentModel.DataAnnotations のスキーマ生成サポート。
JsonSchema.Net.DataGeneration
 ランダムなインスタンスデータ生成（Bogus 利用）
JsonSchema.Net.Data
 インスタンスや外部データへのアクセス用語彙
JsonSchema.Net.ArrayExt
 配列バリデーション拡張用語彙
JsonSchema.Net.OpenApi
 OpenApi 3.1 用語拡張（Graeae で使用）
JsonPath.Net
 JSON Path (RFC 9535)
JsonPatch.Net
 JSON Patch (RFC 6902)
JsonPointer.Net
 JSON Pointer (RFC 6901) および Relative JSON Pointer (仕様)
JsonLogic
 JsonLogic (ウェブサイト)
JsonE.Net
 JSON-e (ウェブサイト)
Json.More.Net
 一般用途の拡張機能で、System.Text.Json[.Nodes] に含まれるべきだったが含まれていないもの
Yaml2JsonNode
 YamlDotNet の YAML ドキュメントモデルと JsonNode 間の変換（両方向）

上記のすべてのライブラリは、各仕様への完全な対応を提供します。

_JsonSchema.Net_ のエラーメッセージ翻訳は以下の言語で利用可能です：

  • イタリア語 it
  • ノルウェー語 nb-NO
  • ポーランド語 pl-PL
  • ロシア語 ru
  • スペイン語 es
  • スウェーデン語 sv-SE
  • トルコ語 tr-TR

注意 各言語パックは、それぞれの Nuget パッケージによって提供されています。

JSON Schema Test Suite の結果は、Bowtie でも確認できます。これは、複数のプラットフォームで様々な実装に対してテストスイートを実行します。

免責事項 私のライブラリ _JsonSchema.Net_ は、https://jsonschema.net というウェブサイトとは関係も提携もありません。ただし、両者とも優れた JSON Schema ツールです。

非推奨パッケージ

これらのパッケージもこのスイートの一部でしたが、現在はサポートされていません。

JsonSchema.Net.CodeGeneration
 スキーマから C# コードの生成 Corvus.JsonSchema 推奨のため非推奨

他プロジェクトによる _System.Text.Json_ サポート

ここで目的のものが見つからない場合は、以下の優れたプロジェクトもお試しください:

  • JsonCons.Net （@danielaparker による）
  • JSON ポインタ
  • JSON パッチ
  • JSON マージパッチ
  • JSON パス
  • JMES パス
  • Corvus.JsonSchema （@mwadams による）
  • C# コード生成を重視した JSON Schema バリデーション
  • （今後追加予定）

JSON を使って、このスイートや上記プロジェクトのライブラリでカバーされていないことを行いたい場合は、機能リクエストを自由に作成してください。

_System.Text.Json_ を拡張して面白いことを行う他のプロジェクトを管理している、またはご存知の場合は、ぜひここに掲載したいので、一般的な問い合わせを作成してお知らせください。
