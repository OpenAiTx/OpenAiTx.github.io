json-everything とは？

json-everything の主な哲学は、一般的な JSON 機能が _System.Text.Json_ 空間で十分にサポートされることを保証することです。

その第一歩は、特定の機能をサポートする他のプロジェクトが存在するかどうかを確認することです。

もし存在する場合、その機能の実装はここで重複しない可能性が高いです。少し下に進むと、JSON を使って面白いことをしている他のプロジェクトのリストがあります。

存在しない場合は、このプロジェクトで検討対象となります。

何が含まれていますか？

実際には複数の「ボックス」があります。それぞれの機能は独自のライブラリとして分割されており、必要なものだけを選んで利用できます。

すべてのライブラリは .Net Standard 2.0 および .Net 8 でビルドされています。

JsonSchema.Net
 JSON Schema、ドラフト6以降に対応
JsonSchema.Net.Generation
 .Net型からスキーマを生成
JsonSchema.Net.Generation.DataAnnotations
 System.ComponentModel.DataAnnotations のスキーマ生成サポート。
JsonSchema.Net.DataGeneration
 ランダムインスタンスデータ生成（Bogus により実現）
JsonSchema.Net.Data
 インスタンスおよび外部データへアクセスするための語彙
JsonSchema.Net.ArrayExt
 配列バリデーションを拡張するための語彙
JsonSchema.Net.OpenApi
 OpenApi 3.1 の語彙拡張（Graeae で使用）
JsonPath.Net
 JSON Path (RFC 9535)
JsonPatch.Net
 JSON Patch (RFC 6902)
JsonPointer.Net
 JSON Pointer (RFC 6901) および Relative JSON Pointer (仕様)
JsonLogic
 JsonLogic (公式サイト)
JsonE.Net
 JSON-e (公式サイト)
Json.More.Net
 System.Text.Json[.Nodes] に本来含まれるべきだったが含まれていない汎用拡張機能
Yaml2JsonNode
 YamlDotNet の YAML ドキュメントモデルと JsonNode との間の変換（双方向）。

上記のすべてのライブラリは、それぞれの仕様に対して完全なサポートを提供します。

_JonSchema.Net_ のエラーメッセージ翻訳は以下の言語で利用可能です:

  • イタリア語 it
  • ノルウェー語 nb-NO
  • ポーランド語 pl-PL
  • ロシア語 ru
  • スペイン語 es
  • スウェーデン語 sv-SE
  • トルコ語 tr-TR

注 各言語パックは個別の Nuget パッケージとして提供されます。

また、Bowtie で JSON Schema テストスイートの結果を確認できます。このサイトでは、複数の実装を異なるプラットフォームでテストしています。

免責事項 私のライブラリ _JsonSchema.Net_ は、https://jsonschema.net というウェブサイトとは関係や提携はありません。ただし、どちらも優れた JSON Schema ツールです。

非推奨パッケージ

これらのパッケージもかつて本スイートの一部でしたが、現在はサポートされていません。

JsonSchema.Net.CodeGeneration
 スキーマから C# コードを生成 Corvus.JsonSchema への移行のため非推奨

_System.Text.Json_ をサポートする他のプロジェクト

ここに探しているものがない場合は、以下の優れたプロジェクトもお試しください:

  • JsonCons.Net （@danielaparker による）
  • JSON Pointer
  • JSON Patch
  • JSON Merge Patch
  • JSON Path
  • JMES Path
  • Corvus.JsonSchema （@mwadams による）
  • C# コード生成を重視した JSON Schema バリデーション
  • （今後追加予定）

もし、このスイートや上記プロジェクトのいずれでもカバーされていない JSON 関連の用途があれば、フィーチャー要望 issue を作成してください。

また、_System.Text.Json_ を拡張して面白いことをする他のプロジェクトを管理している、またはご存知の場合は、ぜひここに掲載したいので、一般的な issue を作成してお知らせください。
