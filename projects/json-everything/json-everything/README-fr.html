Qu'est-ce que json-everything ?

La philosophie principale derrière json-everything est de s'assurer que les fonctionnalités courantes de JSON bénéficient d'un bon support dans l'espace _System.Text.Json_.

La première étape consiste à vérifier s'il existe d'autres projets qui prennent en charge une fonction donnée.

Si c'est le cas, il est probable que cet effort ne sera pas dupliqué ici. Un peu plus bas, vous trouverez une liste d'autres projets qui font des choses intéressantes avec JSON.

Sinon, cela reste ouvert à la considération pour ce projet.

Que trouve-t-on dans la boîte ?

En fait, il y a plusieurs boîtes. Chaque fonctionnalité a été séparée dans sa propre bibliothèque, afin que vous puissiez choisir celle(s) dont vous avez besoin.

Toutes les bibliothèques sont conçues pour .Net Standard 2.0 et .Net 8.

JsonSchema.Net
 JSON Schema, drafts 6 et ultérieurs
JsonSchema.Net.Generation
 Génération de schémas à partir des types .Net
JsonSchema.Net.Generation.DataAnnotations
 Prise en charge de la génération de schémas pour System.ComponentModel.DataAnnotations.
JsonSchema.Net.DataGeneration
 Génération aléatoire de données d’instance (propulsé par Bogus)
JsonSchema.Net.Data
 Un vocabulaire pour accéder aux données d’instance et externes
JsonSchema.Net.ArrayExt
 Un vocabulaire pour étendre la validation des tableaux
JsonSchema.Net.OpenApi
 Extension de vocabulaire OpenApi 3.1 (utilisée par Graeae)
JsonPath.Net
 JSON Path (RFC 9535)
JsonPatch.Net
 JSON Patch (RFC 6902)
JsonPointer.Net
 JSON Pointer (RFC 6901) et Relative JSON Pointer (Spécification)
JsonLogic
 JsonLogic (Site web)
JsonE.Net
 JSON-e (Site web)
Json.More.Net
 Extensions à usage général qui auraient probablement dû être incluses dans System.Text.Json[.Nodes] mais ne l’ont pas été
Yaml2JsonNode
 Conversions entre le modèle de document YAML de YamlDotNet et JsonNode (dans les deux sens).

Toutes les bibliothèques ci-dessus offrent une prise en charge complète de leurs spécifications associées.

Vous pouvez également consulter les résultats de la suite de tests JSON Schema sur Bowtie, qui exécute la suite de tests sur plusieurs implémentations à travers différentes plateformes.

AVERTISSEMENT Ma bibliothèque, _JsonSchema.Net_, n’est pas liée ni associée au site https://jsonschema.net, sauf qu’ils sont tous deux d’excellents outils JSON Schema.

Packages obsolètes

Ces packages ont également fait partie de cette suite mais ne sont plus pris en charge.

JsonSchema.Net.CodeGeneration
 Génération de code C# à partir de schémas Obsolète au profit de Corvus.JsonSchema

Prise en charge de _System.Text.Json_ par d’autres projets

Si vous ne trouvez pas ce que vous cherchez ici, essayez l’un de ces excellents projets :

  • JsonCons.Net par @danielaparker
  • JSON Pointer
  • JSON Patch
  • JSON Merge Patch
  • JSON Path
  • JMES Path
  • Corvus.JsonSchema par @mwadams
  • Validation JSON Schema avec génération de code C# en priorité
  • (d’autres à venir)

Si vous utilisez JSON pour faire quelque chose qui n’est couvert par aucune bibliothèque de cette suite ou des projets ci-dessus, n’hésitez pas à créer une demande de fonctionnalité.

Si vous maintenez ou connaissez un autre projet qui étend _System.Text.Json_ pour faire des choses intéressantes, j’aimerais l’ajouter ici, alors merci de créer une demande générale pour m’en informer.
