什么是 json-everything？

json-everything 背后的主要理念是在 _System.Text.Json_ 领域内确保常见的 JSON 功能获得良好的支持。

实现这一目标的第一步是检查是否已经有其他项目支持某项功能。

如果已经有，那么这里可能不会重复造轮子。往下看你会发现一些正在用 JSON 做很酷事情的其它项目列表。

否则，这个功能就有可能被考虑加入本项目。

有哪些功能？

实际上有多个“盒子”。每一项功能都被拆分为独立的库，你可以根据需要选择使用其中的一个或多个。

所有库都基于 .Net Standard 2.0 和 .Net 8 构建。

JsonSchema.Net
 JSON Schema，支持 draft 6 及以上版本
JsonSchema.Net.Generation
 从 .Net 类型自动生成 schema
JsonSchema.Net.Generation.DataAnnotations
 对 System.ComponentModel.DataAnnotations 的模式生成支持。
JsonSchema.Net.DataGeneration
 随机实例数据生成（由 Bogus 提供支持）
JsonSchema.Net.Data
 用于访问实例和外部数据的词汇表
JsonSchema.Net.ArrayExt
 用于扩展数组校验的词汇表
JsonSchema.Net.OpenApi
 OpenApi 3.1 词汇扩展（被 Graeae 使用）
JsonPath.Net
 JSON Path（RFC 9535）
JsonPatch.Net
 JSON Patch（RFC 6902）
JsonPointer.Net
 JSON Pointer（RFC 6901）与相对 JSON Pointer（规范）
JsonLogic
 JsonLogic（官网）
JsonE.Net
 JSON-e（官网）
Json.More.Net
 通用扩展，本应包含在 System.Text.Json[.Nodes] 中但实际上没有包含
Yaml2JsonNode
 在 YamlDotNet 的 YAML 文档模型与 JsonNode 之间的互相转换。

以上所有库都完全支持各自关联的规范。

_JsonSchema.Net_ 的错误信息翻译现已支持：

  • 意大利语 it
  • 挪威语 nb-NO
  • 波兰语 pl-PL
  • 俄语 ru
  • 西班牙语 es
  • 瑞典语 sv-SE
  • 土耳其语 tr-TR

注意 每种语言包都由其各自的 Nuget 包单独提供。

你还可以在 Bowtie 上查看 JSON Schema 测试套件的结果，该平台会在多个平台的不同实现上运行测试套件。

免责声明 我的库 _JsonSchema.Net_ 与 https://jsonschema.net 网站没有任何关系或关联，除了它们都是出色的 JSON Schema 工具。

已废弃的包

以下包曾经属于本套件，但现在已不再维护。

JsonSchema.Net.CodeGeneration
 从模式生成 C# 代码 已废弃，建议使用 Corvus.JsonSchema

其他项目对 _System.Text.Json_ 的支持

  • JsonCons.Net by @danielaparker
  • JSON Pointer
  • JSON Patch
  • JSON Merge Patch
  • JSON Path
  • JMES Path
  • Corvus.JsonSchema by @mwadams
  • 以 C# 代码生成为重点的 JSON Schema 校验
  • （后续还会有更多）

如果你用 JSON 做了某些本套件或上述项目未涵盖的事情，欢迎创建功能请求。

如果你维护或了解其他扩展 _System.Text.Json_ 的项目，并且它很酷，我很乐意将其列在这里，请创建一般问题让我知道。
