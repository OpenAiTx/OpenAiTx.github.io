Các nhà phát triển RustDesk không ủng hộ hoặc khuyến khích bất kỳ hành vi sử dụng phần mềm này một cách phi đạo đức hoặc bất hợp pháp nào. Việc lạm dụng, chẳng hạn như truy cập trái phép, kiểm soát hoặc xâm phạm quyền riêng tư, là hoàn toàn trái với hướng dẫn của chúng tôi. Một phần mềm điều khiển máy tính từ xa khác, được viết bằng Rust. Hoạt động ngay sau khi cài đặt, không cần cấu hình. Bạn hoàn toàn kiểm soát dữ liệu của mình, không lo ngại về bảo mật. Bạn có thể sử dụng máy chủ rendezvous/relay của chúng tôi, tự thiết lập máy chủ riêng, hoặc tự viết máy chủ rendezvous/relay của bạn.

RustDesk chào đón sự đóng góp từ tất cả mọi người. Xem CONTRIBUTING.md để được hướng dẫn bắt đầu.

Phụ thuộc

Phiên bản máy tính sử dụng Flutter hoặc Sciter (không còn hỗ trợ) cho giao diện, hướng dẫn này chỉ dành cho Sciter, vì dễ bắt đầu hơn. Xem CI để biết cách biên dịch phiên bản Flutter.

Vui lòng tự tải về thư viện động Sciter.

Windows | Linux | macOS

Các bước biên dịch cơ bản

  • Chuẩn bị môi trường phát triển Rust và môi trường biên dịch C++

  • Cài đặt vcpkg, và thiết lập biến môi trường VCPKG_ROOT chính xác

  • Windows: vcpkg install libvpx:x64-windows-static libyuv:x64-windows-static opus:x64-windows-static aom:x64-windows-static
  • Linux/macOS: vcpkg install libvpx libyuv opus aom

  • chạy cargo run

Cài đặt vcpkg

git clone https://github.com/microsoft/vcpkg
cd vcpkg
git checkout 2023.04.15
cd ..
vcpkg/bootstrap-vcpkg.sh
export VCPKG_ROOT=$HOME/vcpkg
vcpkg/vcpkg install libvpx libyuv opus aom

Sửa libvpx (Đối với Fedora)

cd vcpkg/buildtrees/libvpx/src
cd *
./configure
sed -i 's/CFLAGS+=-I/CFLAGS+=-fPIC -I/g' Makefile
sed -i 's/CXXFLAGS+=-I/CXXFLAGS+=-fPIC -I/g' Makefile
make
cp libvpx.a $HOME/vcpkg/installed/x64-linux/lib/
cd

Biên dịch

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
git clone --recurse-submodules https://github.com/rustdesk/rustdesk
cd rustdesk
mkdir -p target/debug
wget https://raw.githubusercontent.com/c-smile/sciter-sdk/master/bin.lnx/x64/libsciter-gtk.so
mv libsciter-gtk.so target/debug
VCPKG_ROOT=$HOME/vcpkg cargo run

Cách biên dịch bằng Docker

Bắt đầu bằng cách clone repository và build container Docker:

git clone https://github.com/rustdesk/rustdesk
cd rustdesk
git submodule update --init --recursive
docker build -t "rustdesk-builder" .

Sau đó, mỗi lần bạn cần biên dịch ứng dụng, hãy chạy lệnh sau:

docker run --rm -it -v $PWD:/home/user/rustdesk -v rustdesk-git-cache:/home/user/.cargo/git -v rustdesk-registry-cache:/home/user/.cargo/registry -e PUID="$(id -u)" -e PGID="$(id -g)" rustdesk-builder

Lưu ý rằng lần build đầu tiên có thể sẽ lâu hơn do cần tải về các phụ thuộc, các lần build sau sẽ nhanh hơn. Ngoài ra, nếu bạn cần chỉ định các đối số khác cho lệnh build, có thể thêm vào cuối lệnh ở vị trí . Ví dụ, nếu bạn muốn build phiên bản tối ưu hóa cho phát hành, hãy thêm --release vào sau lệnh trên. File thực thi kết quả sẽ nằm trong thư mục target trên hệ thống của bạn, có thể chạy bằng:

target/debug/rustdesk

Hoặc nếu bạn chạy file thực thi bản phát hành:

target/release/rustdesk

Hãy đảm bảo bạn đang chạy các lệnh này từ thư mục gốc của repository RustDesk, nếu không ứng dụng có thể không tìm thấy tài nguyên cần thiết. Lưu ý rằng các lệnh cargo khác như install hoặc run hiện không được hỗ trợ qua phương pháp này vì chúng sẽ cài đặt hoặc chạy chương trình bên trong container thay vì trên máy chủ.

Cấu trúc thư mục

  • libs/hbb_common: bộ mã hóa/giải mã video, cấu hình, bọc tcp/udp, protobuf, hàm hệ thống file cho truyền file và một số hàm tiện ích khác
  • libs/scrap: chụp màn hình
  • libs/enigo: điều khiển bàn phím/chuột theo từng nền tảng
  • libs/clipboard: triển khai sao chép và dán file cho Windows, Linux, macOS.
  • src/ui: giao diện Sciter (không còn sử dụng)
  • src/server: dịch vụ âm thanh/bảng tạm/nhập liệu/video và kết nối mạng
  • src/client.rs: khởi tạo kết nối peer
  • src/rendezvous_mediator.rs: giao tiếp với rustdesk-server, chờ kết nối trực tiếp từ xa (TCP hole punching) hoặc kết nối qua relay
  • src/platform: mã nguồn theo từng nền tảng
  • flutter: mã nguồn Flutter cho desktop và di động
  • flutter/web/js: JavaScript cho Flutter web client
