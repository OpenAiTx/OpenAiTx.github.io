RustDesk के डेवलपर्स इस सॉफ़्टवेयर के किसी भी अनैतिक या अवैध उपयोग का समर्थन या अनुमोदन नहीं करते हैं। दुरुपयोग, जैसे कि अनधिकृत पहुँच, नियंत्रण या गोपनीयता का उल्लंघन, हमारे दिशानिर्देशों के सख्त खिलाफ है। लेखक एप्लिकेशन के किसी भी दुरुपयोग के लिए जिम्मेदार नहीं हैं।

हमसे चैट करें: Discord | Twitter | Reddit | YouTube

एक और रिमोट डेस्कटॉप सॉफ्टवेयर, जो Rust में लिखा गया है। बिना किसी कॉन्फ़िगरेशन के तुरंत काम करता है। आपके डेटा पर आपका पूर्ण नियंत्रण है, सुरक्षा की कोई चिंता नहीं। आप हमारे रेंडेवस/रिले सर्वर का उपयोग कर सकते हैं, अपना खुद का सेटअप करें, या अपना खुद का रेंडेवस/रिले सर्वर लिखें।

RustDesk सभी से योगदान का स्वागत करता है। शुरू करने में सहायता के लिए देखें CONTRIBUTING.md।

सामान्य प्रश्न (FAQ)

बाइनरी डाउनलोड

नाइटली बिल्ड

निर्भरताएँ

डेस्कटॉप संस्करण GUI के लिए Flutter या Sciter (अप्रचलित) का उपयोग करते हैं, यह ट्यूटोरियल केवल Sciter के लिए है, क्योंकि यह शुरू करने में आसान और अधिक अनुकूल है। Flutter संस्करण को बिल्ड करने के लिए हमारा CI देखें।

कृपया Sciter डायनामिक लाइब्रेरी स्वयं डाउनलोड करें।

Windows | Linux | macOS

बिल्ड करने के कच्चे चरण

  • अपने Rust डेवलपमेंट एनवायरनमेंट और C++ बिल्ड एनवायरनमेंट की तैयारी करें

  • vcpkg इंस्टॉल करें, और VCPKG_ROOT एनवायरनमेंट वेरिएबल सही से सेट करें

  • Windows: vcpkg install libvpx:x64-windows-static libyuv:x64-windows-static opus:x64-windows-static aom:x64-windows-static
  • Linux/macOS: vcpkg install libvpx libyuv opus aom

  • cargo run चलाएँ

बिल्ड

sudo apt install -y zip g++ gcc git curl wget nasm yasm libgtk-3-dev clang libxcb-randr0-dev libxdo-dev \
        libxfixes-dev libxcb-shape0-dev libxcb-xfixes0-dev libasound2-dev libpulse-dev cmake make \
        libclang-dev ninja-build libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev libpam0g-dev

Fedora 28 (CentOS 8)

sudo yum -y install gcc-c++ git curl wget nasm yasm gcc gtk3-devel clang libxcb-devel libxdo-devel libXfixes-devel pulseaudio-libs-devel cmake alsa-lib-devel gstreamer1-devel gstreamer1-plugins-base-devel pam-devel

Arch (Manjaro)

sudo pacman -Syu --needed unzip git cmake gcc curl wget yasm nasm zip make pkg-config clang gtk3 xdotool libxcb libxfixes alsa-lib pipewire

vcpkg इंस्टॉल करें

git clone https://github.com/microsoft/vcpkg
cd vcpkg
git checkout 2023.04.15
cd ..
vcpkg/bootstrap-vcpkg.sh
export VCPKG_ROOT=$HOME/vcpkg
vcpkg/vcpkg install libvpx libyuv opus aom

libvpx ठीक करें (Fedora के लिए)

cd vcpkg/buildtrees/libvpx/src
cd *
./configure
sed -i 's/CFLAGS+=-I/CFLAGS+=-fPIC -I/g' Makefile
sed -i 's/CXXFLAGS+=-I/CXXFLAGS+=-fPIC -I/g' Makefile
make
cp libvpx.a $HOME/vcpkg/installed/x64-linux/lib/
cd

बिल्ड करें

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
git clone --recurse-submodules https://github.com/rustdesk/rustdesk
cd rustdesk
mkdir -p target/debug
wget https://raw.githubusercontent.com/c-smile/sciter-sdk/master/bin.lnx/x64/libsciter-gtk.so
mv libsciter-gtk.so target/debug
VCPKG_ROOT=$HOME/vcpkg cargo run

डॉकर के साथ कैसे बिल्ड करें

रिपॉजिटरी क्लोन करके और डॉकर कंटेनर बनाकर शुरू करें:

git clone https://github.com/rustdesk/rustdesk
cd rustdesk
git submodule update --init --recursive
docker build -t "rustdesk-builder" .

फिर, जब भी आपको एप्लिकेशन बिल्ड करना हो, निम्न कमांड चलाएँ:

docker run --rm -it -v $PWD:/home/user/rustdesk -v rustdesk-git-cache:/home/user/.cargo/git -v rustdesk-registry-cache:/home/user/.cargo/registry -e PUID="$(id -u)" -e PGID="$(id -g)" rustdesk-builder

ध्यान दें कि पहली बिल्ड में डिपेंडेंसी कैश होने से समय लग सकता है, बाद की बिल्ड तेज़ होंगी। यदि आपको बिल्ड कमांड के लिए अलग पैरामीटर देने हैं, तो आप कमांड के अंत में स्थान पर ऐसा कर सकते हैं। उदाहरण के लिए, यदि आप ऑप्टिमाइज़्ड रिलीज़ संस्करण बनाना चाहते हैं, तो ऊपर दिए गए कमांड के बाद --release जोड़ें। परिणामी एक्जीक्यूटेबल आपके सिस्टम के target फोल्डर में उपलब्ध होगा, और इसे चलाया जा सकता है:

target/debug/rustdesk

या, यदि आप रिलीज़ एक्जीक्यूटेबल चला रहे हैं:

target/release/rustdesk

कृपया सुनिश्चित करें कि आप ये कमांड RustDesk रिपॉजिटरी की रूट से चला रहे हैं, अन्यथा एप्लिकेशन आवश्यक संसाधन नहीं खोज पाएगा। ध्यान दें कि अन्य cargo सबकमांड जैसे install या run इस विधि के माध्यम से वर्तमान में समर्थित नहीं हैं क्योंकि वे प्रोग्राम को कंटेनर के अंदर इंस्टॉल या रन करेंगे, होस्ट पर नहीं।

फ़ाइल संरचना

  • libs/hbb_common: वीडियो कोडेक, कॉन्फ़िग, tcp/udp रैपर, प्रोटोबफ, फ़ाइल ट्रांसफ़र के लिए fs फंक्शंस, और कुछ अन्य उपयोगिता फंक्शंस
  • libs/scrap: स्क्रीन कैप्चर
  • libs/enigo: प्लेटफ़ॉर्म-विशिष्ट कीबोर्ड/माउस नियंत्रण
  • libs/clipboard: Windows, Linux, macOS के लिए फाइल कॉपी और पेस्ट का इम्प्लीमेंटेशन
  • src/ui: अप्रचलित Sciter UI (deprecated)
  • src/server: ऑडियो/क्लिपबोर्ड/इनपुट/वीडियो सेवाएँ, और नेटवर्क कनेक्शन
  • src/client.rs: एक पीयर कनेक्शन शुरू करें
  • src/rendezvous_mediator.rs: rustdesk-server से संवाद करें, रिमोट डायरेक्ट (TCP होल पंचिंग) या रिले कनेक्शन का इंतजार करें
  • src/platform: प्लेटफ़ॉर्म-विशिष्ट कोड
  • flutter: डेस्कटॉप और मोबाइल के लिए Flutter कोड
  • flutter/web/js: Flutter वेब क्लाइंट के लिए जावास्क्रिप्ट
