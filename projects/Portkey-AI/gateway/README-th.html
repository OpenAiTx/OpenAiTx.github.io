AI Gateway

#### เชื่อมต่อกับ LLM กว่า 250+ ตัว ด้วย API เดียวที่รวดเร็วและใช้งานง่าย

เอกสาร | องค์กร | Hosted Gateway | Changelog | API Reference


AI Gateway ถูกออกแบบมาเพื่อการเชื่อมต่อที่รวดเร็ว เชื่อถือได้ และปลอดภัย ไปยังโมเดลภาษา, วิชั่น, ออดิโอ และอิมเมจมากกว่า 1600+ ตัว เป็นโซลูชันน้ำหนักเบา โอเพ่นซอร์ส และพร้อมใช้งานระดับองค์กร ช่วยให้คุณเชื่อมต่อกับโมเดลภาษาใด ๆ ได้ภายในเวลาไม่ถึง 2 นาที

  • [x] เร็วมาก (latency <1ms) ด้วยขนาดไฟล์เล็กมาก (122kb)
  • [x] ผ่านการทดสอบสมบุกสมบัน ประมวลผลโทเค็นกว่า 10 พันล้านต่อวัน
  • [x] พร้อมใช้งานระดับองค์กร ด้วยความปลอดภัยที่เหนือกว่า รองรับการขยาย และการดีพลอยแบบกำหนดเอง


#### คุณสามารถทำอะไรกับ AI Gateway ได้บ้าง?

  • เชื่อมต่อกับ LLM ใดก็ได้ในเวลาไม่ถึง 2 นาที - เริ่มต้นใช้งานด่วน
  • ป้องกันระบบล่มด้วย การลองใหม่อัตโนมัติ และ Fallback
  • ขยายแอป AI ด้วย โหลดบาลานซ์ และ การกำหนดเส้นทางตามเงื่อนไข
  • ปกป้องการดีพลอย AI ด้วย Guardrails
  • ไปให้ไกลกว่าข้อความด้วย ความสามารถมัลติโหมด
  • และสำรวจการเชื่อมต่อ Agentic workflow

เริ่มต้นอย่างรวดเร็ว (2 นาที)

1. ติดตั้ง AI Gateway ของคุณ

# รัน gateway บนเครื่อง (ต้องการ Node.js และ npm)
npx @portkey-ai/gateway
Gateway จะรันที่ http://localhost:8787/v1
Gateway Console จะรันที่ http://localhost:8787/public/

คู่มือดีพลอย:   Portkey Cloud (แนะนำ)   Docker   Node.js   Cloudflare   Replit   อื่น ๆ...

2. ส่งคำขอแรกของคุณ

# pip install -qU portkey-ai
from portkey_ai import Portkey
ไคลเอนต์ที่ใช้งานร่วมกับ OpenAI

client = Portkey(
    provider="openai", # หรือ 'anthropic', 'bedrock', 'groq', ฯลฯ
    Authorization="sk-*" # API key ของผู้ให้บริการ
)
ส่งคำขอผ่าน AI Gateway ของคุณ

client.chat.completions.create(
    messages=[{"role": "user", "content": "สภาพอากาศเป็นอย่างไร?"}],
    model="gpt-4o-mini"
)

ไลบรารีที่รองรับ:   JS   Python   REST   OpenAI SDKs   Langchain   LlamaIndex   Autogen   CrewAI   เพิ่มเติม..

บน Gateway Console (http://localhost:8787/public/) คุณสามารถดู log ทั้งหมดของคุณได้ในที่เดียว

3. Routing & Guardrails

Configs ใน LLM gateway ช่วยให้คุณสร้างกฎการกำหนดเส้นทาง เพิ่มความน่าเชื่อถือ และตั้งค่า guardrail ได้ 
config = {
  "retry": {"attempts": 5},
  "output_guardrails": [{
    "default.contains": {"operator": "none", "words": ["Apple"]},
    "deny": True
  }]
}
แนบ config กับ client

client = client.with_options(config=config)
client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "ตอบแบบสุ่มโดยใช้ Apple หรือ Bat"}]
)
คำตอบจะเป็น "Bat" เสมอ เพราะ guardrail จะปฏิเสธทุกคำตอบที่มี "Apple" การตั้งค่า retry จะพยายามใหม่ 5 ครั้งก่อนจะยอมแพ้

คุณสามารถทำอะไรได้อีกมากมายกับ config ใน AI gateway ของคุณ ดูตัวอย่างเพิ่มเติม →

รับรายงานนี้

ฟีเจอร์หลัก

การกำหนดเส้นทางที่เชื่อถือได้

  • Fallbacks: สำรองไปยังผู้ให้บริการหรือโมเดลอื่นเมื่อคำขอผ่าน LLM gateway ล้มเหลว คุณสามารถระบุข้อผิดพลาดที่ต้องการให้เกิด fallback ได้ ช่วยเพิ่มความน่าเชื่อถือของแอปพลิเคชันของคุณ
  • Automatic Retries: ทำการลองส่งคำขอใหม่โดยอัตโนมัติสูงสุด 5 ครั้ง กลยุทธ์ backoff แบบเอ็กซ์โพเนนเชียลจะเว้นระยะการลองใหม่เพื่อป้องกันความหนาแน่นของเครือข่าย
  • Load Balancing: กระจายคำขอ LLM ไปยังหลาย API key หรือผู้ให้บริการ AI พร้อมน้ำหนัก เพื่อให้มั่นใจถึงความพร้อมใช้งานสูงและประสิทธิภาพที่เหมาะสม
  • Request Timeouts: จัดการ LLM และความหน่วงช้าโดยกำหนด request timeout อย่างละเอียด อนุญาตให้ระบบยกเลิกคำขออัตโนมัติเมื่อเกินระยะเวลาที่กำหนด
  • Multi-modal LLM Gateway: เรียกใช้งานโมเดล vision, audio (text-to-speech & speech-to-text) และ image generation จากผู้ให้บริการหลายราย — ทั้งหมดผ่าน signature แบบเดียวกับ OpenAI ที่คุ้นเคย
  • Realtime APIs: เรียกใช้งาน realtime API ที่เปิดตัวโดย OpenAI ผ่านเซิร์ฟเวอร์ websocket ที่เชื่อมต่อ

ความปลอดภัย & ความถูกต้อง

  • Guardrails: ตรวจสอบอินพุตและเอาต์พุตของ LLM ให้เป็นไปตามข้อกำหนดของคุณ เลือกจาก guardrails ที่สร้างไว้ล่วงหน้ากว่า 40 รายการ เพื่อให้มั่นใจว่าสอดคล้องกับมาตรฐานความปลอดภัยและความถูกต้อง คุณสามารถ นำ guardrails ของคุณเองมาใช้ หรือเลือกจาก พาร์ทเนอร์ของเรา
  • การจัดการคีย์อย่างปลอดภัย: ใช้คีย์ของคุณเองหรือสร้าง virtual key ได้ตามต้องการ
  • การควบคุมการเข้าถึงตามบทบาท: ควบคุมการเข้าถึงอย่างละเอียดสำหรับผู้ใช้, เวิร์กสเปซ และ API key ของคุณ
  • การปฏิบัติตามข้อกำหนด & ความเป็นส่วนตัวของข้อมูล: AI Gateway รองรับมาตรฐาน SOC2, HIPAA, GDPR และ CCPA

การจัดการต้นทุน

  • Smart caching: แคชผลลัพธ์จาก LLM เพื่อลดต้นทุนและปรับปรุง latency รองรับทั้งการแคชแบบ simple และ semantic*
  • การวิเคราะห์การใช้งาน: ติดตามและวิเคราะห์การใช้งาน AI และ LLM ของคุณ รวมถึงปริมาณคำขอ, latency, ต้นทุน และอัตราข้อผิดพลาด
  • Provider optimization*: เปลี่ยนไปใช้ผู้ให้บริการที่มีต้นทุนคุ้มค่าที่สุดโดยอัตโนมัติตามรูปแบบการใช้งานและโมเดลราคา

การทำงานร่วมกัน & เวิร์กโฟลว์

  • รองรับ Agents: เชื่อมต่อกับเฟรมเวิร์ก agent ยอดนิยมอย่างไร้รอยต่อ เพื่อสร้างแอปพลิเคชัน AI ที่ซับซ้อน Gateway สามารถเชื่อมต่อกับ Autogen, CrewAI, LangChain, LlamaIndex, Phidata, Control Flow และ Custom Agents ได้อย่างสมบูรณ์
  • การจัดการ Prompt Template*: สร้าง, จัดการ และเวอร์ชัน prompt template ของคุณร่วมกันผ่าน prompt playground แบบสากล


* มีในเวอร์ชัน hosted และ enterprise

Gateway Enterprise Version

ทำให้แอป AI ของคุณ น่าเชื่อถือ และ รองรับอนาคต พร้อมทั้งรับรอง ความปลอดภัยของข้อมูล และ ความเป็นส่วนตัว อย่างครบถ้วน

✅  การจัดการคีย์อย่างปลอดภัย - สำหรับการควบคุมการเข้าถึงตามบทบาทและการติดตาม

✅  การลบข้อมูล PII - ลบข้อมูลอ่อนไหวออกจาก request อัตโนมัติเพื่อป้องกันการรั่วไหล
