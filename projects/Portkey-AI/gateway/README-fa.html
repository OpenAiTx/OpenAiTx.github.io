دروازه هوش مصنوعی (AI Gateway)

#### مسیر به بیش از ۲۵۰ مدل زبانی بزرگ (LLM) با یک API سریع و کاربرپسند

مستندات | نسخه سازمانی | دروازه میزبان‌شده | تغییرات | مرجع API


دروازه هوش مصنوعی برای مسیر‌دهی سریع، قابل اعتماد و امن به بیش از ۱۶۰۰ مدل زبانی، تصویری، صوتی و تصویری طراحی شده است. این راهکار سبک، متن‌باز و آماده برای سازمان‌ها است که به شما اجازه می‌دهد در کمتر از ۲ دقیقه با هر مدل زبانی ادغام شوید.

  • [x] بسیار سریع (کمتر از ۱ میلی‌ثانیه تاخیر) با حجم کم (۱۲۲ کیلوبایت)
  • [x] آزموده در میدان عمل با پردازش بیش از ۱۰ میلیارد توکن در هر روز
  • [x] مناسب سازمان‌ها با امنیت افزایش‌یافته، مقیاس‌پذیری و استقرار سفارشی


#### با دروازه هوش مصنوعی چه کارهایی می‌توان کرد؟

  • ادغام با هر مدل LLM در کمتر از ۲ دقیقه - شروع سریع
  • جلوگیری از قطعی‌ها با تلاش مجدد خودکار و راهکارهای جایگزین
  • مقیاس‌پذیری برنامه‌های هوش مصنوعی با تعادل بار و مسیردهی شرطی
  • محافظت از استقرارهای هوش مصنوعی با محافظ‌ها
  • فراتر از متن با قابلیت‌های چندوجهی
  • و در نهایت، کشف ادغام جریان کاری عاملی

۲. اولین درخواست خود را ارسال کنید

# pip install -qU portkey-ai
from portkey_ai import Portkey
کلاینت سازگار با OpenAI

client = Portkey(
    provider="openai", # یا 'anthropic'، 'bedrock'، 'groq' و غیره
    Authorization="sk-*" # کلید API ارائه‌دهنده
)
ارسال درخواست از طریق دروازه هوش مصنوعی

client.chat.completions.create(
    messages=[{"role": "user", "content": "آب و هوا چطور است؟"}],
    model="gpt-4o-mini"
)

کتابخانه‌های پشتیبانی‌شده:   جاوااسکریپت   پایتون   REST   OpenAI SDKs   Langchain   LlamaIndex   Autogen   CrewAI   بیشتر...

در کنسول دروازه (http://localhost:8787/public/) می‌توانید همه لاگ‌های محلی خود را یکجا مشاهده کنید.

۳. مسیر‌دهی و محافظ‌ها

Configs در دروازه LLM به شما اجازه می‌دهد قواعد مسیر‌دهی بسازید، قابلیت اطمینان اضافه کنید و محافظ‌ها را تنظیم کنید. 
config = {
  "retry": {"attempts": 5},
  "output_guardrails": [{
    "default.contains": {"operator": "none", "words": ["Apple"]},
    "deny": True
  }]
}
اضافه کردن پیکربندی به کلاینت

client = client.with_options(config=config)
client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "به صورت تصادفی با Apple یا Bat پاسخ بده"}]
)
این همیشه با "Bat" پاسخ می‌دهد زیرا محافظ همه پاسخ‌هایی که شامل "Apple" باشد را رد می‌کند. پیکربندی تلاش مجدد تا ۵ بار تلاش می‌کند تا موفق شود.

مدل‌های زبانی در تولید ۲۰۲۵ (LLMs in Prod'25)

یافته‌هایی از تحلیل بیش از ۲ تریلیون توکن در ۹۰+ منطقه و ۶۵۰+ تیم در محیط تولید. از این گزارش چه انتظاری داشته باشید:

  • روندهایی که پذیرش هوش مصنوعی و رشد ارائه‌دهندگان LLM را شکل می‌دهد.
  • معیارهایی برای بهینه‌سازی سرعت، هزینه و قابلیت اطمینان.
  • استراتژی‌هایی برای مقیاس‌پذیری سیستم‌های هوش مصنوعی در سطح تولید.

امکانات اصلی

مسیر‌دهی قابل اطمینان

  • مسیر جایگزین (Fallbacks): با استفاده از درگاه LLM، در صورت شکست درخواست‌ها، به ارائه‌دهنده یا مدل دیگری منتقل شوید. می‌توانید خطاهایی که باید باعث اجرای مسیر جایگزین شوند را مشخص کنید. این کار قابلیت اطمینان برنامه شما را افزایش می‌دهد.
  • تلاش مجدد خودکار (Automatic Retries): به طور خودکار تا ۵ بار درخواست‌های ناموفق را مجدداً ارسال کنید. یک استراتژی بازگشت نمایی (exponential backoff) تلاش‌های مجدد را با فاصله زمانی تنظیم می‌کند تا از ازدحام شبکه جلوگیری شود.
  • توزیع بار (Load Balancing): درخواست‌های LLM را با وزن‌دهی بین چند کلید API یا ارائه‌دهنده هوش مصنوعی توزیع کنید تا از دسترس‌پذیری بالا و عملکرد بهینه اطمینان حاصل شود.
  • زمان‌سنجی درخواست‌ها (Request Timeouts): با تنظیم زمان‌سنج دقیق برای درخواست‌ها، مدل‌های LLM پرهزینه یا با تاخیر را کنترل کنید؛ این امکان، خاتمه خودکار درخواست‌هایی که از مدت زمان معین عبور می‌کنند را فراهم می‌کند.
  • درگاه چندرسانه‌ای LLM: مدل‌های بینایی، صوت (متن به گفتار و گفتار به متن)، و تولید تصویر را از چندین ارائه‌دهنده با امضای OpenAI فراخوانی کنید.
  • APIهای بلادرنگ (Realtime APIs): APIهای بلادرنگ ارائه شده توسط OpenAI را از طریق سرور WebSocket یکپارچه فراخوانی کنید.

امنیت و دقت

  • ریل‌های محافظ (Guardrails): ورودی و خروجی LLM خود را برای تطابق با بررسی‌های تعیین‌شده تأیید کنید. از میان بیش از ۴۰ ریل محافظ آماده انتخاب کنید تا مطابقت با استانداردهای امنیت و دقت را تضمین نمایید. می‌توانید ریل محافظ خود را اضافه کنید یا از میان شرکای ما انتخاب کنید.
  • مدیریت امن کلیدها: از کلیدهای خود استفاده کنید یا کلیدهای مجازی را به صورت لحظه‌ای تولید نمایید.
  • کنترل دسترسی مبتنی بر نقش: کنترل دسترسی دقیق برای کاربران، محیط‌های کاری و کلیدهای API.
  • تطبیق‌پذیری و حفظ حریم خصوصی داده‌ها: درگاه هوش مصنوعی با استانداردهای SOC2، HIPAA، GDPR و CCPA سازگار است.

مدیریت هزینه

  • ذخیره‌سازی هوشمند (Smart caching): پاسخ‌های مدل‌های LLM را ذخیره کنید تا هزینه‌ها کاهش یافته و تاخیر بهبود یابد. از ذخیره‌سازی ساده و معنایی* پشتیبانی می‌کند.
  • تحلیل مصرف (Usage analytics): مصرف هوش مصنوعی و مدل‌های LLM خود را پایش و تحلیل کنید؛ شامل حجم درخواست، تاخیر، هزینه‌ها و نرخ خطا.
  • بهینه‌سازی ارائه‌دهنده*: به صورت خودکار بر اساس الگوهای مصرف و مدل‌های قیمت‌گذاری، به مقرون‌به‌صرفه‌ترین ارائه‌دهنده سوئیچ کنید.

همکاری و جریان‌کارها

  • پشتیبانی از عامل‌ها (Agents): با ادغام بی‌وقفه با فریمورک‌های محبوب عامل‌ها، برنامه‌های هوش مصنوعی پیچیده بسازید. این درگاه با Autogen، CrewAI، LangChain، LlamaIndex، Phidata، Control Flow و حتی عامل‌های سفارشی یکپارچه می‌شود.
  • مدیریت قالب‌های پرامپت*: قالب‌های پرامپت خود را به صورت گروهی ایجاد، مدیریت و نسخه‌بندی کنید و از محیطی یکپارچه بهره ببرید.


* در نسخه‌های میزبانی‌شده و سازمانی موجود است

نسخه سازمانی Gateway

برنامه هوش مصنوعی خود را قابل‌اعتمادتر و سازگار با آینده کنید، در حالی که امنیت کامل داده‌ها و حفظ حریم خصوصی را تضمین می‌کنید.

✅  مدیریت امن کلیدها - برای کنترل دسترسی مبتنی بر نقش و رهگیری
✅  کشینگ ساده و معنایی - برای پاسخ‌دهی سریع‌تر به پرسش‌های تکراری و صرفه‌جویی در هزینه
✅  کنترل دسترسی و قوانین ورودی - کنترل اینکه کدام IP و مناطق جغرافیایی به استقرارهای شما متصل شوند
✅  حذف اطلاعات شخصی (PII Redaction) - حذف خودکار داده‌های حساس از درخواست‌ها برای جلوگیری از افشای ناخواسته
✅  انطباق با SOC2، ISO، HIPAA، GDPR - جهت رعایت بهترین شیوه‌های امنیتی
✅  پشتیبانی حرفه‌ای - به همراه اولویت‌بندی ویژگی‌ها
