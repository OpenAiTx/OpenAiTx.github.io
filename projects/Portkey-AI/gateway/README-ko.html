AI Gateway

#### 하나의 빠르고 친근한 API로 250개 이상의 LLM을 라우팅하세요

문서 | 엔터프라이즈 | 호스팅 게이트웨이 | 변경 로그 | API 참조


AI Gateway는 1600개 이상의 언어, 비전, 오디오, 이미지 모델에 대해 빠르고 신뢰할 수 있으며 안전하게 라우팅하도록 설계되었습니다. 경량이며 오픈소스, 엔터프라이즈 수준의 솔루션으로, 2분 이내에 어떤 언어 모델과도 통합할 수 있습니다.

  • [x] 눈부시게 빠름 (<1ms 지연)과 작은 용량(122kb)
  • [x] 검증된 안정성, 매일 100억 토큰 이상 처리
  • [x] 엔터프라이즈 준비 완료, 보안 강화, 대규모 확장, 맞춤형 배포 지원

빠른 시작 (2분)

1. AI Gateway 설치

# 게이트웨이를 로컬에서 실행 (Node.js와 npm 필요)
npx @portkey-ai/gateway
게이트웨이는 http://localhost:8787/v1에서 실행 중입니다.
게이트웨이 콘솔은 http://localhost:8787/public/에서 실행 중입니다.

배포 가이드:   Portkey Cloud (권장)   Docker   Node.js   Cloudflare   Replit   기타...

2. 첫 번째 요청 보내기

# pip install -qU portkey-ai
from portkey_ai import Portkey
OpenAI 호환 클라이언트

client = Portkey(
    provider="openai", # 또는 'anthropic', 'bedrock', 'groq' 등
    Authorization="sk-*" # 프로바이더 API 키
)
AI Gateway를 통해 요청 보내기

client.chat.completions.create(
    messages=[{"role": "user", "content": "What's the weather like?"}],
    model="gpt-4o-mini"
)

지원 라이브러리:   JS   Python   REST   OpenAI SDKs   Langchain   LlamaIndex   Autogen   CrewAI   더보기..

Gateway Console(http://localhost:8787/public/)에서 모든 로컬 로그를 한 곳에서 확인할 수 있습니다.

3. 라우팅 & 가드레일

LLM 게이트웨이의 Configs를 이용해 라우팅 규칙 생성, 신뢰성 추가, 가드레일 설정이 가능합니다. 
config = {
  "retry": {"attempts": 5},
  "output_guardrails": [{
    "default.contains": {"operator": "none", "words": ["Apple"]},
    "deny": True
  }]
}
클라이언트에 config 적용

client = client.with_options(config=config)
client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Reply randomly with Apple or Bat"}]
)
이 설정은 "Apple"이 포함된 모든 응답을 거부하므로 항상 "Bat"으로 응답합니다. retry 설정은 5회까지 재시도합니다.

리포트 다운로드

주요 기능

신뢰성 높은 라우팅

  • 폴백(Fallbacks): LLM 게이트웨이를 사용하여 요청 실패 시 다른 공급자나 모델로 폴백할 수 있습니다. 폴백을 트리거할 오류를 지정할 수 있으며, 애플리케이션의 신뢰성을 높여줍니다.
  • 자동 재시도(Automatic Retries): 실패한 요청을 최대 5회까지 자동으로 재시도합니다. 지수 백오프 전략을 적용하여 네트워크 과부하를 방지합니다.
  • 로드 밸런싱(Load Balancing): 여러 API 키나 AI 공급자에 LLM 요청을 가중치 기반으로 분산시켜 고가용성과 최적의 성능을 보장합니다.
  • 요청 타임아웃(Request Timeouts): 세분화된 요청 타임아웃을 설정하여, 지정된 시간을 초과하는 요청을 자동으로 종료함으로써 LLM 및 지연 문제를 관리합니다.
  • 멀티모달 LLM 게이트웨이(Multi-modal LLM Gateway): 친숙한 OpenAI 시그니처를 사용하여, 여러 공급자의 비전, 오디오(텍스트-음성 & 음성-텍스트), 이미지 생성 모델을 호출할 수 있습니다.
  • 실시간 API(Realtime APIs): 통합 웹소켓 서버를 통해 OpenAI가 제공하는 실시간 API를 호출할 수 있습니다.

보안 & 정확성

  • 가드레일(Guardrails): LLM 입력 및 출력을 지정한 검증 절차에 따라 검증할 수 있습니다. 40개 이상의 사전 구축된 가드레일 중 선택하여 보안 및 정확성 표준을 준수할 수 있습니다. 직접 가드레일을 제공하거나, 여러 파트너사의 서비스를 선택할 수 있습니다.
  • 보안 키 관리(Secure Key Management): 직접 키를 사용하거나, 가상 키를 즉시 생성할 수 있습니다.
  • 역할 기반 접근 제어(Role-based access control): 사용자, 워크스페이스, API 키에 대한 세분화된 접근 제어가 가능합니다.
  • 컴플라이언스 & 데이터 프라이버시(Compliance & Data Privacy): AI 게이트웨이는 SOC2, HIPAA, GDPR, CCPA를 준수합니다.

비용 관리

  • 스마트 캐싱(Smart caching): LLM 응답을 캐싱하여 비용을 절감하고 지연 시간을 개선합니다. 단순 및 시맨틱* 캐싱을 지원합니다.
  • 사용량 분석(Usage analytics): 요청량, 지연 시간, 비용, 오류율 등 AI 및 LLM 사용량을 모니터링하고 분석할 수 있습니다.
  • 공급자 최적화(Provider optimization)*: 사용 패턴과 가격 모델에 따라 가장 비용 효율적인 공급자로 자동 전환합니다.

협업 & 워크플로우

  • 에이전트 지원(Agents Support): 인기 있는 에이전트 프레임워크와 매끄럽게 통합하여 복잡한 AI 애플리케이션을 구축할 수 있습니다. 게이트웨이는 Autogen, CrewAI, LangChain, LlamaIndex, Phidata, Control Flow, 심지어 커스텀 에이전트까지 지원합니다.
  • 프롬프트 템플릿 관리(Prompt Template Management)*: 범용 프롬프트 플레이그라운드를 통해 프롬프트 템플릿을 공동으로 생성, 관리, 버전 관리할 수 있습니다.


* 호스팅 및 엔터프라이즈 버전에서 제공

Gateway 엔터프라이즈 버전
