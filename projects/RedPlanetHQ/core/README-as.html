🔥 গৱেষণা উল্লিখিত বিষয়সমূহ

CORE মেমৰীয়ে Locomo ডেটাছেটত সকলো যুক্তি-বুদ্ধি সম্পৰ্কীয় টাস্কত 88.24% গড় সফলতা লাভ কৰিছে, আন মেমৰি প্ৰদানকাৰীসমূহতকৈ উল্লেখযোগ্যভাৱে অধিক। অধিক তথ্যৰ বাবে এই ব্লগ চাওক।

(1) Single-hop প্ৰশ্নসমূহত এটা মাত্ৰ ছেছন ভিত্তিক উত্তৰ প্ৰয়োজন হয়; (2) Multi-hop প্ৰশ্নসমূহত একাধিক ভিন্ন ছেছনৰ তথ্য সংহতি কৰি উত্তৰ প্ৰয়োজন হয়; (3) Open-domain জ্ঞানৰ প্ৰশ্নসমূহত বক্তাৰ প্ৰদান কৰা তথ্যৰ সৈতে বাহ্যিক জ্ঞান যেনে সাধাৰণ জ্ঞান বা বিশ্বতথ্য সংযোগ কৰি উত্তৰ দিব পাৰি; (4) Temporal reasoning প্ৰশ্নসমূহত সময়-সংক্রান্ত তথ্য আৰু কথোপকথনৰ ভিতৰত সময়ৰ সংকেত ধৰি উত্তৰ দিব পাৰি;

পৰ্যাৱৰণ

সমস্যা

ডেভেলপাৰসকলে AI টুলসমূহক পুনৰাই প্ৰসংগ ব্যাখ্যা কৰাত সময় নষ্ট কৰে। Claude-ত টোকেন সীমা শেষ? নতুনকৈ আৰম্ভ কৰক, সকলো হেৰুৱাই পেলাওক। ChatGPT/Claude-ৰ পৰা Cursor-লৈ যাব? পুনৰাই প্ৰসংগ ব্যাখ্যা কৰক। আপোনাৰ কথোপকথন, সিদ্ধান্ত, আৰু অন্তৰ্দৃষ্টি ছেছনসমূহৰ মাজত হেৰুৱাই যায়। প্ৰতিটো নতুন AI টুলৰ লগত প্ৰসংগ পৰিবৰ্তনৰ মূল্য বৃদ্ধি হয়।

সমাধান - CORE (Contextual Observation & Recall Engine)

CORE হৈছে আপোনাৰ সকলো AI টুলৰ বাবে এক মুক্ত-উৎস, একত্ৰিত, স্থায়ী মেমৰি স্তৰ। আপোনাৰ প্ৰসংগ Cursor-ৰ পৰা Claude-লৈ, ChatGPT-লৈ, Claude Code-লৈ আপোনাৰ সৈতে থাকে। এটা জ্ঞান গ্ৰাফে ক'লে, কেতিয়া, আৰু কিয় স্মৰণ ৰাখে। এটা বাৰ সংযোগ কৰক, সকলো ঠাইত মনত ৰাখক। প্ৰসংগ ব্যৱস্থাপনা বন্ধ কৰক আৰু গঢ়ি তুলা আৰম্ভ কৰক।

🚀 CORE নিজে-হোস্টিং

CORE নিজৰ অৱকাঠামোত চলাব বিচাৰেনে? নিজে-হোস্টিংয়ে আপোনাক আপোনাৰ ডাটা আৰু স্থাপনাৰ সম্পূৰ্ণ নিয়ন্ত্ৰণ দিয়ে।

প্ৰয়োজনীয়তাসমূহ:

  • Docker (20.10.0+) আৰু Docker Compose (2.20.0+) ইনষ্টল কৰা
  • OpenAI API key

> মুক্ত-উৎস মডেলৰ টোকা: আমি Ollama আৰু GPT মডেল আদিৰ দৰে OSS বিকল্পসমূহ পৰীক্ষা কৰিছোঁ, কিন্তু সিহঁতৰ তথ্য সংগ্ৰহ আৰু গ্ৰাফ গুণমান আশানুৰূপ নহ'ল। আমি সদায় ভাল বিকল্প বিচাৰি আছোঁ।

CORE কেনেকৈ memory সৃষ্টি কৰে

CORE-ৰ ingestion pipeline-ত evolving context ধৰা বাবে চাৰিটা পৰ্যায় আছে:

  • স্বাভাবিকীকৰণ: নতুন তথ্যক সাম্প্ৰতিক প্ৰসংগত সংযোগ কৰে, দীঘল নথিপত্ৰ সমূহক সুসংগত ভাগত বিভাজিত কৰে য'ত পৰস্পৰ-সংযোগ অক্ষুণ্ণ থাকে, আৰু শব্দসমূহক মানিকৰণ কৰে যাতে CORE-এ জ্ঞান আহৰণ কৰাৰ সময়ত পৰিস্কাৰ, প্ৰসংগভিত্তিক ইনপুটৰ সৈতে কাম কৰে, বিশৃঙ্খল পাঠ্যৰ সৈতে নহয়।
  • উদ্ধৰণ: স্বাভাবিকীকৃত পাঠ্যৰ পৰা অৰ্থ আহৰণ কৰে, সত্তা (মানুহ, সঁজুলি, প্ৰকল্প, ধাৰণা) চিনাক্ত কৰে, তেওঁলোকক প্ৰসংগ, উৎস, আৰু সময়ৰ সৈতে বিবৃতিত ৰূপান্তৰিত কৰে, আৰু সম্পৰ্ক মানচিত্ৰ কৰে। উদাহৰণস্বৰূপে, “আমিয়ে CORE Next.js-ত লিখিছোঁ” ই হয়: সত্তা (CORE, Next.js), বিবৃতি (CORE Next.js ব্যৱহাৰ কৰি উন্নয়ন কৰা হৈছিল), আৰু সম্পৰ্ক (উন্নয়ন কৰা হৈছিল)।
  • সমাধান: বৈৰোধ চিনাক্ত কৰে, পছন্দসমূহ কেনেকৈ বিকাশ হয় তাক অনুসৰণ কৰে, আৰু প্ৰমাণসহ বহু দৃষ্টিভংগী সংৰক্ষণ কৰে যাতে স্মৃতিয়ে আপোনাৰ সম্পূৰ্ণ যাত্ৰা প্ৰতিফলিত কৰে, কেৱল শেহতীয়া দৃশ্য নহয়।
  • গ্ৰাফ সংহতি: সত্তা, বিবৃতি, আৰু ঘটনাসমূহক এটা সময়ভিত্তিক জ্ঞান গ্ৰাফত সংযোগ কৰে, য'ত তথ্যসমূহক তেওঁলোকৰ প্ৰসংগ আৰু ইতিহাসৰ সৈতে সংযোগ কৰে, পৃথক তথ্যক জীৱন্ত জ্ঞান ৱেবত ৰূপান্তৰিত কৰে যি এজেন্টে বাস্তৱতে ব্যৱহাৰ কৰিব পাৰে।

ফলাফল: এটা সমতল ডাটাবেইজৰ পরিবর্তে, CORE-এ আপোনাক এনে এটা স্মৃতি প্ৰদান কৰে যি আপোনাৰ সৈতে বৃদ্ধি আৰু পৰিবর্তিত হয় - প্ৰসংগ, বিকাশ, আৰু অধিকাৰ সংৰক্ষণ কৰি ৰাখে যাতে এজেন্টে বাস্তৱতে ইয়াক ব্যৱহাৰ কৰিব পাৰে।

CORE-এ কেনেকৈ স্মৃতিৰ পৰা স্মৰণ কৰে

আপুনি CORE-লৈ প্ৰশ্ন কৰিলে, ই কেৱল পাঠ্য সন্ধান নকৰে - ই আপোনাৰ সমগ্ৰ জ্ঞান গ্ৰাফত ডুব দিয়ে সকলোতকৈ সহায়ক উত্তৰ বিচাৰিবলৈ।

  • সন্ধান: CORE-এ একেলগে বহু দিশৰ পৰা স্মৃতিৰ সন্ধান কৰে - ঠিক মিলৰ বাবে শব্দ সন্ধান, ভিন্নভাৱে প্ৰকাশিত হলেও সম্পৰ্কীয় চিন্তাৰ বাবে অৰ্থ সন্ধান, আৰু সংযুক্ত ধাৰণাসমূহৰ মাজত সংযোগ অনুসৰণ কৰিবলৈ গ্ৰাফ পৰিক্রমণ।
  • পুনঃ-শ্ৰেণীবিন্যাস: উদ্ধাৰ কৰা ফলাফলসমূহ পুনৰ শ্ৰেণীবিন্যাস কৰি আটাইতকৈ প্ৰাসংগিক আৰু বৈচিত্ৰ্যময় ফলাফলসমূহ আগবঢ়ায়, যাতে আপুনি কেৱল স্পষ্ট মিলসমূহ নহয়, গভীৰ সংযোগসমূহো দেখিবলৈ পায়।
  • পৰিষ্কাৰ: CORE-এ সময়, বিশ্বস্ততা, আৰু সম্পৰ্কৰ দৃঢ়তাৰ ভিত্তিত বুদ্ধিমত্তা পৰিষ্কাৰ প্ৰয়োগ কৰে, যাতে কেৱল আটাইতকৈ অৰ্থপূৰ্ণ জ্ঞান ওপৰলৈ আহে।
  • আউটপুট: আপুনি পায় তথ্য (স্পষ্ট বিবৃতি) আৰু ঘটনা (তেওঁলোকৰ মূল প্ৰসংগ) উভয়, যাতে স্মৰণ সদায় প্ৰসংগ, সময়, আৰু কাহিনিত ভিত্তি থাকে।

ফলাফল: CORE-এ কেৱল তথ্য স্মৰণ নকৰে - ই সঠিক প্ৰসংগ, সময়, আৰু কাহিনিত স্মৰণ কৰে, যাতে এজেন্টে আপোনাৰ দৰে উত্তৰ দিব পাৰে।
