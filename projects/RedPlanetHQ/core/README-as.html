🔥 গৱেষণা হাইলাইট

CORE মেমৰীয়ে Locomo dataset-ত সকলো reasoning টাস্কত গড় 88.24% সঠিকতা লাভ কৰিছে, আন memory provider-সমূহক অধিক পাৰ হৈ। অধিক তথ্যৰ বাবে এই ব্লগ চাওক।

(1) Single-hop প্ৰশ্নসমূহত এটা session-ৰ আধাৰত উত্তৰ লাগে; (2) Multi-hop প্ৰশ্নসমূহত একাধিক session-ৰ তথ্য একত্ৰিত কৰি উত্তৰ লাগে; (3) Open-domain knowledge প্ৰশ্নসমূহত বক্তাই দিয়া তথ্য আৰু বাহ্যিক জ্ঞান (যেনে সাধাৰণ জ্ঞান বা বিশ্বৰ তথ্য) সংহতি কৰি উত্তৰ দিব পাৰি; (4) Temporal reasoning প্ৰশ্নসমূহত temporal reasoning আৰু কণ্ভাৰচেশ্যনত সময়-সংক্রান্ত data cues ধৰি উত্তৰ দিব পাৰি;

সমস্যা

Developer-এ AI tools-ত পুনৰায় context ব্যাখ্যা কৰি সময় নষ্ট কৰে। Claude-ত token limit পোৱা? নতুনকৈ আৰম্ভ কৰক আৰু সকলো হেৰুৱাই পেলাওক। ChatGPT/Claude-ৰ পৰা Cursor-লৈ switch কৰিলে? পুনৰ context ব্যাখ্যা কৰক। আপোনাৰ conversation, সিদ্ধান্ত, আৰু insight session-সমূহৰ মাজত গুচি যায়। প্ৰত্যেক নতুন AI tool-ত context switching-ৰ খৰচ বৃদ্ধি পায়।

সমাধান - CORE (Contextual Observation & Recall Engine)

CORE হৈছে আপোনাৰ সকলো AI tool-ৰ বাবে এটা একত্ৰিত, স্থায়ী মেমৰি layer। আপোনাৰ context Cursor-ৰ পৰা Claude, ChatGPT-লৈ আৰু Claude Code-লৈ অনুসৰণ কৰে। এটা knowledge graph-এ কোনে কি, কেতিয়া, আৰু কিয় কৈছিল সেয়া মনত ৰাখে। এটা বাৰ connect কৰক, সকলোত মনত ৰাখক। Context manage কৰা বন্ধ কৰক আৰু নিৰ্মাণ আৰম্ভ কৰক।

CORE কেনেকৈ মেম'ৰি সৃষ্টি কৰে

CORE-ৰ ইনজেশ্যন পাইপলাইনত বিকাশমান প্ৰসংগ ধৰা-পকড়ৰ বাবে চাৰিটা পৰ্যায় থাকে:

  • Normalization: নতুন তথ্যক সাম্প্ৰতিক প্ৰসংগত সংযোগ কৰে, দীঘল ডকুমেণ্টসমূহক সংগঠিত ভাগত ভাঙে য’ত আন্তঃসংযোগ থাকে, আৰু শব্দসমূহক মানকৰণ কৰে যাতে CORE-এ জ্ঞান উলিওৱাৰ সময়ত পৰিষ্কৃত, প্ৰসংগযুক্ত ইনপুটৰ সৈতে কাম কৰে, অগছল টেক্সটৰ সৈতে নহয়।
  • Extraction: মানকৃত টেক্স্টৰ পৰা অৰ্থ উলিয়ায়, একাই চিনাক্ত কৰে (ব্যক্তি, টুল, প্ৰকল্প, ধাৰণা), সেইবোৰক প্ৰসংগ, উৎস, আৰু সময়সহ বিবৃতি ৰূপে ৰূপান্তৰ কৰে, আৰু সম্পৰ্কসমূহ মানচিত্রত আঁকে। উদাহৰণস্বৰূপে, “আমিয়ে CORE Next.js-ত লিখিছিলো” এইটো হয়: একাই (Core, Next.js), বিবৃতি (CORE Next.js ব্যৱহাৰ কৰি উন্নয়ন কৰা হৈছিল), আৰু সম্পৰ্ক (উন্নয়ন কৰা হৈছিল ব্যৱহাৰ কৰি)।
  • Resolution: বিৰোধ চিনাক্ত কৰে, পছন্দসমূহ কেনেকৈ বিকাশ হয় তাক অনুসৰণ কৰে, আৰু একাধিক দৃষ্টিভংগী প্ৰমাণসহ সংৰক্ষণ কৰে যাতে মেম'ৰিয়ে আপোনাৰ সম্পূৰ্ণ যাত্ৰা প্ৰতিফলিত কৰে, কেৱল নতুন চিত্র নহয়।
  • Graph Integration: একাই, বিবৃতি, আৰু ঘটনাসমূহক ক্ৰমবদ্ধ জ্ঞান গ্ৰাফত সংযোগ কৰে, যি তথ্যসমূহক তেওঁলোকৰ প্ৰসংগ আৰু ইতিহাসৰ সৈতে সংযোগ কৰে, পৃথক তথ্যক জীৱন্ত জ্ঞান জালত ৰূপান্তৰ কৰে যি এজেন্টে বাস্তৱতে ব্যৱহাৰ কৰিব পাৰে।

ফলাফল: সমতল ডেটাবেজৰ সলনি, CORE-এ আপোনাক এটা মেম'ৰি দিয়ে যি আপোনাৰ সৈতে বৃদ্ধি আৰু পৰিবৰ্তন হয় - প্ৰসংগ, বিকাশ, আৰু মালিকানা সংৰক্ষণ কৰে যাতে এজেন্টে বাস্তৱতে ব্যৱহাৰ কৰিব পাৰে।

CORE কেনেকৈ মেম'ৰি পৰা স্মৰণ কৰে

আপুনি CORE-লৈ প্ৰশ্ন কৰিলে, CORE-এ কেৱল টেক্সট চোৱা নকৰে - আপোনাৰ সম্পূৰ্ণ জ্ঞান গ্ৰাফত খনন কৰে আটাইতকৈ উপযোগী উত্তৰ বিচাৰি।

  • Search: CORE-এ একে সময়তে বহুমুখী দৃষ্টিভংগীৰে মেম'ৰি চায় - ঠিক মিলৰ বাবে শব্দ অনুসন্ধান, ভিন্নভাৱে প্ৰকাশ কৰা হলেও সম্পর্কিত ধাৰণাৰ বাবে সেমান্টিক অনুসন্ধান, আৰু সংযুক্ত ধাৰণাসমূহৰ মাজত সংযোগ অনুসৰণ কৰিবলৈ গ্ৰাফ ট্রেভাৰ্ছেল।
  • Re-Rank: উদ্ধৃত ফলাফলসমূহ পুনৰবিন্যাস কৰা হয়, আটাইতকৈ প্ৰাসংগিক আৰু বৈচিত্ৰ্যময় ফলাফলসমূহ আগত ৰাখে, যাতে আপুনি কেৱল স্পষ্ট মিল নহয়, গভীৰ সংযোগো দেখিব পাৰে।
  • Filtering: CORE-এ সময়, বিশ্বাসযোগ্যতা, আৰু সম্পৰ্কৰ দৃঢ়তাৰ ভিত্তিত বুদ্ধিমত্তা ফিল্টাৰ প্ৰয়োগ কৰে, যাতে কেৱল আটাইতকৈ অৰ্থপূর্ণ জ্ঞান মূৰত উঠে।
  • আউটপুট: আপুনি পোৱা তথ্যবোৰ (স্পষ্ট বিবৃতি) আৰু পৰ্ববোৰ (মূল প্ৰসংগ য'ৰ পৰা আহিছে), গতিকে CORE-ত স্মৰণ সদায় প্ৰসংগ, সময় আৰু গল্পত ভিত্তি থকা।

ফলাফল: CORE কেৱল তথ্য স্মৰণ নকৰে - ই সেইবোৰ ঠিক প্ৰসংগ, সময় আৰু গল্পত স্মৰণ কৰে, যেতিয়া এজেন্টে আপোনাৰ দৰে প্ৰতিক্ৰিয়া কৰিব পাৰে।
