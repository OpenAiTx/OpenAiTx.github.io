CORE: Bộ Não Số của Bạn

Nó Ghi Nhớ. Nó Suy Nghĩ. Nó Hành Động

CORE là bộ não số của bạn - một AI hợp nhất, luôn hoạt động, ghi nhớ ngữ cảnh, làm việc với các tác nhân AI của bạn và thực hiện trên các công cụ của bạn.

Mã nguồn mở. Tự triển khai. Dữ liệu của bạn, kiểm soát của bạn.

Điều Gì Tạo Nên Một Bộ Não Số

Nó Ghi Nhớ Mọi Thứ

Mỗi cuộc trò chuyện, quyết định, sở thích và mối quan hệ đều được lưu dưới dạng tri thức có cấu trúc trong đồ thị tri thức thời gian. Không phải văn bản thô mà là các sự thật được phân loại (sở thích, quyết định, mục tiêu, chỉ thị) sẽ hiện lên đúng lúc bạn cần. Hãy hỏi về điều gì đó từ vài tuần trước và câu trả lời vẫn ở đó, với đầy đủ ngữ cảnh.

Nó Có Thể Thực Hiện Hành Động

Tạo các issue trên GitHub, gửi tin nhắn Slack, quản lý lịch của bạn mà không cần rời khỏi cuộc trò chuyện. Quy trình đa bước phức tạp cũng được: yêu cầu agent lập trình kiểm tra Sentry để tìm lỗi mới, tạo một issue trên GitHub cho mỗi lỗi, phân công đúng kỹ sư dựa trên git blame, và đăng tổng kết lên kênh Slack của nhóm bạn, tất cả chỉ từ một prompt. Nó Suy Nghĩ và Làm Việc Qua Các Hệ Thống

Các công cụ của bạn bị chia nhỏ. Claude Code không thể nói chuyện với WhatsApp của bạn. Cursor không thể kiểm tra lịch của bạn. Agent của CORE phá vỡ những rào cản này - nó là người điều phối kết nối bộ nhớ, toolkit và các ứng dụng của bạn lại với nhau.

Những gì CORE Agent có thể làm:

  • Hiểu ý định: phân loại nhu cầu của bạn và điều hướng đến công cụ và bộ nhớ phù hợp
  • Khởi tạo agent khác: mở một phiên Claude Code hoặc phiên trình duyệt từ WhatsApp để sửa lỗi hoặc nghiên cứu gì đó
  • Phối hợp quy trình đa bước: lấy bối cảnh từ bộ nhớ, hành động qua nhiều ứng dụng, và báo cáo lại
  • Làm việc từ bất cứ đâu: nhắn từ WhatsApp, email, Slack hoặc dashboard web — cùng một bộ não, cùng một bối cảnh

Nó Có Thể Chủ Động Hành Động

Bạn không nên phải yêu cầu mọi thứ. CORE giám sát các sự kiện - email mới, cảnh báo GitHub, thay đổi lịch — và đánh giá chúng dựa trên những gì nó biết về bạn. Một email khách hàng đến; CORE đã biết cách bạn xử lý mối quan hệ đó và soạn sẵn phản hồi phù hợp. Một cảnh báo Sentry xuất hiện ban đêm; CORE khởi tạo một phiên Claude Code để điều tra.

Kiến trúc

CORE được xây dựng trên ba lớp thực hiện bốn trụ cột này:

  • Bộ nhớ: Một đồ thị tri thức thời gian lưu trữ các tập, thực thể và dữ kiện đã được phân loại. Mỗi thông tin đều được phân loại (sở thích, quyết định, chỉ thị, mục tiêu, v.v.) và kết nối theo thời gian. Điều này giúp truy xuất dựa trên ý định thay vì dựa trên từ khóa. Tài liệu →

  • Toolkit: Một lớp hành động hợp nhất dành cho bất kỳ agent tương thích MCP nào. Kết nối ứng dụng của bạn một lần (GitHub, Linear, Slack, Gmail, Calendar, v.v.) và mọi công cụ AI được kết nối đều truy cập hơn 200 hành động qua một endpoint duy nhất. Tài liệu →

  • CORE Agent: Người điều phối kết nối bộ nhớ và toolkit lại với nhau. Nó hiểu ý định, tìm kiếm bộ nhớ, chọn công cụ, khởi tạo các agent khác, và có thể hành động chủ động dựa trên trigger và quy tắc. Tài liệu →

Claude Code (Khuyến nghị: Dùng Plugin) 

npm install -g @redplanethq/corebrain
Sau đó trong Claude Code:

/plugin marketplace add redplanethq/core
/plugin install core_brain

Khởi động lại Claude Code và chạy /mcp để xác thực.

Plugin sẽ tự động tải hồ sơ cá nhân của bạn (sở thích, quy tắc, quyết định) vào mỗi lần bắt đầu phiên và ghi nhận các cuộc hội thoại vào bộ nhớ khi bạn hoàn thành.

Claude Code (MCP Thủ Công)

claude mcp add --transport http --scope user core-memory https://app.getcore.me/api/v1/mcp?source=Claude-Code
Sau đó gõ /mcp và mở core-memory để xác thực.

Dành cho Nhà Phát Triển Agent

Xây dựng agent AI? Hãy chuyển giao bộ nhớ và tích hợp cho CORE để bạn có thể tập trung vào logic của agent.

  • Chuyển giao bộ nhớ — Sử dụng đồ thị tri thức thời gian của CORE như bộ nhớ dài hạn cho agent của bạn. Lưu trữ hội thoại, truy xuất ngữ cảnh với tìm kiếm dựa trên ý định, và để agent tự xây dựng kiến thức theo thời gian mà không cần quản lý vector DB hay graph riêng.
  • Chuyển giao tích hợp — Kết nối ứng dụng một lần trên CORE, agent của bạn sẽ có công cụ MCP cho tất cả. Không cần xây dựng luồng OAuth, không bảo trì API, không cần viết mã cho từng tích hợp.
  • Xây dựng qua MCP hoặc API — Kết nối agent với CORE qua MCP (một điểm cuối) hoặc sử dụng trực tiếp REST API.
