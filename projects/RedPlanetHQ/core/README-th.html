🔥 ไฮไลท์งานวิจัย

หน่วยความจำ CORE ทำความแม่นยำเฉลี่ย 88.24% ในชุดข้อมูล Locomo สำหรับทุกงานด้านการให้เหตุผล เหนือกว่าผู้ให้บริการหน่วยความจำรายอื่นอย่างชัดเจน ดูข้อมูลเพิ่มเติมได้ที่ บล็อกนี้

(1) คำถามแบบ Single-hop ต้องการคำตอบจากเซสชันเดียว; (2) คำถามแบบ Multi-hop ต้องการการสังเคราะห์ข้อมูลจากหลายเซสชันที่แตกต่างกัน; (3) คำถามความรู้แบบเปิดสามารถตอบโดยผสานข้อมูลที่ผู้พูดให้กับความรู้ภายนอก เช่น สามัญสำนึกหรือข้อเท็จจริงของโลก; (4) คำถามเกี่ยวกับการให้เหตุผลเชิงเวลา สามารถตอบโดยใช้เหตุผลเชิงเวลาและจับข้อมูลเกี่ยวกับเวลาในบทสนทนา;

ปัญหา

นักพัฒนาสูญเสียเวลาในการอธิบายบริบทซ้ำกับเครื่องมือ AI ติดข้อจำกัดโทเคนใน Claude? ต้องเริ่มใหม่และสูญเสียทุกอย่าง เปลี่ยนจาก ChatGPT/Claude ไป Cursor? อธิบายบริบทใหม่อีกครั้ง บทสนทนา การตัดสินใจ และข้อมูลเชิงลึกของคุณหายไปทุกครั้งที่เปลี่ยนเซสชัน ทุกครั้งที่ใช้เครื่องมือ AI ใหม่ ค่าใช้จ่ายในการเปลี่ยนบริบทยิ่งเพิ่มขึ้น

ทางแก้ - CORE (Contextual Observation & Recall Engine)

CORE คือเลเยอร์หน่วยความจำแบบเปิดที่เป็นหนึ่งเดียวและถาวรสำหรับเครื่องมือ AI ทั้งหมดของคุณ บริบทของคุณจะตามคุณไปตั้งแต่ Cursor ถึง Claude ถึง ChatGPT ถึง Claude Code กราฟความรู้เดียวจำได้ว่าใครพูดอะไร เมื่อไร และทำไม เชื่อมต่อครั้งเดียว จำได้ทุกที่ เลิกจัดการบริบทแล้วเริ่มสร้างสรรค์ได้เลย

สร้างกราฟหน่วยความจำของคุณใน 5 นาที:

  • สมัครสมาชิก ที่ core.heysol.ai และสร้างบัญชีของคุณ
  • เพิ่มหน่วยความจำแรกของคุณ - แชร์ข้อมูลเกี่ยวกับตัวคุณ

  • ดูกราฟหน่วยความจำของคุณ และดูว่า CORE เชื่อมโยงข้อเท็จจริงโดยอัตโนมัติอย่างไร
  • ทดสอบใช้งาน - ถามว่า "คุณรู้อะไรเกี่ยวกับฉันบ้าง?" ในส่วนสนทนา
  • เชื่อมต่อกับเครื่องมือของคุณ:
  • Claude & Cursor - เขียนโค้ดพร้อมบริบท
  • CLaude Code CLI & Codex CLI - เขียนโค้ดผ่านเทอร์มินัลพร้อมหน่วยความจำ
  • ติดตั้งส่วนขยายเบราว์เซอร์ - นำหน่วยความจำของคุณไปยังเว็บไซต์ใดก็ได้
  • Linear, Github - เพิ่มบริบทของโปรเจกต์โดยอัตโนมัติ

CORE สร้างหน่วยความจำอย่างไร

กระบวนการ ingest ของ CORE มีสี่ขั้นตอนเพื่อจับบริบทที่เปลี่ยนแปลงตลอดเวลา:

  • Normalization: เชื่อมโยงข้อมูลใหม่กับบริบทล่าสุด, แบ่งเอกสารยาวเป็นส่วนที่เข้าใจง่ายพร้อมคงการเชื่อมโยงระหว่างกัน และปรับมาตรฐานคำศัพท์ เพื่อให้ตอนที่ CORE สกัดความรู้จะได้ทำงานกับอินพุตที่สะอาดและเป็นบริบท ไม่ใช่ข้อความที่ยุ่งเหยิง
  • Extraction: ดึงความหมายจากข้อความที่ถูกปรับรูปแบบโดยการระบุเอนทิตี้ (บุคคล, เครื่องมือ, โครงการ, แนวคิด), แปลงเป็นข้อความที่มีบริบท แหล่งที่มา และเวลา พร้อมแมปความสัมพันธ์ ตัวอย่างเช่น “เราเขียน CORE ด้วย Next.js” จะกลายเป็น: เอนทิตี้ (Core, Next.js), ข้อความ (CORE ถูกพัฒนาโดยใช้ Next.js), และความสัมพันธ์ (ถูกพัฒนาโดยใช้)
  • Resolution: ตรวจจับความขัดแย้ง, ติดตามความเปลี่ยนแปลงของความชอบ, และเก็บมุมมองที่หลากหลายพร้อมแหล่งที่มาแทนที่จะเขียนทับ เพื่อให้หน่วยความจำสะท้อนการเดินทางทั้งหมดของคุณ ไม่ใช่แค่ภาพล่าสุด
  • Graph Integration: เชื่อมโยงเอนทิตี้ ข้อความ และเหตุการณ์เข้าเป็นกราฟความรู้ตามเวลา ที่เชื่อมโยงข้อเท็จจริงกับบริบทและประวัติศาสตร์ เปลี่ยนข้อมูลโดดเดี่ยวให้กลายเป็นเครือข่ายความรู้ที่ตัวแทนสามารถใช้งานได้จริง

ผลลัพธ์: แทนที่จะเป็นฐานข้อมูลแบบแบน CORE มอบหน่วยความจำที่เติบโตและเปลี่ยนแปลงไปกับคุณ—รักษาบริบท การพัฒนา และความเป็นเจ้าของ เพื่อให้ตัวแทนสามารถใช้งานได้จริง

CORE เรียกคืนข้อมูลจากหน่วยความจำอย่างไร

เมื่อคุณถามคำถามกับ CORE มันไม่ใช่แค่ค้นหาข้อความ—แต่เจาะลึกเข้าไปในกราฟความรู้ทั้งหมดเพื่อหาคำตอบที่เป็นประโยชน์ที่สุด

  • Search: CORE ค้นหาข้อมูลในหน่วยความจำจากหลายมุมมองพร้อมกัน—ค้นหาคำหลักเพื่อหาคำที่ตรงกัน, ค้นหาเชิงความหมายเพื่อหาความคิดที่เกี่ยวข้องแม้ใช้คำต่างกัน, และเดินตามกราฟเพื่อเชื่อมโยงแนวคิดที่เกี่ยวข้องกัน
  • Re-Rank: ผลลัพธ์ที่ดึงมาได้จะถูกจัดอันดับใหม่เพื่อเน้นข้อมูลที่เกี่ยวข้องและหลากหลายที่สุด เพื่อให้คุณไม่เห็นแค่คำตอบพื้น ๆ แต่ยังได้เห็นการเชื่อมโยงลึก ๆ ด้วย
  • Filtering: CORE ใช้ตัวกรองอัจฉริยะตามเวลา, ความน่าเชื่อถือ, และความแข็งแรงของความสัมพันธ์ เพื่อให้เฉพาะความรู้ที่มีความหมายมากที่สุดปรากฏขึ้น
  • ผลลัพธ์: คุณจะได้รับทั้งข้อเท็จจริง (แถลงการณ์ที่ชัดเจน) และเหตุการณ์ (บริบทดั้งเดิมที่ข้อมูลเหล่านั้นมาจาก) ดังนั้นการเรียกคืนข้อมูลจึงมีพื้นฐานในบริบท เวลา และเรื่องราวเสมอ

ผลลัพธ์: CORE ไม่เพียงแต่เรียกคืนข้อเท็จจริง - แต่ยังเรียกคืนข้อเท็จจริงในบริบท เวลา และเรื่องราวที่ถูกต้อง เพื่อให้ตัวแทนสามารถตอบสนองได้เหมือนที่คุณจดจำ

🔒 ความปลอดภัย

CORE ให้ความสำคัญกับความปลอดภัย เรานำแนวปฏิบัติมาตรฐานอุตสาหกรรมมาใช้เพื่อปกป้องข้อมูลของคุณ:

  • การเข้ารหัสข้อมูล: ข้อมูลทั้งหมดระหว่างส่ง (TLS 1.3) และขณะพัก (AES-256)
  • การยืนยันตัวตน: OAuth 2.0 และการยืนยันตัวตนด้วย magic link
  • การควบคุมการเข้าถึง: การแยกพื้นที่ทำงานและการกำหนดสิทธิ์ตามบทบาท
  • การรายงานช่องโหว่: โปรดรายงานปัญหาด้านความปลอดภัยที่ harshith@tegon.ai

สำหรับข้อมูลความปลอดภัยโดยละเอียด ดูที่ Security Policy
