당신만의 AI OS.

단순히 여는 챗봇이 아닙니다. 항상 켜져 있고, 항상 관찰하는 AI입니다. 이름을 붙이고, 원하는 대로 모양을 만들고, 사용하는 모든 것에 연결하세요. 당신이 일하는 어떤 방식으로도 접근하세요. 오픈 소스, 자체 호스팅, 영원히 당신의 것입니다.

작동하는 모습 보기

CORE가 일반 텍스트 작업을 받아 GitHub와 메모리에서 컨텍스트를 수집하고, 작업을 계획하고, Claude Code 세션을 실행한 후 PR을 여는 모습을 확인해보세요:

항상 지켜보고. 항상 준비되어 있습니다.

대부분의 AI 도구는 요청을 기다립니다. CORE는 관찰합니다.

앱에 연결하면 모든 앱의 활동을 모니터링합니다. 클라이언트로부터 이메일이 도착하거나, GitHub 이슈가 할당되거나, Sentry 경고가 발생하거나, 회의가 끝납니다. CORE는 이를 감지하고, 메모리와 설정된 스킬을 확인한 후 직접 처리하거나 판단을 위해 표면화합니다. 사용자가 직접 트리거하지 않아도 스스로 감지합니다.

Claude Code, Codex 또는 Cursor에 CORE 플러그인을 설치하면 에이전트 대화도 모니터링됩니다. 논의된 컨텍스트, 내린 결정, 작성된 코드 모두가 메모리 지식 그래프에 반영됩니다. CORE가 다음 작업을 맡을 때 이미 지난 세션에서 무슨 일이 있었는지 알고 있습니다.

동작 시, 직접적으로 실행합니다. 이메일 답장, Linear 이슈 업데이트, GitHub PR 생성, Slack 메시지 발송, 터미널 명령 실행, 브라우저 조작, Claude Code나 Codex 세션을 어떤 인터페이스에서든 시작할 수 있습니다. 공항에서 WhatsApp 메시지를 보내면 CORE가 코딩 세션을 실행하고 탑승 전에 PR을 열어둘 수 있습니다.

어디서 독자적으로 동작하고 어디서 당신의 호출을 기다릴지는 당신이 결정합니다. Sentry 알림은 자동으로 처리하지만 병합 전에는 항상 확인을 요청합니다. 코딩 세션 시작 전에 계획을 승인합니다. 이메일 발송 전에는 확인을 요구합니다. 자율성의 수준은 작업별, 앱별, 행동별로 직접 설정할 수 있습니다.

접근 방식 네 가지

챗봇은 하나의 인터페이스만 있습니다. OS는 여러 개가 있습니다.

음성. Mac에서 Ctrl+Option을 누르고 해야 할 일을 말하세요. CORE가 백그라운드에서 실행하여 흐름을 방해하지 않습니다.

스크래치패드. 오늘의 페이지를 열고 [ ] 이슈 #47에서 인증 버그 수정이라고 적으세요. CORE가 3분 내로 이를 인식하고, 저장소와 메모리에서 맥락을 불러와 계획을 작성합니다.

메시징. WhatsApp, Slack, Telegram. 공항, 휴대폰, 침대 어디서든 작업을 전송하세요. 메시지 출처와 관계없이 CORE는 전체 맥락을 파악합니다.

채팅. 대시보드를 열고 직접 대화하세요. 어떤 비서를 대하듯이. 위임 전에 대화를 주고받고 싶을 때 유용합니다.

하나의 AI, 네 가지 인터페이스, 모두 동일한 메모리와 맥락이 적용됩니다.

나만의 방식으로

이름을 붙이세요. 말투를 고르세요. 모든 곳에 적용될 규칙을 설정하세요.

다섯 가지 내장 성격 중에서 선택하세요 — TARS의 건조한 효율성, Alfred의 충직한 격식, Hudson의 따뜻한 실용성 — 또는 직접 작성할 수 있습니다. 이 성격은 실행하는 모든 작업, 작성하는 모든 계획, 보내는 모든 메시지에 반영됩니다.

음성 상호작용을 위한 목소리를 고르세요. 그 후부터 모든 인터페이스에서 설정한 AI의 목소리와 행동이 유지됩니다.

CORE의 실제 활용

말로 지시하고, PR로 돌아오세요.

Ctrl+Option을 누르고 말하세요: "이슈 #312의 결제 플로우에서 경쟁 상태를 수정해 줘."

CORE가 이슈를 불러오고, 관련 커밋과 Slack 스레드를 가져오고, 계획을 작성한 뒤 Claude Code 세션을 실행합니다. 당신은 diff만 확인하면 됩니다. 자리에 앉아 있을 필요도 없습니다.

밤에 적고, 아침에 리뷰하세요.

스크래치패드: [ ] 오늘 밤 11시에 백로그 처리 시작

CORE는 Linear, GitHub, 그리고 메모리에서 작업을 불러와 우선순위를 매기고, 당신이 자는 동안 처리합니다. 매끄럽게 실행된 작업은 리뷰를 기다리고 있습니다. 막힌 세션은 멈춘 탭이 아니라, 하나의 명확한 질문과 함께 돌아옵니다.

어디서든 메시지를 보내세요.

공항에서 WhatsApp 메시지: "auth 리팩터링 배포해줘."

CORE는 이미 브랜치, 컨텍스트, 그리고 당신의 선호도를 알고 있습니다. Railway에서 실행 중입니다. 탑승 전에 세션을 시작합니다.

경고를 사건이 되기 전에 조사하세요.

Sentry가 새벽 2시에 알림을 보냅니다. CORE는 관련 트레이스와 이전 사건을 메모리에서 불러와, 해결책을 제안하고 Slack에 알립니다: "이슈 #847, 해결책 제안됨, 리뷰 대기 중." 당신은 휴대폰으로 승인합니다.

이미 당신의 한 주를 아는 브리핑을 받으세요.

매일 오전 8시에 반복되는 작업. CORE는 이메일, GitHub, Linear, Slack에서 정보를 불러와, 실제로 신경 써야 할 것만 표면화하고, 필요 없는 것은 건너뜁니다. 후속 조치는 자동으로 작업으로 전환됩니다.

빠른 시작

오픈소스 및 자체 호스팅. 데이터는 귀하의 인프라에 남아 있습니다.

진행할 경로를 선택하세요:

| 하고 싶은 것 | 방법 | |---|---| | 내 컴퓨터에서 체험 | 아래의 원스텝 설치 실행(도커 필요) | | 서버 또는 VPS에 배포 | Railway 원클릭 배포 | | 맥 앱 사용 | 대기자 명단 등록 |

CORE 설치 및 시작:

npm install -g @redplanethq/corebrain && corebrain setup
설정 마법사는 설치 디렉토리, AI 공급자, API 키, 그리고 챗 모델을 입력하도록 요청합니다. 비밀키를 생성하고, 스택을 시작하며, http://localhost:3033을 엽니다.

대부분의 로컬 설치는 Docker가 실행된 후 몇 분 정도 소요됩니다.

또는 Railway에 배포하기:

게이트웨이 연결하기를 통해 CORE가 코딩 에이전트를 실행하고, 브라우저를 제어하며, 로컬 폴더에 접근할 수 있습니다:

corebrain login
corebrain gateway setup

필수 조건: Docker 20.10+, Docker Compose 2.20+, 4 vCPU / 8GB RAM

전체 셀프 호스팅 가이드

설정 후 첫 번째 작업 (2분 소요):

  • 스크래치패드를 엽니다 (http://localhost:3033의 데일리 페이지)
  • [ ] 내 열린 GitHub 이슈 요약 또는 평소에 직접 하던 작업을 입력합니다
  • CORE가 3분 이내로 작업을 감지하고, 연결된 앱에서 컨텍스트를 수집하여 계획을 초안합니다
  • 계획을 승인하면 CORE가 실행하고 결과를 반환합니다

첫 앱 연결하기

문서

  • 메모리 - 시계열 지식 그래프, 사실 분류, 의도 기반 검색
  • 스크래치패드 - 작업과 아이디어가 시작되는 일일 작업 공간
  • 작업 - 계획, 상태, 반복 작업 및 작업 범위 컨텍스트
  • 툴킷 - MCP를 통한 50개 이상 앱, 1000개 이상의 액션
  • CORE 에이전트 - 트리거, 메모리, 도구 및 실행
  • 게이트웨이 - WhatsApp, Slack, Telegram, 이메일, 웹, API 접근
  • 스킬 - 반복 가능한 워크플로우를 위한 재사용 가능한 지침
  • 셀프 호스팅 - 전체 배포 가이드
  • 변경 로그 - 출시 내역

벤치마크

CORE는 단일-홉, 멀티-홉, 오픈도메인, 시간적 추론 전반에서 LoCoMo 벤치마크 기준 88.24% 평균 정확도를 달성했습니다. 전체 결과 및 베이스라인 비교는 벤치마크 레포를 참고하세요.

  • 전송 중 TLS 1.3 적용

  • 귀하의 데이터는 모델 학습에 절대 사용되지 않습니다
  • 완전한 격리를 위한 자체 호스팅 지원
