त्वरित प्रारंभ

कई त्वरित प्रारंभ विकल्प उपलब्ध हैं:

  • नवीनतम रिलीज़ डाउनलोड करें
  • रेपो क्लोन करें: git clone https://github.com/twbs/bootstrap.git
  • npm के साथ इंस्टॉल करें: npm install bootstrap@v5.3.6
  • yarn के साथ इंस्टॉल करें: yarn add bootstrap@v5.3.6
  • Bun के साथ इंस्टॉल करें: bun add bootstrap@v5.3.6
  • Composer के साथ इंस्टॉल करें: composer require twbs/bootstrap:5.3.6
  • NuGet के साथ इंस्टॉल करें: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

फ्रेमवर्क की सामग्री, टेम्प्लेट, उदाहरण, और अधिक जानकारी के लिए आरंभ करें पृष्ठ पढ़ें।

स्थिति

क्या शामिल है

डाउनलोड के भीतर आपको निम्नलिखित निर्देशिकाएं और फाइलें मिलेंगी, जो सामान्य संसाधनों को तार्किक रूप से समूहित करती हैं और संकलित तथा मिनिफाइड दोनों प्रकार उपलब्ध कराती हैं।

डाउनलोड सामग्री

``text bootstrap/ ├── css/ │ ├── bootstrap-grid.css │ ├── bootstrap-grid.css.map │ ├── bootstrap-grid.min.css │ ├── bootstrap-grid.min.css.map │ ├── bootstrap-grid.rtl.css │ ├── bootstrap-grid.rtl.css.map │ ├── bootstrap-grid.rtl.min.css │ ├── bootstrap-grid.rtl.min.css.map │ ├── bootstrap-reboot.css │ ├── bootstrap-reboot.css.map │ ├── bootstrap-reboot.min.css │ ├── bootstrap-reboot.min.css.map │ ├── bootstrap-reboot.rtl.css │ ├── bootstrap-reboot.rtl.css.map │ ├── bootstrap-reboot.rtl.min.css │ ├── bootstrap-reboot.rtl.min.css.map │ ├── bootstrap-utilities.css │ ├── bootstrap-utilities.css.map │ ├── bootstrap-utilities.min.css │ ├── bootstrap-utilities.min.css.map │ ├── bootstrap-utilities.rtl.css │ ├── bootstrap-utilities.rtl.css.map │ ├── bootstrap-utilities.rtl.min.css │ ├── bootstrap-utilities.rtl.min.css.map │ ├── bootstrap.css │ ├── bootstrap.css.map │ ├── bootstrap.min.css │ ├── bootstrap.min.css.map │ ├── bootstrap.rtl.css │ ├── bootstrap.rtl.css.map │ ├── bootstrap.rtl.min.css │ └── bootstrap.rtl.min.css.map └── js/ ├── bootstrap.bundle.js ├── bootstrap.bundle.js.map ├── bootstrap.bundle.min.js ├── bootstrap.bundle.min.js.map ├── bootstrap.esm.js ├── bootstrap.esm.js.map ├── bootstrap.esm.min.js ├── bootstrap.esm.min.js.map ├── bootstrap.js ├── bootstrap.js.map ├── bootstrap.min.js └── bootstrap.min.js.map `

हम संकलित CSS और JS (bootstrap.), साथ ही संकलित और मिनिफाइड CSS और JS (bootstrap.min.) उपलब्ध कराते हैं। सोर्स मैप्स (bootstrap.*.map) कुछ ब्राउज़र डेवलपर टूल्स के साथ उपयोग के लिए उपलब्ध हैं। बंडल किए गए JS फाइलें (bootstrap.bundle.js और मिनिफाइड bootstrap.bundle.min.js) में Popper शामिल है।

बग्स और फीचर अनुरोध

क्या आपके पास कोई बग या फीचर अनुरोध है? कृपया पहले इश्यू दिशानिर्देश पढ़ें और मौजूदा तथा बंद इश्यू खोजें। यदि आपकी समस्या या विचार अभी संबोधित नहीं किया गया है, कृपया एक नया इश्यू खोलें।

दस्तावेज़ीकरण

बूटस्ट्रैप का दस्तावेज़, इस रेपो की रूट डायरेक्टरी में शामिल है, Astro के साथ बनाया गया है और GitHub Pages पर सार्वजनिक रूप से होस्ट किया गया है: । दस्तावेज़ को स्थानीय रूप से भी चलाया जा सकता है।

दस्तावेज़ खोज Algolia's DocSearch द्वारा संचालित है।

दस्तावेज़ को स्थानीय रूप से चलाना

  • Node.js डिपेंडेंसीज, जिसमें Astro (साइट बिल्डर) शामिल है, इंस्टॉल करने के लिए npm install चलाएं।
  • वितरित CSS और JavaScript फाइलों, साथ ही हमारे दस्तावेज़ीकरण संसाधनों को पुनर्निर्माण के लिए npm run test (या कोई विशिष्ट npm स्क्रिप्ट) चलाएं।
  • रूट /bootstrap डायरेक्टरी से, कमांड लाइन में npm run docs-serve चलाएं।
  • अपने ब्राउज़र में http://localhost:9001/ खोलें, और बस।

Astro का उपयोग करने के बारे में और जानने के लिए इसकी दस्तावेज़ीकरण पढ़ें।

पिछले रिलीज़ के लिए दस्तावेज़ीकरण

आप हमारे सभी पिछले रिलीज़ के दस्तावेज़ पर पा सकते हैं।

पिछले रिलीज़ और उनका दस्तावेज़ भी डाउनलोड के लिए उपलब्ध है।

योगदान

कृपया हमारे योगदान दिशानिर्देश पढ़ें। इसमें इश्यू खोलने, कोडिंग मानकों, और विकास पर नोट्स के लिए निर्देश शामिल हैं।

इसके अलावा, यदि आपके पुल अनुरोध में JavaScript पैच या फीचर्स हैं, तो आपको संबंधित यूनिट परीक्षण शामिल करना होगा। सभी HTML और CSS को कोड गाइड का पालन करना चाहिए, जिसे Mark Otto द्वारा बनाए रखा गया है।

संपादक प्राथमिकताएं सामान्य टेक्स्ट संपादकों में आसान उपयोग के लिए एडिटर कॉन्फिग में उपलब्ध हैं। और अधिक पढ़ें और प्लगइन्स डाउनलोड करें: ।

समुदाय

बूटस्ट्रैप के विकास पर अपडेट प्राप्त करें और प्रोजेक्ट मेंटेनर्स और समुदाय के सदस्यों के साथ बातचीत करें।

  • @getbootstrap on X को फॉलो करें।
  • आधिकारिक बूटस्ट्रैप ब्लॉग पढ़ें और सदस्यता लें।
  • हमारे GitHub Discussions पर प्रश्न पूछें और जानें।
  • समुदाय Discord या Bootstrap subreddit पर चर्चा करें, प्रश्न पूछें, और अधिक।
  • IRC में Bootstrappers के साथ चैट करें। irc.libera.chat सर्वर पर, #bootstrap चैनल में।
  • कार्यान्वयन सहायता Stack Overflow (टैग bootstrap-5) पर मिल सकती है।
  • डेवलपर्स को bootstrap` कीवर्ड का उपयोग करना चाहिए जब npm या इसी तरह के वितरण माध्यमों के माध्यम से बूटस्ट्रैप की कार्यक्षमता में संशोधन या जोड़ने वाले पैकेज वितरित करें, ताकि अधिकतम खोज योग्यता सुनिश्चित हो।

संस्करण

हमारे रिलीज़ चक्र में पारदर्शिता के लिए और पीछे की संगतता बनाए रखने के प्रयास में, बूटस्ट्रैप को सेमांटिक वर्शनिंग दिशानिर्देश के तहत बनाए रखा जाता है। कभी-कभी हम गलती करते हैं, लेकिन जहां संभव हो वहां उन नियमों का पालन करते हैं।

हमारे GitHub प्रोजेक्ट के रिलीज़ अनुभाग में प्रत्येक रिलीज़ संस्करण के लिए चेंजलॉग देखें। आधिकारिक बूटस्ट्रैप ब्लॉग पर रिलीज़ घोषणा पोस्ट में प्रत्येक रिलीज़ में किए गए सबसे उल्लेखनीय परिवर्तनों का सारांश होता है।
