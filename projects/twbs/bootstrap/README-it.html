Avvio rapido

Sono disponibili diverse opzioni di avvio rapido:

  • Scarica l’ultima versione
  • Clona il repository: git clone https://github.com/twbs/bootstrap.git
  • Installa con npm: npm install bootstrap@v5.3.6
  • Installa con yarn: yarn add bootstrap@v5.3.6
  • Installa con Bun: bun add bootstrap@v5.3.6
  • Installa con Composer: composer require twbs/bootstrap:5.3.6
  • Installa con NuGet: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

Consulta la pagina di introduzione per informazioni sui contenuti del framework, template, esempi e altro ancora.

Cosa è incluso

Forniamo CSS e JS compilati (bootstrap.), così come CSS e JS compilati e minificati (bootstrap.min.). Le source map (bootstrap.*.map) sono disponibili per l’uso con alcuni strumenti per sviluppatori dei browser. I file JS bundle (bootstrap.bundle.js e il minificato bootstrap.bundle.min.js) includono Popper.

Documentazione

La documentazione di Bootstrap, inclusa in questo repository nella directory principale, è costruita con Astro ed è ospitata pubblicamente su GitHub Pages all’indirizzo . La documentazione può anche essere eseguita localmente.

La ricerca nella documentazione è alimentata da DocSearch di Algolia.

Eseguire la documentazione in locale

  • Esegui npm install per installare le dipendenze Node.js, incluso Astro (il generatore del sito).
  • Esegui npm run test (o uno script npm specifico) per ricostruire i file CSS e JavaScript distribuiti, così come le risorse della documentazione.
  • Dalla directory principale /bootstrap, esegui npm run docs-serve da linea di comando.
  • Apri http://localhost:9001/ nel tuo browser, et voilà.

Scopri di più su Astro leggendo la sua documentazione.

Contributi

Ti invitiamo a leggere le nostre linee guida per i contributori. Sono incluse indicazioni su come aprire issue, standard di codifica e note sullo sviluppo.

Inoltre, se la tua pull request contiene patch o funzionalità JavaScript, devi includere i relativi test unitari. Tutto l’HTML e il CSS deve essere conforme alla Code Guide, mantenuta da Mark Otto.

Le preferenze dell’editor sono disponibili nel file editor config per un utilizzo semplice nei comuni editor di testo. Scopri di più e scarica i plugin su .

Comunità

Ricevi aggiornamenti sullo sviluppo di Bootstrap e chatta con i manutentori del progetto e con la community.

  • Segui @getbootstrap su X.
  • Leggi e iscriviti a The Official Bootstrap Blog.
  • Fai domande e partecipa alle nostre discussioni su GitHub.
  • Discuti, poni domande e altro sulla community Discord o sul subreddit di Bootstrap.
  • Chatta con altri utenti Bootstrap su IRC. Sul server irc.libera.chat, nel canale #bootstrap.
  • Per aiuto nell’implementazione puoi consultare Stack Overflow (tag bootstrap-5).
  • Gli sviluppatori dovrebbero usare la parola chiave bootstrap` nei pacchetti che modificano o estendono le funzionalità di Bootstrap quando distribuiti tramite npm o simili, per massima visibilità.

Versionamento

Per trasparenza riguardo il ciclo di rilascio e per mantenere la compatibilità all’indietro, Bootstrap è mantenuto secondo le linee guida Semantic Versioning. A volte possiamo commettere errori, ma seguiamo queste regole ogni volta che è possibile.

Consulta la sezione Releases del nostro progetto su GitHub per i changelog di ogni versione di Bootstrap. Gli annunci di rilascio pubblicati su il blog ufficiale di Bootstrap contengono i riassunti delle modifiche più importanti di ogni versione.
