Démarrage rapide

Plusieurs options de démarrage rapide sont disponibles :

  • Téléchargez la dernière version
  • Clonez le dépôt : git clone https://github.com/twbs/bootstrap.git
  • Installez avec npm : npm install bootstrap@v5.3.6
  • Installez avec yarn : yarn add bootstrap@v5.3.6
  • Installez avec Bun : bun add bootstrap@v5.3.6
  • Installez avec Composer : composer require twbs/bootstrap:5.3.6
  • Installez avec NuGet : CSS : Install-Package bootstrap Sass : Install-Package bootstrap.sass

Lisez la page de démarrage pour plus d’informations sur le contenu du framework, les modèles, les exemples, et plus encore.

Ce qui est inclus

Dans le téléchargement, vous trouverez les répertoires et fichiers suivants, regroupant logiquement les ressources communes et fournissant à la fois des variantes compilées et minifiées.

Nous fournissons le CSS et le JS compilés (bootstrap.), ainsi que les fichiers CSS et JS compilés et minifiés (bootstrap.min.). Les source maps (bootstrap.*.map) sont disponibles pour une utilisation avec certains outils de développement de navigateurs. Les fichiers JS groupés (bootstrap.bundle.js et bootstrap.bundle.min.js minifié) incluent Popper.

Documentation

La documentation de Bootstrap, incluse dans ce dépôt à la racine, est construite avec Astro et hébergée publiquement sur GitHub Pages à l’adresse . La documentation peut également être exécutée en local.

La recherche dans la documentation est propulsée par DocSearch d’Algolia.

Exécuter la documentation en local

  • Exécutez npm install pour installer les dépendances Node.js, y compris Astro (le générateur du site).
  • Exécutez npm run test (ou un script npm spécifique) pour reconstruire les fichiers CSS et JavaScript distribués, ainsi que nos ressources de documentation.
  • Depuis le répertoire racine /bootstrap, lancez npm run docs-serve dans la ligne de commande.
  • Ouvrez http://localhost:9001/ dans votre navigateur, et voilà.

En savoir plus sur l’utilisation d’Astro en lisant sa documentation.

Contribuer

Veuillez lire nos directives de contribution. Vous y trouverez des instructions pour ouvrir des issues, des standards de codage et des notes sur le développement.

De plus, si votre pull request contient des correctifs ou des fonctionnalités JavaScript, vous devez inclure des tests unitaires pertinents. Tout le HTML et le CSS doivent être conformes au Code Guide, maintenu par Mark Otto.

Les préférences d’éditeur sont disponibles dans le fichier editor config pour une utilisation facile dans les éditeurs de texte courants. En savoir plus et télécharger les plugins sur .

Communauté

Tenez-vous informé du développement de Bootstrap et discutez avec les mainteneurs et les membres de la communauté.

  • Suivez @getbootstrap sur X.
  • Lisez et abonnez-vous au Blog Officiel Bootstrap.
  • Posez vos questions et explorez nos discussions GitHub.
  • Discutez, posez des questions, et plus encore sur le Discord communautaire ou le subreddit Bootstrap.
  • Discutez avec d’autres utilisateurs de Bootstrap sur IRC. Sur le serveur irc.libera.chat, dans le canal #bootstrap.
  • De l’aide pour l’implémentation peut être trouvée sur Stack Overflow (tag bootstrap-5).
  • Les développeurs devraient utiliser le mot-clé bootstrap` pour les paquets qui modifient ou ajoutent des fonctionnalités à Bootstrap lors de la distribution via npm ou des mécanismes similaires afin d’optimiser la découvrabilité.

Gestion des versions

Pour assurer la transparence de notre cycle de publication et dans le souci de maintenir la rétrocompatibilité, Bootstrap est maintenu selon les principes du versionnage sémantique. Il peut arriver que nous fassions des erreurs, mais nous respectons ces règles autant que possible.

Consultez la section Releases de notre projet GitHub pour les journaux de modifications de chaque version de Bootstrap. Les annonces de nouvelles versions sur le blog officiel Bootstrap contiennent des résumés des changements les plus notables de chaque version.
