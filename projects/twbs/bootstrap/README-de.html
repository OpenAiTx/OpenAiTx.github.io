Schnellstart

Es stehen mehrere Schnellstart-Optionen zur Verfügung:

  • Laden Sie das neueste Release herunter
  • Klonen Sie das Repository: git clone https://github.com/twbs/bootstrap.git
  • Installation mit npm: npm install bootstrap@v5.3.6
  • Installation mit yarn: yarn add bootstrap@v5.3.6
  • Installation mit Bun: bun add bootstrap@v5.3.6
  • Installation mit Composer: composer require twbs/bootstrap:5.3.6
  • Installation mit NuGet: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

Lesen Sie die Getting Started-Seite für Informationen über den Framework-Inhalt, Vorlagen, Beispiele und mehr.

Dokumentation

Die Bootstrap-Dokumentation, die in diesem Repo im Root-Verzeichnis enthalten ist, wird mit Astro erstellt und öffentlich auf GitHub Pages unter gehostet. Die Dokumentation kann auch lokal ausgeführt werden.

Die Dokumentationssuche wird von Algolias DocSearch unterstützt.

Dokumentation lokal ausführen

  • Führen Sie npm install aus, um die Node.js-Abhängigkeiten, einschließlich Astro (dem Site-Builder), zu installieren.
  • Führen Sie npm run test (oder ein spezifisches npm-Script) aus, um die verteilten CSS- und JavaScript-Dateien sowie unsere Dokumentations-Assets neu zu erstellen.
  • Vom Root-Verzeichnis /bootstrap aus führen Sie npm run docs-serve in der Kommandozeile aus.
  • Öffnen Sie http://localhost:9001/ in Ihrem Browser – und voilà.

Erfahren Sie mehr über die Verwendung von Astro, indem Sie die Dokumentation lesen.

Mitwirken

Bitte lesen Sie unsere Mitwirkungsrichtlinien sorgfältig durch. Sie enthalten Anweisungen zum Erstellen von Issues, zu Programmierstandards und Hinweise zur Entwicklung.

Wenn Ihr Pull Request JavaScript-Patches oder -Features enthält, müssen relevante Unit-Tests beigefügt werden. Sämtliches HTML und CSS sollte dem Code Guide entsprechen, der von Mark Otto gepflegt wird.

Editor-Einstellungen sind in der Editor Config verfügbar, um die Nutzung in gängigen Texteditoren zu erleichtern. Weitere Informationen und Plugins finden Sie unter .

Community

Bleiben Sie auf dem Laufenden über die Entwicklung von Bootstrap und chatten Sie mit den Projektbetreuern und anderen Community-Mitgliedern.

  • Folgen Sie @getbootstrap auf X.
  • Lesen und abonnieren Sie den offiziellen Bootstrap Blog.
  • Stellen Sie Fragen und diskutieren Sie in unseren GitHub Discussions.
  • Diskutieren Sie, stellen Sie Fragen und mehr auf dem Community-Discord oder im Bootstrap-Subreddit.
  • Chatten Sie mit anderen Bootstrap-Anwendern im IRC. Auf dem Server irc.libera.chat, im Channel #bootstrap.
  • Hilfe zur Implementierung finden Sie auf Stack Overflow (mit dem Tag bootstrap-5).
  • Entwickler sollten das Stichwort bootstrap` für Pakete verwenden, die Funktionalitäten von Bootstrap erweitern oder ändern, wenn sie diese über npm oder ähnliche Mechanismen bereitstellen, um maximale Auffindbarkeit zu gewährleisten.

Versionierung

Zur Transparenz unseres Release-Zyklus und um die Rückwärtskompatibilität zu wahren, wird Bootstrap gemäß den Semantic Versioning Richtlinien gepflegt. Manchmal machen wir Fehler, aber wir halten uns so gut wie möglich an diese Regeln.

Im Releases-Bereich unseres GitHub-Projekts finden Sie Changelogs zu jeder Bootstrap-Version. Veröffentlichungsankündigungen im offiziellen Bootstrap Blog enthalten Zusammenfassungen der wichtigsten Änderungen jeder Version.
