Inicio rápido

Varias opciones de inicio rápido están disponibles:

  • Descarga la última versión
  • Clona el repositorio: git clone https://github.com/twbs/bootstrap.git
  • Instala con npm: npm install bootstrap@v5.3.6
  • Instala con yarn: yarn add bootstrap@v5.3.6
  • Instala con Bun: bun add bootstrap@v5.3.6
  • Instala con Composer: composer require twbs/bootstrap:5.3.6
  • Instala con NuGet: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

Lee la página de introducción para obtener información sobre el contenido del framework, plantillas, ejemplos y más.

Qué está incluido

Dentro de la descarga encontrarás los siguientes directorios y archivos, agrupando lógicamente los recursos comunes y proporcionando tanto variaciones compiladas como minificadas.

Proveemos CSS y JS compilados (bootstrap.), así como CSS y JS compilados y minificados (bootstrap.min.). Los mapas de origen (bootstrap.*.map) están disponibles para su uso con ciertas herramientas de desarrollo de los navegadores. Los archivos JS empaquetados (bootstrap.bundle.js y el minificado bootstrap.bundle.min.js) incluyen Popper.

Documentación

La documentación de Bootstrap, incluida en este repositorio en el directorio raíz, está construida con Astro y alojada públicamente en GitHub Pages en . La documentación también puede ejecutarse localmente.

La búsqueda en la documentación está potenciada por DocSearch de Algolia.

Ejecutar la documentación localmente

  • Ejecuta npm install para instalar las dependencias de Node.js, incluido Astro (el generador del sitio).
  • Ejecuta npm run test (o un script específico de npm) para reconstruir los archivos distribuidos de CSS y JavaScript, así como los recursos de la documentación.
  • Desde el directorio raíz /bootstrap, ejecuta npm run docs-serve en la línea de comandos.
  • Abre http://localhost:9001/ en tu navegador, y voilà.

Aprende más sobre el uso de Astro leyendo su documentación.

Contribuir

Por favor, lee nuestras directrices de contribución. Se incluyen instrucciones para abrir issues, estándares de codificación y notas sobre desarrollo.

Además, si tu pull request contiene parches o funciones de JavaScript, debes incluir pruebas unitarias relevantes. Todo el HTML y CSS debe cumplir con la Guía de Código, mantenida por Mark Otto.

Las preferencias de editor están disponibles en la configuración del editor para facilitar su uso en editores de texto comunes. Lee más y descarga plugins en .

Comunidad

Recibe actualizaciones sobre el desarrollo de Bootstrap y chatea con los mantenedores del proyecto y miembros de la comunidad.

  • Sigue a @getbootstrap en X.
  • Lee y suscríbete a El Blog Oficial de Bootstrap.
  • Haz preguntas y explora nuestras discusiones en GitHub.
  • Debate, haz preguntas y más en el Discord de la comunidad o en el subreddit de Bootstrap.
  • Chatea con otros Bootstrappers en IRC. En el servidor irc.libera.chat, en el canal #bootstrap.
  • Puedes encontrar ayuda de implementación en Stack Overflow (etiquetado bootstrap-5).
  • Los desarrolladores deben usar la palabra clave bootstrap` en los paquetes que modifiquen o añadan funcionalidad a Bootstrap al distribuirse mediante npm o mecanismos de entrega similares para máxima visibilidad.

Versionado

Por transparencia en nuestro ciclo de lanzamientos y con el objetivo de mantener la compatibilidad hacia atrás, Bootstrap se mantiene bajo las directrices de Versionado Semántico. A veces cometemos errores, pero nos adherimos a esas reglas siempre que es posible.

Consulta la sección de Lanzamientos de nuestro proyecto en GitHub para ver los registros de cambios de cada versión de Bootstrap. Las publicaciones de anuncios de lanzamientos en el blog oficial de Bootstrap contienen resúmenes de los cambios más relevantes realizados en cada versión.
