快速开始

提供多种快速开始方式：

  • 下载最新发布版本
  • 克隆仓库：git clone https://github.com/twbs/bootstrap.git
  • 使用 npm 安装：npm install bootstrap@v5.3.6
  • 使用 yarn 安装：yarn add bootstrap@v5.3.6
  • 使用 Bun 安装：bun add bootstrap@v5.3.6
  • 使用 Composer 安装：composer require twbs/bootstrap:5.3.6
  • 使用 NuGet 安装：CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

阅读 入门页面，了解框架内容、模板、示例等信息。

包含内容

在下载包中，你会看到如下目录和文件，对常用资源进行了逻辑分组，并同时提供了编译和压缩后的版本。

我们提供了已编译的 CSS 和 JS 文件（bootstrap.），以及编译和压缩后的 CSS 和 JS 文件（bootstrap.min.）。Source map 文件（bootstrap.*.map）可与部分浏览器开发者工具配合使用。捆绑的 JS 文件（bootstrap.bundle.js 及其压缩版 bootstrap.bundle.min.js）包含了 Popper。

文档

Bootstrap 的文档包含在本仓库根目录下，使用 Astro 构建，并托管在 GitHub Pages 上：。文档也可以在本地运行。

文档搜索由 Algolia 的 DocSearch 提供支持。

本地运行文档

  • 执行 npm install 安装 Node.js 依赖，包括站点生成器 Astro。
  • 执行 npm run test（或指定 npm 脚本）以重新构建分发的 CSS 和 JavaScript 文件，以及文档资源。
  • 在 /bootstrap 根目录下，命令行运行 npm run docs-serve。
  • 在浏览器中打开 http://localhost:9001/，即可访问。

阅读 Astro 的 文档，了解更多用法。

参与贡献

请阅读我们的 贡献指南。其中包括提交 issue、编码规范和开发说明。

版本管理

为确保发布周期透明并努力保持向后兼容性，Bootstrap 遵循 语义化版本号规范。有时我们也会犯错，但会尽量遵守这些规则。

各版本的变更日志请参见 GitHub 项目的 Releases 区域。每次发布的要点总结会在 官方 Bootstrap 博客 公布。
