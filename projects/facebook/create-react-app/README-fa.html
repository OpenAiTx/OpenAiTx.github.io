Create React App

[!CAUTION]
>
## منسوخ شده
>
Create React App یکی از ابزارهای کلیدی برای راه‌اندازی سریع پروژه‌های React در سال‌های ۲۰۱۷ تا ۲۰۲۱ بود، اما اکنون در وضعیت نگهداری بلندمدت قرار دارد و توصیه می‌کنیم به یکی از فریمورک‌های React که در شروع یک پروژه React جدید مستند شده‌اند مهاجرت کنید.
>
اگر برای یادگیری React در حال دنبال کردن یک آموزش هستید، ادامه دادن آموزش ارزشمند است، اما توصیه نمی‌کنیم برای ساخت برنامه‌های تولیدی جدید از Create React App استفاده کنید.

ایجاد برنامه‌های React بدون نیاز به پیکربندی ساخت.

  • ایجاد یک برنامه – نحوه ایجاد یک برنامه جدید.
  • راهنمای کاربر – نحوه توسعه برنامه‌هایی که با Create React App ساخته شده‌اند.

Create React App بر روی macOS، ویندوز و لینوکس کار می‌کند.
اگر چیزی کار نکرد، لطفاً یک issue ثبت کنید.
اگر سوالی دارید یا نیاز به کمک دارید، لطفاً در گفتگوهای گیت‌هاب مطرح کنید.

مرور سریع

npx create-react-app my-app
cd my-app
npm start

اگر قبلاً create-react-app را به صورت سراسری با دستور npm install -g create-react-app نصب کرده‌اید، توصیه می‌کنیم بسته را با npm uninstall -g create-react-app یا yarn global remove create-react-app حذف کنید تا اطمینان حاصل شود که npx همواره جدیدترین نسخه را استفاده می‌کند.

_(npx همراه با npm 5.2+ و بالاتر ارائه می‌شود، دستورالعمل‌ها برای نسخه‌های قدیمی‌تر npm)_

سپس http://localhost:3000/ را باز کنید تا برنامه خود را مشاهده کنید.
زمانی که آماده انتشار به محیط تولید شدید، یک بسته‌ی کم‌حجم با npm run build بسازید.

بلافاصله شروع کنید

شما نیازی به نصب یا پیکربندی ابزارهایی مانند webpack یا Babel ندارید.
این ابزارها از پیش پیکربندی شده و مخفی هستند تا بتوانید روی کد تمرکز کنید.

یک پروژه ایجاد کنید و آماده‌ی شروع باشید.

ایجاد یک برنامه

شما باید Node نسخه ۱۴.۰.۰ یا بالاتر را روی دستگاه توسعه محلی خود داشته باشید (اما روی سرور الزامی نیست). توصیه می‌کنیم از آخرین نسخه LTS استفاده کنید. می‌توانید از nvm (برای macOS/Linux) یا nvm-windows برای تغییر نسخه Node بین پروژه‌های مختلف استفاده کنید.

برای ایجاد یک برنامه جدید، می‌توانید یکی از روش‌های زیر را انتخاب کنید:

npx

npx create-react-app my-app

_(npx یک ابزار اجرای بسته است که همراه با npm 5.2+ و بالاتر ارائه می‌شود، دستورالعمل‌ها برای نسخه‌های قدیمی‌تر npm)_

npm

npm init react-app my-app

_npm init در npm 6+ قابل استفاده است_

Yarn

yarn create react-app my-app

_yarn create در Yarn 0.25+ در دسترس است_

این دستور یک پوشه با نام my-app در داخل پوشه فعلی ایجاد می‌کند.
در داخل این پوشه، ساختار اولیه پروژه و وابستگی‌های مورد نیاز را نصب می‌کند:

my-app
├── README.md
├── node_modules
├── package.json
├── .gitignore
├── public
│   ├── favicon.ico
│   ├── index.html
│   └── manifest.json
└── src
    ├── App.css
    ├── App.js
    ├── App.test.js
    ├── index.css
    ├── index.js
    ├── logo.svg
    └── serviceWorker.js
    └── setupTests.js

بدون پیکربندی و ساختار پوشه‌های پیچیده، فقط فایل‌هایی که برای ساخت برنامه نیاز دارید.
پس از پایان نصب، می‌توانید پوشه پروژه را باز کنید:

cd my-app

در داخل پروژه ایجاد شده، می‌توانید چند دستور داخلی اجرا کنید:

npm start یا yarn start

برنامه را در حالت توسعه اجرا می‌کند.
http://localhost:3000 را در مرورگر باز کنید تا برنامه را ببینید.

صفحه به طور خودکار با تغییر کد، بارگذاری مجدد می‌شود.
خطاهای ساخت و هشدارهای lint در کنسول نمایش داده می‌شوند.

npm test یا yarn test

واچر تست را به صورت تعاملی اجرا می‌کند.
به طور پیش‌فرض، تست‌های مرتبط با فایل‌هایی که از آخرین commit تغییر یافته‌اند را اجرا می‌کند.

بیشتر درباره تست بخوانید.

npm run build یا yarn build

برنامه را برای محیط تولید به پوشه build می‌سازد.
React را به درستی در حالت production bundle و build را برای بهترین عملکرد بهینه می‌کند.

ساخت نهایی کم‌حجم است و نام فایل‌ها شامل hash می‌شود.

برنامه شما آماده انتشار است.

راهنمای کاربر

دستورالعمل‌های کامل برای استفاده از Create React App و نکات فراوان را در مستندات آن بیابید.

چگونه به نسخه‌های جدید به‌روزرسانی کنیم؟

لطفاً برای این مورد و اطلاعات بیشتر به راهنمای کاربر مراجعه کنید.

فلسفه

  • یک وابستگی: تنها یک وابستگی build وجود دارد. این ابزار از webpack، Babel، ESLint و پروژه‌های دیگر استفاده می‌کند، اما تجربه‌ای یکپارچه و منسجم بر روی آن‌ها ارائه می‌دهد.

  • بدون نیاز به پیکربندی: نیازی به پیکربندی چیزی ندارید. یک پیکربندی مناسب برای هر دو حالت توسعه و تولید برای شما مدیریت می‌شود تا بتوانید روی نوشتن کد تمرکز کنید.

  • بدون قفل شدن: هر زمان می‌توانید به تنظیمات دلخواه خود “eject” کنید. با اجرای یک دستور، تمام پیکربندی و وابستگی‌های ساخت مستقیماً به پروژه شما منتقل می‌شود تا بتوانید از همان‌جایی که بودید ادامه دهید.

چه چیزهایی شامل می‌شود؟

محیط شما هر آنچه برای ساخت یک برنامه React تک‌صفحه‌ای مدرن نیاز دارید را خواهد داشت:

  • پشتیبانی از React، JSX، ES6، TypeScript و Flow.
  • قابلیت‌های اضافی زبانی فراتر از ES6 مانند عملگر spread شیء.
  • CSS با پیشوند خودکار، پس نیازی به -webkit- یا پیشوندهای دیگر ندارید.
  • اجرای سریع تست‌های واحد تعاملی با پشتیبانی داخلی از گزارش‌گیری پوشش تست.
  • سرور توسعه زنده که درباره خطاهای متداول هشدار می‌دهد.
  • اسکریپت build برای بسته‌بندی JS، CSS و تصاویر برای تولید، همراه با hash و sourcemap.
  • service worker آفلاین و manifest برنامه وب که تمام معیارهای برنامه وب پیش‌رونده را دارا هستند. (_توجه: استفاده از service worker از نسخه react-scripts@2.0.0 به بعد اختیاری است_)
  • به‌روزرسانی آسان ابزارهای بالا تنها با یک وابستگی.

برای درک نحوه یکپارچگی این ابزارها این راهنما را ببینید.

نکته این است که این ابزارها به صورت پیش‌فرض برای کار به یک روش خاص پیکربندی شده‌اند. اگر پروژه شما به سفارشی‌سازی بیشتری نیاز دارد، می‌توانید "eject" کنید و آن را سفارشی کنید، اما در این صورت باید پیکربندی را خودتان نگهداری کنید.

جایگزین‌های محبوب

Create React App انتخابی عالی برای موارد زیر است:

  • یادگیری React در یک محیط توسعه راحت و پر امکانات.
  • شروع برنامه‌های جدید تک‌صفحه‌ای React.
  • ایجاد مثال برای کتابخانه‌ها و کامپوننت‌های خود با React.

در این چند حالت معمول شاید بخواهید ابزار دیگری را امتحان کنید:

  • اگر می‌خواهید React را بدون وابستگی‌های متعدد ابزارهای build امتحان کنید، به‌جای آن از یک فایل HTML ساده یا sandbox آنلاین استفاده کنید.

  • اگر نیاز دارید کد React را با یک فریمورک قالب‌ساز سمت سرور مانند Rails، Django یا Symfony ترکیب کنید، یا اگر برنامه تک‌صفحه‌ای نمی‌سازید، از nwb یا Neutrino که انعطاف‌پذیرتر هستند استفاده کنید. برای Rails به طور خاص می‌توانید از Rails Webpacker استفاده کنید. برای Symfony از webpack Encore سیمفونی بهره بگیرید.

  • اگر می‌خواهید یک کامپوننت React منتشر کنید، nwb این کار را هم انجام می‌دهد، همین‌طور preset کامپوننت Neutrino.

  • اگر می‌خواهید render سمت سرور با React و Node.js انجام دهید، به Next.js یا Razzle مراجعه کنید. Create React App مستقل از backend است و فقط بسته‌های HTML/JS/CSS ایستا تولید می‌کند.

  • اگر وب‌سایت شما عمدتاً ایستا است (مثلاً پرتفولیو یا وبلاگ)، از Gatsby یا Next.js استفاده کنید. برخلاف Create React App، Gatsby وب‌سایت را در زمان build به HTML پیش‌پردازش می‌کند. Next.js هم رندر سمت سرور و هم پیش‌رندر را پشتیبانی می‌کند.

  • در نهایت، اگر به سفارشی‌سازی بیشتر نیاز دارید، به Neutrino و preset React آن مراجعه کنید.

تمام ابزارهای بالا با کمترین پیکربندی یا بدون نیاز به پیکربندی کار می‌کنند.

اگر ترجیح می‌دهید build را خودتان پیکربندی کنید، این راهنما را دنبال کنید.
