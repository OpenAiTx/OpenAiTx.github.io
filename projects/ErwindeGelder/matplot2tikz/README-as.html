matplot2tikz

পূৰ্বে যি "tikzplotlib" নামেৰে জনা গৈছিল।

এইটো matplot2tikz, এটা Python উপকৰণ যি matplotlib চিত্ৰসমূহক PGFPlots (PGF/TikZ) চিত্ৰসমূহত ৰূপান্তৰ কৰে যেন

LaTeX অথবা ConTeXt নথিসমূহত প্ৰাকৃতিকভাৱে সংযোজনৰ বাবে।

matplot2tikz ৰ আউটপুট PGFPlotsত থাকে, যি এটা TeX লাইব্রেৰী PGF/TikZৰ ওপৰত অৱস্থিত আৰু অক্ষ, তথ্য আদি হিচাপে গ্ৰাফ বৰ্ণনা কৰে। ফলস্বৰূপ, matplot2tikz ৰ আউটপুট

  • অধিক তথ্য সংৰক্ষণ কৰে,
  • সহজে বুজিব পৰা যায়, আৰু
  • সহজে সম্পাদনা কৰিব পৰা যায়

কাঁচা TikZ আউটপুটতকৈ। উদাহৰণস্বৰূপ, matplotlib চিত্ৰ

import matplotlib.pyplot as plt
import numpy as np
plt.style.use("ggplot")
t = np.arange(0.0, 2.0, 0.1)
s = np.sin(2  np.pi  t)
s2 = np.cos(2  np.pi  t)
plt.plot(t, s, "o-", lw=4.1)
plt.plot(t, s2, "o-", lw=4.1)
plt.xlabel("time (s)")
plt.ylabel("Voltage (mV)")
plt.title("Simple plot $\\frac{\\alpha}{2}$")
plt.grid(True)
import matplot2tikz
matplot2tikz.save("test.tex")

(save()ৰ বদলে get_tikz_code() ব্যৱহাৰ কৰক যদি আপুনি কোডটো string হিচাপে বিচাৰে।)

প্লটটো Tweaking কৰা সহজ আৰু আপোনাৰ TeX workflow-ৰ অংশ হিচাপে কৰিব পাৰি। অসাধাৰণ PGFPlots ম্যানুৱেল-ত আপোনাৰ plot-টো আৰু ভাল দেখুৱাবলৈ উৎকৃষ্ট উদাহৰণ আছে।

নিশ্চয়, matplotlib-এ তৈয়াৰ কৰা সকলো ছবিয়েই নিৰ্বিঘ্নে ৰূপান্তৰ কৰিব নোৱাৰি। বিশেষকৈ, 3D plot-সমূহ কাম নকৰে।

ইনস্টলেশ্যন

matplot2tikz Python Package Index-ত উপলব্ধ, সেয়ে সহজে কৰক

pip install matplot2tikz
ইন্সটল কৰিবলৈ।

ব্যৱহাৰ

  • সাধাৰণতে আপোনাৰ matplotlib plot সৃষ্টি কৰক।

  • pyplot.show()ৰ পৰিৱৰ্তে, matplot2tikz আহ্বান কৰক

``python import matplot2tikz

matplot2tikz.save("mytikz.tex") # or matplot2tikz.save("mytikz.tex", flavor="context") ` TikZ ফাইলখন mytikz.tex হিচাপে সংৰক্ষণ কৰিবলৈ।

  • আপোনাৰ TeX উৎস সংহিতাত mytikz.tex ৰ সামগ্ৰীসমূহ যোগ কৰক। এইটো কৰিবৰ বাবে এটা সুবিধাজনক উপায় হৈছে
তলত উল্লেখ কৰা দৰে

`latex \input{/path/to/mytikz} `

লগতে নিশ্চিত কৰক যে PGFPlots আৰু সঠিক ইউনিক'ড সমৰ্থনৰ পেকেজসমূহ আপোনাৰ নথিপত্ৰৰ হেডাৰত অন্তর্ভুক্ত কৰা হৈছে:

`latex \usepackage[utf8]{inputenc} \usepackage{pgfplots} \DeclareUnicodeCharacter{2212}{−} \usepgfplotslibrary{groupplots,dateplot} \usetikzlibrary{patterns,shapes.arrows} \pgfplotsset{compat=newest} `

or:

`latex \setupcolors[state=start] \usemodule[tikz] \usemodule[pgfplots] \usepgfplotslibrary[groupplots,dateplot] \usetikzlibrary[patterns,shapes.arrows] \pgfplotsset{compat=newest} \unexpanded\def\startgroupplot{\groupplot} \unexpanded\def\stopgroupplot{\endgroupplot} `

আপুনি কোডটো এইদৰে পাব পাৰে:

`python import matplot2tikz

matplot2tikz.Flavors.latex.preamble() # or matplot2tikz.Flavors.context.preamble() `

  • [বৈকল্পিক] clean_figure কমান্ড ব্যৱহাৰ কৰি আউটপোৰ্ট কৰাৰ আগতে চিত্ৰটো পৰিষ্কাৰ কৰক।

`python import matplotlib.pyplot as plt import numpy as np

# ... do your plotting

import matplot2tikz

matplot2tikz.clean_figure() matplot2tikz.save("test.tex") ` এই আদেশটোয়ে সেই বিন্দুবোৰ আঁতৰাই দিব যিবোৰ অক্ষ সীমাৰ বাহিৰত আছে, বক্রৰেখাবোৰ সৰল কৰিব আৰু নিৰ্দিষ্ট লক্ষ্য ৰিজলিউশ্যনৰ বাবে বিন্দুৰ ঘনত্ব হ্ৰাস কৰিব।

matplot2tikz বনাম tikzplotlib

এই matplot2tikz লাইব্রেৰীটো tikzplotlib প্ৰকল্পৰ পৰা উৎপত্তি হৈছে। নতুন লাইব্রেৰী সৃষ্টি কৰাৰ কাৰণ হৈছে tikzplotlib এতিয়া ৰক্ষণাবেক্ষণ হোৱা নাই আৰু tikzplotlib লাইব্রেৰীৰ একমাত্ৰ গৰাকী দ্বাৰা কেৱল ৰক্ষণাবেক্ষণ সম্ভৱ। যদি আপোনাৰ প্ৰয়োজন তৃতীয়-পক্ষৰ কোড ব্যৱহাৰ কৰাৰ যি ইতিমধ্যে tikzplotlibত নিৰ্ভৰ কৰে, তেন্তে পৰামৰ্শ দিয়া হৈছে tikzplotlib নিৰ্ভৰশীলতা matplot2tikzলৈ সলনি কৰক। যদি এইটো সম্ভৱ নহয়, তেন্তে এটা সমাধান হৈছে তৃতীয়-পক্ষৰ কোডটো আমদানি কৰাৰ আগতে তলত দিয়া কোডটো ৰাখক:

` import sys import matplot2tikz sys.modules["tikzplotlib"] = matplot2tikz # Do other imports, e.g., using import my_third_party_library # If tikzplotlib is used in this library, it will automatically use matplot2tikz instead. `

অৱদান

আপুনি যদি বাগ্‌সমূহ অনুভৱ কৰে, অৱদান আগবঢ়াব বিচাৰে, matplot2tikz এ কি কৰিব পাৰে তাৰ ভাল উদাহৰণ থাকে, অথবা অধিক তথ্য বিচাৰি থাকিলে, অনুগ্ৰহ কৰি matplot2tikz ৰ GitHub পৃষ্ঠা চাওক।

অৱদান আগবঢ়াবলৈ, তলৰ পদক্ষেপবোৰ অনুসৰণ কৰক:

  • git repository ডাউনলোড কৰক, যেনে
git clone git@github.com:ErwindeGelder/matplot2tikz.git ব্যৱহাৰ কৰি।
  • এটা virtual environment সৃষ্টি কৰক, যেনে python -m venv venv ব্যৱহাৰ কৰি।
  • virtual environment সক্ৰিয় কৰক (যেনে Windows-ত, venv\Scripts\activate)।
  • uv ইনষ্টল কৰক pip install uv ব্যৱহাৰ কৰি আৰু তাৰপিছত tox-uv ইনষ্টল কৰক uv pip install tox-uv ব্যৱহাৰ কৰি।
  • মূল শাখাখন সংৰক্ষিত, অৰ্থাৎ আপুনি এই শাখাত সোজাকৈ পৰিৱর্তন ঠেলিব নোৱাৰে।
সেয়া বাবে, যদি আপুনি পৰিৱর্তন কৰিব বিচাৰে, তেন্তে পৃথক শাখাত কৰক। উদাহৰণস্বৰূপে, আপুনি নতুন শাখা সৃষ্টি কৰিব পাৰে git checkout -b feature/my_awesome_new_feature ব্যৱহাৰ কৰি।
  • পৰিৱর্তন ঠেলাৰ আগতে, নিশ্চিত কৰক যে কোডটো linting নিয়ম অনুসৰণ কৰে আৰু সকলো টেষ্ট সফল হৈছে।
tox চলাওক। এইটো linting পৰীক্ষা কৰে আৰু সকলো test script চলায়। এইবোৰ হাতেকলীয়া ভাবে কৰিবলৈ, তলৰ কমাণ্ডবোৰ ব্যৱহাৰ কৰক:
  • tox -e lint চলাওক। আপুনি linting কমাণ্ডবোৰ হাতেকলীয়া ভাবে কৰিব পাৰে:
  • (এবাৰ) uv pip install -r requirements-lint.txt
  • ruff format . --check (--check ফ্লেগ আঁতৰালে ruff এ format কৰিব)
  • ruff check .
  • mypy .
  • tox -e py310 চলাওক।
  • tox -e py311 চলাওক।
  • tox -e py312 চলাওক।
  • tox -e py313 চলাওক।
  • tox -e py314 চলাওক।
  • tox -e combine-test-reports চলাওক।
  • coverage report /reports/coverage_html/index.html ত চেক কৰক যাতে টেষ্টে সকলো ধৰা হৈছে।

টোকা: বৰ্তমান, সকলো কোড covered হোৱা নাই। আদৰ্শ ৰূপে, সকলো কোড covered হোৱা উচিত, কিন্তু বৰ্তমানলৈকে নিশ্চিত কৰক যে সকলো নতুন কোড টেষ্টিং দ্বারা covered হৈছে।
  • পৰিৱর্তনবোৰ GitHub-ত ঠেলক। যদি সকলো ঠিক আছে আৰু আপুনি আপোনাৰ পৰিৱর্তন main
শাখালৈ মাৰ্জ কৰিব বিচাৰে, তেন্তে এটা pull request সৃষ্টি কৰক। আদৰ্শ ৰূপে, মাৰ্জৰ আগতে কমেও এজন reviewer এ pull request ৰ পৰ্যালোচনা কৰে।

মন কৰক যে বৰ্তমান কেৱল "Code owners" এহে main` শাখালৈ pull request মাৰ্জ কৰিব পাৰে। এইটো নিশ্চিত কৰিবলৈ যে প্ৰত্যেকে মূল কোড ভাঙি নেপায় (অজানিতে হ’লেও)। যদি আপুনি "Code owner" হ'ব বিচাৰে, আমাক জনাব।
