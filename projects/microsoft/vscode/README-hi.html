Visual Studio Code - ओपन सोर्स ("Code - OSS")

रिपॉजिटरी

यह रिपॉजिटरी ("Code - OSS") वह स्थान है जहाँ हम (माइक्रोसॉफ्ट) समुदाय के साथ मिलकर Visual Studio Code उत्पाद का विकास करते हैं। यहाँ हम न केवल कोड और समस्याओं पर कार्य करते हैं, बल्कि अपनी रोडमैप, मासिक इटरेशन योजनाएँ, और अपने एंडगेम योजनाएँ भी प्रकाशित करते हैं। यह स्रोत कोड सभी के लिए मानक MIT लाइसेंस के अंतर्गत उपलब्ध है।

Visual Studio Code

Visual Studio Code Code - OSS रिपॉजिटरी का एक वितरण है जिसमें माइक्रोसॉफ्ट-विशिष्ट अनुकूलन पारंपरिक माइक्रोसॉफ्ट उत्पाद लाइसेंस के अंतर्गत जारी किए गए हैं।

Visual Studio Code कोड संपादक की सादगी को डेवलपर्स की मुख्य संपादन-निर्माण-डीबग चक्र की आवश्यकताओं के साथ जोड़ता है। यह व्यापक कोड संपादन, नेविगेशन, और समझ समर्थन प्रदान करता है, साथ ही हल्का डिबगिंग, समृद्ध एक्सटेंशन मॉडल, और मौजूदा टूल्स के साथ हल्का एकीकरण भी उपलब्ध कराता है।

Visual Studio Code को हर महीने नई सुविधाओं और बग फिक्स के साथ अपडेट किया जाता है। आप इसे Windows, macOS, और Linux के लिए Visual Studio Code की वेबसाइट से डाउनलोड कर सकते हैं। प्रतिदिन नवीनतम रिलीज़ प्राप्त करने के लिए, Insiders build इंस्टॉल करें।

योगदान

इस परियोजना में भाग लेने के कई तरीके हैं, उदाहरण के लिए:

  • बग्स और फीचर अनुरोध सबमिट करें, और जब वे चेक इन हों तो हमें सत्यापित करने में मदद करें
  • सोर्स कोड परिवर्तन की समीक्षा करें
  • प्रलेखन की समीक्षा करें और टाइपो से लेकर अतिरिक्त या नया सामग्री जोड़ने के लिए पुल अनुरोध बनाएं

यदि आप समस्याओं को ठीक करने और सीधे कोड बेस में योगदान करने में रुचि रखते हैं, कृपया दस्तावेज़ How to Contribute देखें, जिसमें निम्नलिखित शामिल हैं:

  • स्रोत से कैसे बनाएं और चलाएं
  • डेवलपमेंट वर्कफ़्लो, जिसमें डिबगिंग और परीक्षण चलाना शामिल है
  • कोडिंग दिशानिर्देश
  • पुल अनुरोध सबमिट करना
  • काम करने के लिए कोई समस्या ढूंढना
  • अनुवादों में योगदान देना

फीडबैक

  • Stack Overflow पर प्रश्न पूछें
  • नई सुविधा का अनुरोध करें
  • लोकप्रिय फीचर अनुरोधों को वोट दें
  • समस्या दर्ज करें
  • GitHub Discussions या Slack पर एक्सटेंशन लेखक समुदाय से जुड़ें
  • @code को फॉलो करें और हमें अपनी राय बताएं!

इनमें से प्रत्येक चैनल का विवरण और कुछ अन्य उपलब्ध समुदाय-प्रेरित चैनलों की जानकारी के लिए हमारा wiki देखें।

संबंधित परियोजनाएँ

VS Code के कई मुख्य घटक और एक्सटेंशन GitHub पर अपनी-अपनी रिपॉजिटरी में स्थित हैं। उदाहरण के लिए, node debug adapter और mono debug adapter रिपॉजिटरी एक-दूसरे से अलग हैं। पूरी सूची के लिए, कृपया हमारे wiki के Related Projects पृष्ठ पर जाएँ।

बंडल की गई एक्सटेंशन

VS Code में extensions फ़ोल्डर में स्थित कुछ बिल्ट-इन एक्सटेंशन शामिल हैं, जिनमें कई भाषाओं के लिए ग्रामर और स्निपेट्स हैं। जो एक्सटेंशन किसी भाषा के लिए समृद्ध भाषा समर्थन (कोड पूर्णता, Go to Definition) प्रदान करते हैं, उनके नाम के अंत में language-features लगा होता है। उदाहरण के लिए, json एक्सटेंशन JSON के लिए कलरिंग प्रदान करता है और json-language-features एक्सटेंशन JSON के लिए समृद्ध भाषा समर्थन देता है।

डेवलपमेंट कंटेनर

इस रिपॉजिटरी में Visual Studio Code Dev Containers / GitHub Codespaces डेवलपमेंट कंटेनर शामिल है।

  • Dev Containers के लिए, Dev Containers: Clone Repository in Container Volume... कमांड का उपयोग करें, जो macOS और Windows पर बेहतर डिस्क I/O के लिए एक Docker वॉल्यूम बनाता है।
  • यदि आपके पास पहले से VS Code और Docker इंस्टॉल है, तो आप यहाँ क्लिक करें भी कर सकते हैं। इससे आवश्यकता पड़ने पर VS Code स्वतः Dev Containers एक्सटेंशन इंस्टॉल करेगा, स्रोत कोड को कंटेनर वॉल्यूम में क्लोन करेगा, और उपयोग के लिए एक dev कंटेनर तैयार करेगा।

  • Codespaces के लिए, VS Code में GitHub Codespaces एक्सटेंशन इंस्टॉल करें, और Codespaces: Create New Codespace कमांड का उपयोग करें।

Docker / Codespace को पूरा बिल्ड चलाने के लिए कम से कम 4 कोर और 6 GB RAM (8 GB अनुशंसित) होना चाहिए। अधिक जानकारी के लिए development container README देखें।
