MMAR 概览

我们推出了 MMAR，一项旨在评估音频语言模型（ALMs）在大规模多学科任务中深度推理能力的新基准。 MMAR 包含 1,000 个精心策划的音频-问题-答案三元组，采集自真实互联网视频，并通过反复错误修正和质量检查确保高质量。 基准中的每个条目都要求多步骤的深度推理，超越表层理解。此外，部分问题需要研究生级别的感知与领域知识，提升了基准的难度和深度。 示例如下：

MMAR 的元数据可在此文件中获取。不同于以往基准，MMAR 不仅涵盖语音、音频和音乐等传统模态，还扩展至它们的混合，数据采集自野外视频。数据在各模态间的分布见左侧图。每个问题还被标注了指定的类别和子类别，如右侧图所示。

对于每个问题，我们还提供了原始视频的 URL 及对应时间戳，以及片段中所说的语言（若有）。为防止推理模型训练中的潜在数据泄露，我们暂未公开推理线索和思维链注释，相关内容将在适当时机发布。

基准测试结果

我们在 MMAR 上对五类模型进行了基准测试：
  • 大型音频语言模型（LALMs）
  • 大型音频推理模型（LARMs）
  • 全能语言模型（OLMs）
  • 以音频字幕作为输入的大型语言模型（LLMs）
  • 以音频字幕作为输入的大型推理模型（LRMs）

数据集构建

MMAR 基准通过全面的流程构建完成。过程包括：
  • 头脑风暴设计挑战性问题
  • 通过人机协作建立分类体系
  • 基于启发式方法进行数据采集与标注
  • 抓取音频数据并在多个槽位丰富内容
  • 反复修正与质量检查，确保数据高保真

测试您的模型！

为了确保顺利集成到现有评估流程，我们采用了基于 MMAU 修改的评估方法，实现于 evaluation.py。评估脚本的输入应与 MMAR-meta.json 格式相同，并额外包含名为 model_prediction 的键，用于存储模型对每个问题的预测结果。 运行脚本命令如下：

python evaluation.py  --input INPUT_JSON_PATH
