RMCP

첫 번째 릴리스를 기다려주세요.

tokio async 런타임을 사용하는 공식 rust Model Context Protocol SDK 구현입니다.

이 저장소에는 다음 크레이트가 포함되어 있습니다:

  • rmcp: RMCP 프로토콜 구현을 제공하는 핵심 크레이트(자세한 정보를 원하시면 rmcp를 방문해 주세요)
  • rmcp-macros: RMCP 도구 구현을 위한 프로시저 매크로 크레이트(자세한 정보를 원하시면 rmcp-macros를 방문해 주세요)

사용법

크레이트 가져오기

rmcp = { version = "0.1", features = ["server"] }

또는 개발 채널

rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

서드파티 의존성

기본 의존성:
  • tokio 필수
  • serde 필수

클라이언트 구축

클라이언트 시작

``rust, ignore use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}}; use tokio::process::Command;

#[tokio::main] async fn main() -> Result<(), Box> { let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| { cmd.arg("-y").arg("@modelcontextprotocol/server-everything"); }))?).await?; Ok(()) }

서버 구축

트랜스포트 구축 rust, ignore use tokio::io::{stdin, stdout}; let transport = (stdin(), stdout());

서비스 구축

ServerHandler 또는 ClientHandler를 사용하여 쉽게 서비스를 구축할 수 있습니다. rust, ignore let service = common::counter::Counter::new();

서버 시작 rust, ignore // 이 호출은 초기화 과정을 마칩니다 let server = service.serve(transport).await?;

서버와 상호작용

서버가 초기화되면 요청 또는 알림을 보낼 수 있습니다: rust, ignore // 요청 let roots = server.list_roots().await?;

// 또는 알림 전송 server.notify_cancelled(...).await?;

서비스 종료 대기 rust, ignore let quit_reason = server.waiting().await?; // 또는 취소 let quit_reason = server.cancel().await?; ``
