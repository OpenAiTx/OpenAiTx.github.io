RMCP

पहली रिलीज़ का इंतजार करें।

tokio असिंक रनटाइम के साथ एक आधिकारिक rust Model Context Protocol SDK इम्प्लीमेंटेशन।

इस रिपॉजिटरी में निम्नलिखित क्रेट्स शामिल हैं:

  • rmcp: आरएमसीपी प्रोटोकॉल इम्प्लीमेंटेशन प्रदान करने वाला कोर क्रेट (अधिक जानकारी के लिए कृपया देखें rmcp)
  • rmcp-macros: आरएमसीपी टूल इम्प्लीमेंटेशन के लिए एक प्रक्रियात्मक मैक्रो क्रेट (अधिक जानकारी के लिए कृपया देखें rmcp-macros)

उपयोग

क्रेट इम्पोर्ट करें

rmcp = { version = "0.1", features = ["server"] }

या dev चैनल

rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

तीसरे पक्ष की निर्भरताएँ

मूलभूत निर्भरताएँ:
  • tokio आवश्यक
  • serde आवश्यक

क्लाइंट बनाएं

एक क्लाइंट शुरू करें

``rust, ignore use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}}; use tokio::process::Command;

#[tokio::main] async fn main() -> Result<(), Box> { let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| { cmd.arg("-y").arg("@modelcontextprotocol/server-everything"); }))?).await?; Ok(()) }

सर्वर बनाएं

ट्रांसपोर्ट बनाएं rust, ignore use tokio::io::{stdin, stdout}; let transport = (stdin(), stdout());

सर्विस बनाएं

आप आसानी से ServerHandler या ClientHandler का उपयोग करके एक सर्विस बना सकते हैं। rust, ignore let service = common::counter::Counter::new();

सर्वर शुरू करें rust, ignore // यह कॉल इनिशियलाइज़ेशन प्रक्रिया को पूरा कर देगा let server = service.serve(transport).await?;

सर्वर के साथ इंटरैक्ट करें

एक बार सर्वर इनिशियलाइज़ हो जाने के बाद, आप अनुरोध या नोटिफिकेशन भेज सकते हैं: rust, ignore // अनुरोध let roots = server.list_roots().await?;

// या नोटिफिकेशन भेजें server.notify_cancelled(...).await?;

सर्विस शटडाउन का इंतजार करना rust, ignore let quit_reason = server.waiting().await?; // या इसे रद्द करें let quit_reason = server.cancel().await?; ``
