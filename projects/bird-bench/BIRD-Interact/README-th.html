BIRD-INTERACT 1.0

⚠️ ประกาศ

โปรดทราบว่าในขั้นตอนการประเมินผล ก่อนที่ Docker จะโหลดฐานข้อมูล อาจเกิดข้อผิดพลาดเป็นครั้งคราวเนื่องจากความไม่สอดคล้องของสภาพแวดล้อม (ข้อผิดพลาดเหล่านี้จะไม่หยุดกระบวนการแต่จะปรากฏใน log ของ Docker) ส่งผลให้ฐานข้อมูลบางส่วนอาจโหลดไม่สมบูรณ์และกลายเป็นฐานข้อมูลเปล่า ซึ่งจะทำให้ผลการประเมินต่ำกว่าปกติ 👉 ดังนั้น เราขอแนะนำอย่างยิ่งให้ตรวจสอบ log ของ Docker เพื่อหาข้อผิดพลาด ก่อนทำการประเมิน และตรวจสอบให้แน่ใจว่าฐานข้อมูลทั้งหมดถูกโหลดสำเร็จแล้ว

👉 เราได้อัปเดต แนวทางการส่งผลงาน ซึ่งขณะนี้รองรับ scaffolds ของ agent ที่ปรับแต่งเองได้ สามารถดูแนวทางการส่งผลงานโดยละเอียดได้ ที่นี่

🧸 ภาพรวม

BIRD-INTERACT คือชุดวัดผล text-to-SQL แบบอินเทอร์แอคทีฟที่ นิยามใหม่การประเมิน Text-to-SQL ผ่านมุมมองของปฏิสัมพันธ์แบบไดนามิก สภาพแวดล้อมนี้ผสมผสานฐานความรู้แบบลำดับชั้น เอกสารฐานข้อมูล และตัวจำลองผู้ใช้ที่ขับเคลื่อนด้วยฟังก์ชัน เพื่อสร้างสถานการณ์องค์กรที่แท้จริงในทุกการดำเนินการ CRUD ให้โหมดทดสอบที่เข้มข้น 2 แบบ: (1) Conversational Interaction แบบพาสซีฟ และ (2) Agentic Interaction แบบแอคทีฟ ครอบคลุม 600 งานที่มีการอธิบายประกอบ รวมถึง Business Intelligence (BI), การดำเนินการ CRUD ฯลฯ ทุกงานมี test case ที่สามารถรันได้จริง การประเมินโดยทั่วไปจะเกิดปฏิสัมพันธ์ 1,968-5,496 รอบระหว่างโมเดลกับตัวจำลองผู้ใช้ ขณะที่โมเดลอาร์กิวเมนต์ล่าสุดแก้ปัญหาได้เพียง ≈24% และ ≈18% ของงานทั้งหมด แสดงถึงความท้าทายของชุดวัดผลนี้

✅ โหมดการประเมิน 2 แบบ

BIRD-INTERACT รองรับโหมดการประเมิน 2 แบบดังที่กล่าวไว้ข้างต้น:

  • c-Interact: Conversational Interaction เป็นโหมดพาสซีฟและลำดับงานจะคงที่ โค้ดและข้อมูลรายละเอียดดูได้ใน bird_interact_conv
  • a-Interact: Agentic Interaction เป็นโหมดแอคทีฟที่ลำดับงานเปลี่ยนแปลงได้และนำโดยโมเดล โค้ดและข้อมูลรายละเอียดดูได้ใน bird_interact_agent

🐣 เวอร์ชัน Lite

เรากำลังปล่อยเวอร์ชัน lite ของ BIRD-INTERACT, bird-interact-lite-exp ซึ่งมี 270 งานจริงคุณภาพสูงสำหรับ PostgreSQL เหมาะสำหรับการทดลองใช้งานเบื้องต้น

🦜 เวอร์ชันเต็ม

BIRD-INTERACT เวอร์ชันเต็ม, bird-interact-full, เป็นชุดวัดผลที่ครอบคลุม 600 งานสำหรับ PostgreSQL ครอบคลุมการดำเนินการ SQL และคำถามผู้ใช้หลากหลาย เวอร์ชันเต็มกำลังจะมาเร็วๆ นี้

การปรับขนาดเวลาปฏิสัมพันธ์ (ITS)

การปรับขนาดเวลาปฏิสัมพันธ์ (ITS) หมายถึงความสามารถของโมเดลในการเพิ่มประสิทธิภาพสูงสุดอย่างต่อเนื่องผ่านการโต้ตอบแบบหลายรอบ เมื่อผลการโต้ตอบนี้สูงกว่าประสิทธิภาพที่คาดหวังจากการโต้ตอบรอบเดียวบนงานที่ระบุครบถ้วนและไม่คลุมเครือ เราถือว่าโมเดลนั้นเป็นไปตาม กฎ ITS เมื่อความอดทนของผู้ใช้เพิ่มขึ้นและจำนวนรอบปฏิสัมพันธ์สะสม ประสิทธิภาพจะยังคงดีขึ้น แสดงให้เห็นว่าโมเดลสามารถรักษาการสื่อสารที่มีประสิทธิภาพในบทสนทนายาวได้ ปัจจุบันเราพบว่าเฉพาะ claude-3-7-sonnet เท่านั้นที่เป็นไปตามกฎ ITS

การตั้งค่าสภาพแวดล้อม

  • รัน Docker container สำหรับฐานข้อมูล bird-interact-lite, ฐานข้อมูล bird-interact-full และสภาพแวดล้อมการประเมินผล:
> หากคุณต้องการประเมินผลเฉพาะ bird-interact-lite คุณสามารถคอมเมนต์บริการ postgresql_full ใน docker-compose.yml เพื่อเร่งการตั้งค่าสภาพแวดล้อม เริ่มต้นสภาพแวดล้อมโดยรัน: 
   cd env
   docker compose pull 
   docker compose up -d
   ``
   รอเป็นเวลาหลายนาทีเพื่อการเริ่มต้นฐานข้อมูล
  คุณสามารถติดตามความคืบหน้าการสร้างได้โดย:
  `bash
  docker compose logs -f --tail=100 bird_interact_postgresql_full # or bird_interact_postgresql for bird-interact-lite
  `
  หากเสร็จสิ้นแล้ว คุณควรเห็นบันทึกโดยไม่มีข้อผิดพลาด เช่น:
  `bash
  bird_interact_postgresql_full  | 2025-10-28 17:58:30.413 HKT [1] LOG:  database system is ready to accept connection
  `
  
  หากคุณเคยสร้างคอนเทนเนอร์มาก่อนและต้องการสร้างใหม่อีกครั้ง คุณสามารถรันคำสั่งต่อไปนี้:
  `bash
  docker compose down -v # this cmd removes the containers and the volumes
  docker compose pull   # pull the latest images from Docker Hub
  docker compose up -d --force-recreate # build and start the containers again. --force-recreate means force the recreation of the containers. 
  # Or docker compose up -d --force-recreate bird_interact_eval to only recreate the bird_interact_eval container about evalution code environment.
  `
   
   คำสั่งนี้จะรัน 3 คอนเทนเนอร์โดยใช้อิมเมจที่สร้างไว้ล่วงหน้าจาก Docker Hub:
   
  • bird_interact_postgresql: ฐานข้อมูล PostgreSQL สำหรับ bird-interact-lite
    • 
   
  • bird_interact_postgresql_full: ฐานข้อมูล PostgreSQL สำหรับ bird-interact-full
    • 
   
  • bird_interact_eval: สภาพแวดล้อมสำหรับประเมินผลทั้ง a-Interact และ c-Interact
    •    
ตอนนี้ คุณสามารถเริ่มสภาพแวดล้อมประเมินผลได้โดยรันคำสั่งต่อไปนี้:
   `bash
   docker compose exec bird_interact_eval bash
   `

    •

    •

    •

กรุณาอีเมลไปที่ bird.bench25@gmail.com พร้อมแท็กหัวข้อว่า [bird-interact-lite GT&Test Cases] หรือ [bird-interact-full GT&Test Cases] สำหรับรับ ground truth และกรณีทดสอบของชุดข้อมูล bird-interact-lite หรือ bird-interact-full ซึ่งจะถูกส่งให้อัตโนมัติ
การรวมข้อมูลสาธารณะกับ ground truth และกรณีทดสอบ
จากนั้นใช้สคริปต์ต่อไปนี้เพื่อรวมข้อมูลสาธารณะกับ ground truth และกรณีทดสอบ:
ขอยกตัวอย่างเวอร์ชันเต็ม:
(1) รัน:
bash python combine_public_with_gt.py /path/to/bird-interact-full/bird_interact_data.jsonl /path/to/bird_interact_full_gt_kg_testcases_08022.jsonl /path/to/bird_interact_data.jsonl # bird_interact_full_gt_kg_testcases_08022.jsonl is the data of ground-truth fields, which is obtained by emailing us. 
สิ่งนี้จะสร้างไฟล์ใหม่ที่ /path/to/bird_interact_data.jsonl ซึ่งมีข้อมูลที่รวมไว้แล้ว
(2) จากนั้นแทนที่ข้อมูลสาธารณะเดิมด้วยข้อมูลที่รวมแล้ว:
bash cp /path/to/bird_interact_data.jsonl /path/to/bird-interact-full/bird_interact_data.jsonl
