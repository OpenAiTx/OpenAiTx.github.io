🧸 ภาพรวม

BIRD-INTERACT เป็น benchmark แปลงข้อความเป็น SQL แบบอินเทอร์แอคทีฟ ที่ ออกแบบใหม่การประเมิน Text-to-SQL ผ่านมุมมองของการโต้ตอบแบบไดนามิก สภาพแวดล้อมผสมผสานฐานความรู้แบบลำดับชั้น เอกสารฐานข้อมูล และผู้ใช้จำลองที่ขับเคลื่อนด้วยฟังก์ชัน เพื่อสร้างสถานการณ์องค์กรที่แท้จริง ครอบคลุมการดำเนินการ CRUD ทั้งหมด มีโหมดทดสอบที่เข้มงวดสองรูปแบบ: (1) โต้ตอบแบบ สนทนา (Passive) และ (2) โต้ตอบแบบ Agentic (Active) ครอบคลุม 600 งานที่มีการใส่คำอธิบาย ประกอบด้วย BI, CRUD และอื่นๆ แต่ละงานมี test cases ที่สามารถรันได้ การประเมินโดยทั่วไปจะมี 1,968-5,496 รอบการโต้ตอบระหว่างโมเดลกับผู้ใช้จำลอง ขณะที่โมเดล reasoning ที่ทันสมัยสามารถแก้ไขได้เพียง ≈24% และ ≈18% ของงานทั้งหมด สะท้อนถึงความท้าทายของ benchmark นี้

✅ โหมดการประเมินสองแบบ

BIRD-INTERACT รองรับโหมดการประเมินสองแบบดังที่กล่าวไว้ข้างต้น:

  • c-Interact: ปฏิสัมพันธ์เชิงสนทนา ซึ่งเป็นโหมดแบบพาสซีฟและมีขั้นตอนการทำงานที่ตายตัว สามารถดูโค้ดและข้อมูลรายละเอียดได้ใน bird_interact_conv
  • a-Interact: ปฏิสัมพันธ์เชิงเอเจนต์ ซึ่งเป็นโหมดแบบแอคทีฟที่ฝังตัวโดยมีขั้นตอนการทำงานที่เปลี่ยนแปลงและนำโดยโมเดล สามารถดูโค้ดและข้อมูลรายละเอียดได้ใน bird_interact_agent

🐣 เวอร์ชัน Lite

เราได้เปิดตัวเวอร์ชัน Lite ของ BIRD-INTERACT, bird-interact-lite-exp ซึ่งประกอบด้วยงานจริงคุณภาพสูงจำนวน 270 งานสำหรับ PostgreSQL โดยเฉพาะ เหมาะสำหรับการทดลองใช้อย่างรวดเร็ว

🦜 เวอร์ชันเต็ม

เวอร์ชันเต็มของ BIRD-INTERACT, bird-interact-full เป็นชุดเบนช์มาร์กแบบครบถ้วนซึ่งประกอบด้วยงาน 600 งานสำหรับ PostgreSQL ครอบคลุมการดำเนินการ SQL และคำถามของผู้ใช้หลากหลายรูปแบบ โดยเวอร์ชันเต็มจะเปิดตัวเร็วๆ นี้

การปรับขนาดเวลาโต้ตอบ (ITS)

การปรับขนาดเวลาโต้ตอบ (ITS) หมายถึงความสามารถของโมเดลในการเพิ่มประสิทธิภาพปลายทางอย่างต่อเนื่องผ่านการโต้ตอบหลายรอบ เมื่อประสิทธิภาพในการโต้ตอบนี้สูงกว่าประสิทธิภาพในอุดมคติของโมเดลในงานที่ระบุไว้อย่างสมบูรณ์และไม่มีความคลุมเครือ เราถือว่าโมเดลนั้นเป็นไปตาม กฎ ITS เมื่อความอดทนของผู้ใช้เพิ่มขึ้นและจำนวนรอบการโต้ตอบมากขึ้น ประสิทธิภาพจะยังคงดีขึ้น แสดงว่าโมเดลสามารถสื่อสารได้อย่างมีประสิทธิภาพต่อเนื่องในการสนทนายาว ขณะนี้เราพบว่าเฉพาะ claude-3-7-sonnet เท่านั้นที่เป็นไปตามกฎ ITS

📦 รายละเอียดชุดข้อมูล

คำอธิบายชุดข้อมูล

  • ฐานข้อมูล: สามารถดาวน์โหลดฐานข้อมูล PostgreSQL ฉบับสมบูรณ์ได้จาก Google Drive ดูรายละเอียดเพิ่มเติมในส่วน Quick Eval
  • data: แต่ละอินสแตนซ์ข้อมูลประกอบด้วยส่วนสำคัญดังนี้:
  • selected_database: ชื่อฐานข้อมูล
  • query: คำถามของผู้ใช้ที่ไม่มีความคลุมเครือ
  • amb_user_query: คำถามของผู้ใช้ที่ถูกเพิ่มความคลุมเครือ
  • user_query_ambiguity: ความคลุมเครือที่เพิ่มเข้าไปในคำถามของผู้ใช้
  • non_critical_ambiguity: ความคลุมเครือที่ไม่สำคัญ เช่น การจัดลำดับ, การจำกัดจำนวน ฯลฯ
  • knowledge_ambiguity: ความคลุมเครือที่เกิดจากการปิดบังความรู้ภายนอก
  • sol_sql: คำตอบ SQL ที่ถูกต้อง
  • preprocess_sql: คำสั่ง SQL ที่ต้องรันก่อนดำเนินการคำตอบหรือการทำนาย
  • clean_up_sql: คำสั่ง SQL ที่ต้องรันหลังจากกรณีทดสอบ เพื่อย้อนการเปลี่ยนแปลงฐานข้อมูล
  • test_cases: ชุดกรณีทดสอบเพื่อใช้ตรวจสอบ SQL ที่ทำนายแก้ไขแล้ว
  • follow_up: คำถามติดตามผลที่มีการระบุป้ายกำกับ
  • external_knowledge: ความรู้ภายนอกที่เกี่ยวข้องกับงานเฉพาะ

  • evaluation: โค้ดสำหรับการประเมินผลมีอยู่ในไดเรกทอรี ./evaluation
