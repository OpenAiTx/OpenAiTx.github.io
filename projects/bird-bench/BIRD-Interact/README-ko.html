🧸 개요

BIRD-INTERACT는 인터랙티브 텍스트-투-SQL 벤치마크로, 동적 상호작용 관점에서 Text-to-SQL 평가를 재구성합니다. 계층적 지식 베이스, 데이터베이스 문서화, 함수 기반 사용자 시뮬레이터가 결합된 환경에서 실제 기업 환경의 전체 CRUD 작업을 재현합니다. 두 가지 엄격한 테스트 모드: (1) 수동적 대화형 상호작용과 (2) 능동적 에이전트 상호작용을 제공하며, 600개의 주석이 달린 작업(BI, CRUD 작업 등)마다 실행 가능한 테스트 케이스로 보호됩니다. 일반적인 평가는 모델과 사용자 시뮬레이터 간에 1,968~5,496번의 상호작용 턴을 유발하며, 최첨단 추론 모델도 현재 약 24% 및 약 18% 작업만 해결하고 있어 벤치마크의 난이도를 보여줍니다.

✅ 두 가지 평가 모드

BIRD-INTERACT는 위에서 언급한 두 가지 평가 모드를 지원합니다:

  • c-Interact: 대화형 상호작용으로, 수동 모드이며 워크플로우가 고정되어 있습니다. 코드와 자세한 정보는 bird_interact_conv에서 확인할 수 있습니다.
  • a-Interact: 에이전트형 상호작용으로, 능동적이고 모델이 주도하는 동적 워크플로우입니다. 코드와 자세한 정보는 bird_interact_agent에서 확인할 수 있습니다.

🐣 라이트 버전

BIRD-INTERACT의 라이트 버전인 bird-interact-lite-exp를 출시합니다. 이 버전에는 PostgreSQL을 위한 270개의 고품질 실제 작업이 포함되어 있습니다. 빠른 실험을 시작하기에 적합합니다.

🦜 풀 버전

BIRD-INTERACT의 풀 버전인 bird-interact-full은 PostgreSQL을 위한 600개의 작업이 포함된 종합 벤치마크입니다. 다양한 SQL 연산과 사용자 쿼리를 포괄합니다. 풀 버전은 곧 출시될 예정입니다.

상호작용 시간 스케일링 (ITS)

상호작용 시간 스케일링(ITS)은 모델이 다중 턴 상호작용을 통해 최종 성능을 지속적으로 향상시키는 능력을 의미합니다. 이 상호작용 성능이 모델의 이상적인 단일 턴 성능(완전히 명확한 과제에서)을 능가할 때, 우리는 이를 ITS 법칙을 만족한다고 합니다. 사용자의 인내심이 높아지고 상호작용 턴이 쌓임에 따라 성능이 계속 향상되어, 모델이 장기간 대화에서도 효과적으로 소통할 수 있음을 보여줍니다. 현재로서는 claude-3-7-sonnet만이 ITS 법칙을 만족하는 것으로 나타났습니다.

📦 데이터셋 상세

데이터셋 설명

  • 데이터베이스: 전체 PostgreSQL 데이터베이스는 Google 드라이브에서 다운로드할 수 있습니다. 자세한 내용은 Quick Eval 섹션을 확인하세요.
  • data: 각 데이터 인스턴스는 다음과 같은 주요 파트로 구성됩니다:
  • selected_database: 데이터베이스 이름
  • query: 명확한 사용자 질의
  • amb_user_query: 모호성이 주입된 사용자 질의
  • user_query_ambiguity: 사용자 질의에 주입된 모호성
  • non_critical_ambiguity: 순서, 제한 등과 같은 비핵심 모호성
  • knowledge_ambiguity: 외부 지식이 마스킹되어 생성된 모호성
  • sol_sql: 정답 SQL 솔루션
  • preprocess_sql: 솔루션 또는 예측을 실행하기 전에 수행할 SQL 쿼리
  • clean_up_sql: 테스트 케이스 후 데이터베이스 변경 사항을 되돌리기 위한 SQL 쿼리
  • test_cases: 예측된 수정 SQL을 검증하기 위한 테스트 케이스 세트
  • follow_up: 라벨링된 후속 질문
  • external_knowledge: 특정 작업과 관련된 외부 지식

  • evaluation: 평가 코드는 ./evaluation 디렉토리에 있습니다.
