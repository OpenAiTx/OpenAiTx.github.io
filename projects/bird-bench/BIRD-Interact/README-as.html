⚠️ ঘোষণা

অনুগ্ৰহ কৰি লক্ষ্য কৰক, আপোনাৰ মূল্যাংকন প্ৰক্ৰিয়া আৰম্ভ কৰাৰ আগতে, যেতিয়া Docker-এ ডেটাবেসসমূহ লোড কৰে, পৰিৱেশৰ অসঙ্গতিৰ বাবে কেতবোৰ সময়ত ত্ৰুটি দেখা দিব পাৰে (এইবোৰে প্ৰক্ৰিয়া বন্ধ নকৰে, কিন্তু Docker লগত উপস্থিত হয়)। ফলস্বৰূপ, কিছুমান ডেটাবেস ঠিককৈ লোড নহব পাৰে, যাৰ ফলত ডেটাবেসসমূহ খালী হৈ থাকে। ইয়াৰ ফলত মূল্যাংকনৰ ফলাফল সাধাৰণকৈ যথেষ্ট কম হব। 👉 সেয়ে, আমাৰ দৃঢ়ভাৱে পৰামৰ্শ, মূল্যাংকন চলোৱাৰ আগতে Docker লগত কোনো ত্ৰুটি আছে নে চাব আৰু সকলো ডেটাবেস সফলভাৱে লোড হৈছে নে নিশ্চিত কৰিব।

🧸 সংক্ষিপ্ত বিবৰণ

BIRD-INTERACT, এটা ইণ্টাৰেক্টিভ টেক্সট-টু-SQL বেঞ্চমাৰ্ক, ডাইনামিক ইণ্টাৰেকশ্যনৰ দৃষ্টিভংগীত টেক্সট-টু-SQL মূল্যায়ন পুনৰ কল্পনা কৰে। এই পৰিৱেশে এটা স্তৰীয় জ্ঞানভাণ্ডাৰ, ডেটাবেইস ডকুমেন্টেশ্যন আৰু ফাংক্সন-চলিত ইউজাৰ সিমুলেটৰ সংমিশ্ৰণ কৰি সম্পূর্ণ CRUD কাৰ্যত সত্যিকাৰ এণ্টাৰপ্ৰাইজ পৰিৱেশ পুনৰুদ্ধাৰ কৰে। ই দুটা কঠোৰ টেষ্ট মোড অফাৰ কৰে: (১) নীৰৱ কনভার্সেশ্যনেল ইণ্টাৰেকশন আৰু (২) সক্ৰিয় এজেণ্টিক ইণ্টাৰেকশন, ৬০০ টা এনোটেটেড টাস্ক বিস্তৃত, য'ত Business Intelligence (BI), CRUD কাৰ্য আৰু আদি সন্নিৱিষ্ট, প্ৰতিটা টেষ্ট কেছ দ্বাৰা সুৰক্ষিত। সাধাৰণ মূল্যায়নে মডেল আৰু ইউজাৰ সিমুলেটৰৰ মাজত ১,৯৬৮-৫,৪৯৬ টা ইণ্টাৰেকশন টাৰ্ন সৃষ্টি কৰে, যেতিয়া শ্ৰেষ্ঠ যুক্তি মডেলবোৰে কেৱল ≈২৪% আৰু ≈১৮% টাস্ক সমাধান কৰে, যি বেঞ্চমাৰ্কৰ চেলেঞ্জ দেখুৱায়।

✅ দুটা মূল্যায়ন মোড

BIRD-INTERACT ওপৰত উল্লেখ কৰা দুটা মূল্যায়ন মোড সমৰ্থন কৰে:

  • c-Interact: কনভার্সেশ্যনেল ইণ্টাৰেকশন যি এটা নীৰৱ মোড আৰু কার্যপদ্ধতি স্থিৰ। ক'ড আৰু বিস্তৃত তথ্য bird_interact_conv ত পোৱা যাব।
  • a-Interact: এজেণ্টিক ইণ্টাৰেকশন যি এটা দেহী সক্ৰিয় মোড, য'ত কার্যপদ্ধতি ডাইনামিক আৰু মডেলৰ দ্বাৰা নিয়ন্ত্ৰিত। ক'ড আৰু বিস্তৃত তথ্য bird_interact_agent ত পোৱা যাব।

Interaction-Time Scaling (ITS)

Interaction-Time Scaling (ITS) মানে হৈছে এটা মডেলৰ বহু-পৰ্যায় আন্তঃক্ৰিয়া যোগে তাৰ অন্তিম প্ৰদর্শন ধাৰাবাহিকভাৱে বৃদ্ধি কৰাৰ সক্ষমতা। এই আন্তঃক্ৰিয়া প্ৰদর্শনে যদি মডেলৰ আদর্শ এক-পৰ্যায় প্ৰদর্শনক সম্পূর্ণ নিৰ্দিষ্ট, অস্পষ্টতাহীন টাস্কত অতিক্ৰম কৰে, তেন্তে আমি ক'ব পাৰো যে মডেল ITS আইন মানে। ইউজাৰৰ ধৈৰ্য বৃদ্ধি হ'লে আৰু আন্তঃক্ৰিয়া পৰ্যায় সংগ্ৰহ হ'লে, প্ৰদর্শন উন্নত হয়, যি মডেলটো দীঘল সময়ৰ কথোপকথনত কার্যকৰী যোগাযোগ বজাই ৰাখিব পাৰে বুলি দেখুৱায়। বৰ্তমান, আমি কেৱল claude-3-7-sonnet মডেলক ITS আইন মানে বুলি পাইছো।

পৰিৱেশ স্থাপন

  • bird-interact-lite ডাটাবেইজ, bird-interact-full ডাটাবেইজ, আৰু মূল্যায়ন পৰিৱেশৰ বাবে Docker কন্টেইনাৰ চলাওক:
> যদি আপুনি কেৱল bird-interact-lite ত মূল্যায়ন কৰিব বিচাৰে, তেন্তে postgresql_full service টো docker-compose.yml ত মন্তব্য কৰক যাতে পৰিৱেশ স্থাপন দ্রুত হয়। পৰিৱেশ আৰম্ভ কৰিবলৈ এইটো চলাওক: 
   cd env
   docker compose pull 
   docker compose up -d
   ``
   ডেটাবেছ আৰম্ভণিৰ বাবে কেইমিনিটমান অপেক্ষা কৰক।
  আপুনি নিৰ্মাণৰ প্ৰগতি তলত দৰ্শন কৰিব পাৰে:
  `bash
  docker compose logs -f --tail=100 bird_interact_postgresql_full # or bird_interact_postgresql for bird-interact-lite
  `
  যদি সম্পূৰ্ণ হোৱা থাকে, আপুনি ত্ৰুটিবিহীনভাৱে লগসমূহ চাব লাগিব যেনে:
  `bash
  bird_interact_postgresql_full  | 2025-10-28 17:58:30.413 HKT [1] LOG:  database system is ready to accept connection
  `
  
  যদি আপুনি পূৰ্বে containers সৃষ্টি কৰিছে আৰু পুনৰ সৃষ্টি কৰিব বিচাৰে, তেন্তে তলত দিয়া নিৰ্দেশনা চলাব পাৰে:
  `bash
  docker compose down -v # this cmd removes the containers and the volumes
  docker compose pull   # pull the latest images from Docker Hub
  docker compose up -d --force-recreate # build and start the containers again. --force-recreate means force the recreation of the containers. 
  # Or docker compose up -d --force-recreate bird_interact_eval to only recreate the bird_interact_eval container about evalution code environment.
  `

bash python combine_public_with_gt.py /path/to/bird-interact-full/bird_interact_data.jsonl /path/to/bird_interact_full_gt_kg_testcases_08022.jsonl /path/to/bird_interact_data.jsonl # bird_interact_full_gt_kg_testcases_08022.jsonl is the data of ground-truth fields, which is obtained by emailing us. 
এইটো /path/to/bird_interact_data.jsonl-ত সংযুক্ত তথ্যৰ সৈতে এটা নতুন ফাইল সৃষ্টি কৰিব। 
(2) তাৰপিছত মূল পাব্লিক তথ্যটো সংযুক্ত তথ্যৰ সৈতে প্ৰতিস্থাপন কৰক:
