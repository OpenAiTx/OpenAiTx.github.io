Hanyang Wang*, Fangfu Liu*, Jiawei Chi, Yueqi Duan

VideoScene es un modelo de difusión de video en un solo paso que cierra la brecha del video al 3D.

https://github.com/user-attachments/assets/dca733b1-b78f-49ac-ae47-5d1b1e8a689b

Basado en ReconX, VideoScene ha logrado un avance en versión turbo.

Instalación

Para comenzar, clona este proyecto, crea un entorno virtual conda usando Python 3.10+ e instala los requisitos:

  • Clona VideoScene.
git clone https://github.com/hanyang-21/VideoScene
cd VideoScene
  • Crea el entorno, aquí mostramos un ejemplo usando conda.

conda create -y -n videoscene python=3.10
conda activate videoscene
pip install torch==2.1.2 torchvision==0.16.2 torchaudio==2.1.2 --index-url https://download.pytorch.org/whl/cu118
pip install -r requirements.txt

  • Opcional, compilar los kernels de cuda para RoPE (como en CroCo v2).
# NoPoSplat relies on RoPE positional embeddings for which you can compile some cuda kernels for faster runtime.
cd src/model/encoder/backbone/croco/curope/
python setup.py build_ext --inplace
cd ../../../../../..

Adquisición de Conjuntos de Datos

RealEstate10K y ACID

Nuestro VideoScene usa los mismos conjuntos de datos de entrenamiento que pixelSplat. A continuación citamos las instrucciones detalladas de pixelSplat para obtener conjuntos de datos.

pixelSplat fue entrenado usando versiones de los conjuntos de datos RealEstate10k y ACID que fueron divididos en fragmentos de ~100 MB para su uso en sistemas de archivos de clústeres de servidores. Pequeños subconjuntos de los conjuntos de datos Real Estate 10k y ACID en este formato pueden encontrarse aquí. Para usarlos, simplemente descomprímelos en una carpeta datasets recién creada en el directorio raíz del proyecto.

Si quieres convertir versiones descargadas de los conjuntos de datos Real Estate 10k y ACID a nuestro formato, puedes usar los scripts aquí. Contáctanos (pixelSplat) si deseas las versiones completas de nuestros conjuntos de datos procesados, que son aproximadamente 500 GB y 160 GB para Real Estate 10k y ACID respectivamente.

Descarga de puntos de control

  • descarga nuestros pesos preentrenados (VideoScene/checkpoints/model.safetensors y VideoScene/checkpoints/prompt_embeds.pt), y guárdalos en checkpoints.

  • para entradas de imagen personalizadas, obtén los modelos preentrenados NoPoSplat aquí, y guárdalos en checkpoints/noposplat.

  • para los conjuntos de datos RealEstate10K, obtén los modelos preentrenados MVSplat aquí, y guárdalos en checkpoints/mvsplat.

Ejecución del Código

Demo Gradio

En esta demo, puedes ejecutar VideoScene en tu máquina para generar un video con vistas de entrada no posadas.

  • selecciona pares de imágenes que representen la misma escena y pulsa "RUN" para obtener un video de la escena.

python -m noposplat.src.app \
    checkpointing.load=checkpoints/noposplat/mixRe10kDl3dv_512x512.ckpt \
    test.video=checkpoints/model.safetensors
also "bash demo.sh"
  • el video generado se almacenará en outputs/gradio

Inferencia

Para generar videos en los conjuntos de datos RealEstate10K, usamos un modelo preentrenado MVSplat,

  • ejecute lo siguiente:

# re10k
python -m mvsplat.src.main +experiment=re10k \
checkpointing.load=checkpoints/mvsplat/re10k.ckpt \
mode=test \
dataset/view_sampler=evaluation \
dataset.view_sampler.index_path=mvsplat/assets/evaluation_index_re10k_video.json \
test.save_video=true \
test.save_image=false \
test.compute_scores=false \
test.video=checkpoints/model.safetensors
also "bash inference.sh"

  • el video generado se almacenará en outputs/test
