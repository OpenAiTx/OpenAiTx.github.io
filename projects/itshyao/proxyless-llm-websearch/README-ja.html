🧠 プロキシ不要のLLMウェブ検索エンジン

プロキシ不要のマルチ検索エンジン型LLMウェブ検索ツールです。URLコンテンツ解析とウェブクローリングをサポートし、LangGraphとLangGraph-MCPを組み合わせたモジュール型エージェントチェーンを実現します。大規模言語モデルによる外部知識参照のために設計され、Playwright + Crawl4AIによるウェブ取得と解析、非同期並列処理、コンテンツ分割・再構成・フィルタリングをサポートします。

🚀 更新履歴

  • 🔥 2025-09-05：langgraph-mcp対応
  • 🔥 2025-09-03：Docker展開追加、組み込み型リランカー、自作テキスト分割器・リランカー対応

✨ 特徴一覧

  • 🌐 プロキシ不要：Playwright で国内ブラウザを設定することで、プロキシなしでもウェブ検索が可能です。
  • 🔍 複数検索エンジン対応：Bing、夸克、百度、搜狗など主要な検索エンジンに対応し、情報ソースの多様性を強化します。
  • 🤖 インテント認識：システムがユーザーの入力内容に基づき、ウェブ検索かURL解析かを自動判別します。
  • 🔄 クエリ分解：ユーザーの検索意図に応じてクエリを自動的に複数のサブタスクに分解し、順次実行することで検索の関連性と効率を向上させます。
  • ⚙️ エージェントアーキテクチャ：LangGraph ベースでラップされた「web_search」と「link_parser」。
  • 🏃‍♂️ 非同期並列タスク処理：非同期並列タスク処理に対応し、複数の検索タスクを効率的に処理できます。
  • 📝 コンテンツ処理の最適化：

  • ✂️ コンテンツ分割：ウェブページの長文を段落ごとに分割します。

  • 🔄 コンテンツ再配置：インテリジェントな並べ替えで情報の関連性を向上。

  • 🚫 コンテンツフィルタリング：無関係または重複する内容を自動的に除外します。
  • 🌐 マルチプラットフォーム対応：
  • 🐳 Docker デプロイ対応：ワンクリックで起動、バックエンドサービスを迅速に構築。

  • 🖥️ FastAPI バックエンドAPIを提供し、任意のシステムに統合可能。

  • 🌍 Gradio Web UI を提供し、すぐに可視化アプリとしてデプロイ可能。

  • 🧩 ブラウザ拡張機能対応：Edge対応、インテリジェントなURL解析拡張機能を提供し、ブラウザ上で直接ウェブ解析・コンテンツ抽出リクエストが可能。

import requests
url = "http://localhost:8800/search"
data = {
  "question": "广州今日天气",
  "engine": "bing",
  "split": {
    "chunk_size": 512,
    "chunk_overlap": 128
  },
  "rerank": {
    "top_k": 5
  }
}
try:
    response = requests.post(
        url,
        json=data
    )
    if response.status_code == 200:
        print("✅ 请求成功！")
        print("响应内容：", response.json())
    else:
        print(f"❌ 请求失败，状态码：{response.status_code}")
        print("错误信息：", response.text)
except requests.exceptions.RequestException as e:
    print(f"⚠️ 请求异常：{str(e)}")

🙏 謝辞

本プロジェクトの一部機能は、以下のオープンソースプロジェクトの支援とインスピレーションによるものです。ここに感謝の意を表します。

  • 🧠 LangGraph：モジュール式エージェントリンク構築フレームワーク。複雑なエージェントシステムの迅速な構築を支援。
  • 🕷 Crawl4AI：強力なウェブコンテンツ解析ツール。効率的なウェブクローリングとデータ抽出を実現。
  • 🌐 Playwright：最新のブラウザ自動化ツール。クロスブラウザ対応のウェブクロール・テスト自動化に対応。
  • 🔌 Langchain MCP Adapters：マルチチェーン処理MCPの構築用。
