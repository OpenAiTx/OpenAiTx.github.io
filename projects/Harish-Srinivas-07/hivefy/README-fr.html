Hivefy 🎧

Hivefy est une application musicale FOSS, inspirée de Spotify, sans publicité et prête à l’utilisation hors ligne développée avec Flutter utilisant l’API non officielle JioSaavn. Explorez les classements tendances, albums, playlists et contenus multilingues — tous open-source, respectueux de la vie privée, et entièrement personnalisables. 🎵

Télécharger pour Android 😍

⚠️ Assurez-vous que « Installer depuis des sources inconnues » est activé dans les paramètres de votre appareil pour installer Hivefy.

✨ Fonctionnalités

🎨 Expérience Moderne

  • Interface et animations inspirées de Spotify
  • Thématisation dynamique Material You
  • Icônes et typographie adaptatives utilisant SpotifyMix

🎧 Lecteur Puissant

  • Lecture en arrière-plan via audio_service
  • Gestes de balayage et contrôles visuels de recherche
  • Transition animée du minilecteur → lecteur complet

💾 Priorité au Hors-Ligne

  • Téléchargement et stockage local de musique (albums / playlists / chansons)
  • Nettoyage automatique des fichiers invalides
  • Notifications de téléchargement en arrière-plan

🔍 Recherche Intelligente & Bibliothèque

  • Recherche globale unifiée (chansons, albums, artistes, playlists)
  • Bibliothèque persistante alimentée par Hive DB
  • Fonctionne parfaitement sans Internet

⚙️ Paramètres & Personnalisation

  • Sélecteur de serveur (Principal / Sauvegarde / Miroir)
  • Gestion du cache & moniteur de téléchargement
  • Changement de thème, effacement du stockage, et plus

🚀 Pour commencer

Prérequis

  • SDK Flutter (>=3.7.2)
  • Android Studio / VS Code configuré pour Flutter
  • Java JDK 17+ (pour la signature de l’application)
  • Appareil / émulateur sous Android 7.0+

Configuration & Exécution 

git clone https://github.com/Harish-Srinivas-07/hivefy.git
cd hivefy
flutter pub get
flutter run

🧩 Stack Technologique

| Catégorie | Packages | | ----------- | ---------------------------------------------------------------------------------- | | État | flutter_riverpod | | Audio | just_audio, audio_service, just_audio_background | | Stockage| hive, shared_preferences, path_provider | | UI/UX | shimmer, cached_network_image, figma_squircle, flashy_tab_bar2, readmore | | Réseau | dio, http, html_unescape | | Système | flutter_local_notifications, disk_space_plus, package_info_plus | | Utilitaires | toastification, permission_handler, internet_connection_checker_plus |

💡 Notes pour les Développeurs

  • Les builds de débogage utilisent le point de terminaison DebugServer (pas de signature requise)
  • Les builds de production sont automatiquement optimisés avec ProGuard
  • Tous les modèles de données incluent toJson et fromJson avec une normalisation d’échappement
  • Vous pouvez modifier la logique backend sous services/ et les flux UI sous views/
