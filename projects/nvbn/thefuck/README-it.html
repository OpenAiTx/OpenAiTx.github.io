The Fuck è un'app magnifica, ispirata da un tweet di @liamosaur, che corregge gli errori nei comandi precedenti della console.

The Fuck è troppo lento? Prova la modalità istantanea sperimentale!

##### Torna all'indice

Installazione

Su macOS o Linux, puoi installare The Fuck tramite [Homebrew][homebrew]:

brew install thefuck

Su Ubuntu / Mint, installa The Fuck con i seguenti comandi: 

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

Su FreeBSD, installa The Fuck con i seguenti comandi: 

pkg install thefuck

Su ChromeOS, installa The Fuck usando chromebrew con il seguente comando: 

crew install thefuck

Su sistemi basati su Arch, installa The Fuck con il seguente comando: 

sudo pacman -S thefuck

Su altri sistemi, installa The Fuck usando pip:

pip install thefuck

In alternativa, puoi usare un gestore di pacchetti del sistema operativo (OS X, Ubuntu, Arch).

# Si raccomanda di inserire questo comando nel tuo .bash_profile, .bashrc, .zshrc o altro script di avvio:

eval $(thefuck --alias)

Puoi usare qualsiasi alias tu voglia, ad esempio per i lunedì:

eval $(thefuck --alias FUCK)

Oppure nella configurazione della tua shell (Bash, Zsh, Fish, Powershell, tcsh).

Le modifiche sono disponibili solo in una nuova sessione della shell. Per renderle disponibili immediatamente, esegui source ~/.bashrc (o il file di configurazione della tua shell come .zshrc).

Per eseguire i comandi corretti senza conferma, usa l'opzione --yeah (o semplicemente -y come abbreviazione, o --hard se sei particolarmente frustrato):

fuck --yeah

Per correggere i comandi ricorsivamente fino al successo, usa l'opzione -r:

fuck -r

##### Torna all'indice

Aggiornamento

pip3 install thefuck --upgrade

Nota: La funzionalità di alias è cambiata nella versione v1.34 di The Fuck

Disinstallazione

Per rimuovere The Fuck, esegui il processo inverso di installazione:

  • cancella o commenta la riga dell'alias thefuck dal file di configurazione della tua shell (Bash, Zsh, Fish, Powershell, tcsh, ...)
  • usa il tuo gestore di pacchetti (brew, pip3, pkg, crew, pip) per disinstallare i binari

Come funziona

The Fuck cerca di abbinare il comando precedente a una regola. Se trova una corrispondenza, viene creato un nuovo comando usando la regola trovata ed eseguito. Le seguenti regole sono abilitate di default:

  • adb_unknown_command – corregge comandi scritti male come adb logcta;
  • ag_literal – aggiunge -Q a ag quando suggerito;
  • aws_cli – corregge comandi scritti male come aws dynamdb scan;
  • az_cli – corregge comandi scritti male come az providers;
  • cargo – esegue cargo build invece di cargo;
  • cargo_no_command – corregge comandi errati come cargo buid;
  • cat_dir – sostituisce cat con ls quando provi a fare cat su una directory;
  • cd_correction – controlla l'ortografia e corregge i comandi cd falliti;
  • cd_cs – cambia cs in cd;
  • cd_mkdir – crea directory prima di entrarvi con cd;
  • cd_parent – cambia cd.. in cd ..;
  • chmod_x – aggiunge il bit di esecuzione;
  • choco_install – aggiunge suffissi comuni per pacchetti chocolatey;
  • composer_not_command – corregge il nome del comando composer;
  • conda_mistype – corregge i comandi conda;
  • cp_create_destination – crea una nuova directory quando tenti di cp o mv verso una non esistente;
  • cp_omitting_directory – aggiunge -a quando fai cp di una directory;
  • cpp11 – aggiunge -std=c++11 mancante a g++ o clang++;
  • dirty_untar – corregge il comando tar x che scompatta nella directory corrente;
  • dirty_unzip – corregge il comando unzip che scompatta nella directory corrente;
  • django_south_ghost – aggiunge --delete-ghost-migrations se fallisce a causa di ghost migration django south;
  • django_south_merge – aggiunge --merge a migrazione django south incoerente;
  • docker_login – esegue un docker login e ripete il comando precedente;
  • docker_not_command – corregge comandi docker errati come docker tags;
  • docker_image_being_used_by_container ‐ rimuove il container che sta usando l'immagine prima di rimuovere l'immagine;
  • dry – corregge ripetizioni come git git push;
  • fab_command_not_found – corregge comandi fabric scritti male;
  • fix_alt_space – sostituisce Alt+Spazio con carattere Spazio;
  • fix_file – apre un file con errore nel tuo $EDITOR;
  • gem_unknown_command – corregge comandi gem errati;
git_add – corregge "pathspec 'foo' did not match any file(s) known to git."*;
