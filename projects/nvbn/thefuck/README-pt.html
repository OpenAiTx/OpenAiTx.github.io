##### Voltar ao Conteúdo

Instalação

No macOS ou Linux, você pode instalar o The Fuck via [Homebrew][homebrew]:

brew install thefuck

No Ubuntu / Mint, instale o The Fuck com os seguintes comandos: 

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

No FreeBSD, instale o The Fuck com os seguintes comandos: 

pkg install thefuck

No ChromeOS, instale o The Fuck usando o chromebrew com o seguinte comando: 

crew install thefuck

Em sistemas baseados em Arch, instale o The Fuck com o seguinte comando: 

sudo pacman -S thefuck

Em outros sistemas, instale o The Fuck usando o pip:

pip install thefuck

Alternativamente, você pode usar um gerenciador de pacotes do sistema operacional (OS X, Ubuntu, Arch).

# É recomendado que você coloque este comando no seu .bash_profile, .bashrc, .zshrc ou outro script de inicialização:

eval $(thefuck --alias)

Você pode usar qualquer alias que quiser, por exemplo para as segundas-feiras:

eval $(thefuck --alias FUCK)

Ou em sua configuração de shell (Bash, Zsh, Fish, Powershell, tcsh).

As alterações só estarão disponíveis em uma nova sessão do shell. Para aplicar imediatamente, execute source ~/.bashrc (ou seu arquivo de configuração de shell, como .zshrc).

Para executar comandos corrigidos sem confirmação, use a opção --yeah (ou apenas -y para abreviar, ou --hard se estiver especialmente frustrado):

fuck --yeah

Para corrigir comandos recursivamente até obter sucesso, use a opção -r:

fuck -r

Atualização

pip3 install thefuck --upgrade

Nota: A funcionalidade de alias mudou na versão v1.34 do The Fuck

Desinstalação

Para remover o The Fuck, reverta o processo de instalação:

  • apague ou comente a linha do alias thefuck do seu Bash, Zsh, Fish, Powershell, tcsh, ... configuração do shell
  • use seu gerenciador de pacotes (brew, pip3, pkg, crew, pip) para desinstalar os binários

Como funciona

The Fuck tenta corresponder o comando anterior com uma regra. Se for encontrada uma correspondência, um novo comando é criado usando a regra correspondente e executado. As seguintes regras estão habilitadas por padrão:

  • adb_unknown_command – corrige comandos digitados incorretamente como adb logcta;
  • ag_literal – adiciona -Q ao ag quando sugerido;
  • aws_cli – corrige comandos digitados incorretamente como aws dynamdb scan;
  • az_cli – corrige comandos digitados incorretamente como az providers;
  • cargo – executa cargo build em vez de cargo;
  • cargo_no_command – corrige comandos errados como cargo buid;
  • cat_dir – substitui cat por ls quando você tenta dar cat em um diretório;
  • cd_correction – verifica ortografia e corrige comandos cd que falharam;
  • cd_cs – muda cs para cd;
  • cd_mkdir – cria diretórios antes de entrar neles com cd;
  • cd_parent – muda cd.. para cd ..;
  • chmod_x – adiciona permissão de execução;
  • choco_install – adiciona sufixos comuns para pacotes chocolatey;
  • composer_not_command – corrige nome de comando do composer;
  • conda_mistype – corrige comandos do conda;
  • cp_create_destination – cria um novo diretório quando você tenta cp ou mv para um que não existe
  • cp_omitting_directory – adiciona -a quando você tenta cp em um diretório;
  • cpp11 – adiciona o -std=c++11 ausente ao g++ ou clang++;
  • dirty_untar – corrige comando tar x que extraiu na pasta atual;
  • dirty_unzip – corrige comando unzip que descompactou na pasta atual;
  • django_south_ghost – adiciona --delete-ghost-migrations para falhas por migração ghost do django south;
  • django_south_merge – adiciona --merge para migração inconsistente do django south;
  • docker_login – executa um docker login e repete o comando anterior;
  • docker_not_command – corrige comandos errados do docker como docker tags;
  • docker_image_being_used_by_container ‐ remove o container que está usando a imagem antes de remover a imagem;
  • dry – corrige repetições como git git push;
  • fab_command_not_found – corrige comandos digitados incorretamente do fabric;
  • fix_alt_space – substitui Alt+Espaço por espaço;
  • fix_file – abre um arquivo com erro no seu $EDITOR;
  • gem_unknown_command – corrige comandos errados do gem;
git_add – corrige "pathspec 'foo' did not match any file(s) known to git."*;
  • git_add_force – adiciona --force ao git add ... quando caminhos estão no .gitignore;
  • git_bisect_usage – corrige git bisect strt, git bisect goood, git bisect rset, etc. durante bisect;
  • git_branch_delete – muda git branch -d para git branch -D;
  • git_branch_delete_checked_out – muda git branch -d para git checkout master && git branch -D ao tentar deletar um branch em uso;
  • git_branch_exists – sugere git branch -d foo, git branch -D foo ou git checkout foo ao criar um branch já existente;
  • git_branch_list – captura git branch list no lugar de git branch e remove o branch criado;
  • git_branch_0flag – corrige comandos como git branch 0v e git branch 0r removendo o branch criado;
  • git_checkout – corrige nome do branch ou cria novo branch;
  • git_clone_git_clone – substitui git clone git clone ... por git clone ...
  • git_clone_missing – adiciona git clone a URLs que parecem apontar para um repositório git.
  • git_commit_add – sugere git commit -a ... ou git commit -p ... após commit anterior se falhou por nada ter sido preparado;
  • git_commit_amend – sugere git commit --amend após commit anterior;
  • git_commit_reset – sugere git reset HEAD~ após commit anterior;
  • git_diff_no_index – adiciona --no-index ao git diff anterior em arquivos não rastreados;
  • git_diff_staged – adiciona --staged ao git diff anterior com saída inesperada;
  • git_fix_stash – corrige comandos git stash (subcomando digitado incorretamente e ausência de save);
  • git_flag_after_filename – corrige fatal: bad flag '...' after filename
  • git_help_aliased – corrige comandos git help substituindo pelo comando real;
  • git_hook_bypass – adiciona flag --no-verify antes de git am, git commit ou git push;
  • git_lfs_mistype – corrige comandos git lfs digitados incorretamente;
  • git_main_master – corrige nome de branch incorreto entre main e master
  • git_merge – adiciona remoto aos nomes de branch;
  • git_merge_unrelated – adiciona --allow-unrelated-histories quando necessário
  • git_not_command – corrige comandos errados do git como git brnch;
  • git_pull – define upstream antes de executar git pull anterior;
  • git_pull_clone – faz clone ao invés de pull quando o repo não existe;
  • git_pull_uncommitted_changes – faz stash das alterações antes do pull e restaura depois;
  • git_push – adiciona --set-upstream origin $branch ao git push anterior que falhou;
  • git_push_different_branch_names – corrige push quando o nome do branch local não corresponde ao do remoto;
  • git_push_pull – executa git pull quando o push foi rejeitado;
  • git_push_without_commits – cria um commit inicial se você esquecer e apenas fizer git add . ao configurar um novo projeto;
  • git_rebase_no_changes – executa git rebase --skip ao invés de git rebase --continue quando não há alterações;
  • git_remote_delete – substitui git remote delete remote_name por git remote remove remote_name;
  • git_rm_local_modifications – adiciona -f ou --cached ao tentar rm um arquivo modificado localmente;
  • git_rm_recursive – adiciona -r ao tentar rm um diretório;
  • git_rm_staged – adiciona -f ou --cached ao tentar rm um arquivo com alterações staged
  • git_rebase_merge_dir – sugere git rebase (--continue | --abort | --skip) ou remover o dir .git/rebase-merge quando um rebase está em andamento;
  • git_remote_seturl_add – executa git remote add ao usar git remote set_url em um remoto inexistente;
  • git_stash – faz stash das modificações locais antes de rebase ou troca de branch;
  • git_stash_pop – adiciona suas modificações locais antes de aplicar o stash, depois faz reset;
  • git_tag_force – adiciona --force ao git tag quando a tag já existe;
  • git_two_dashes – adiciona um traço ausente a comandos como git commit -amend ou git rebase -continue;
  • go_run – adiciona extensão .go ao compilar/executar programas Go;
  • go_unknown_command – corrige comandos errados do go, por exemplo go bulid;
  • gradle_no_task – corrige task não encontrada ou ambígua do gradle;
  • gradle_wrapper – substitui gradle por ./gradlew;
  • grep_arguments_order – corrige ordem de argumentos do grep para situações como grep -lir . test;
  • grep_recursive – adiciona -r ao tentar grep em diretório;
  • grunt_task_not_found – corrige comandos grunt digitados incorretamente;
  • gulp_not_task – corrige tasks gulp digitadas incorretamente;
  • has_exists_script – adiciona ./ quando script/binário existe;
  • heroku_multiple_apps – adiciona --app aos comandos heroku como heroku pg;
  • heroku_not_command – corrige comandos errados do heroku como heroku log;
  • history – tenta substituir comando pelo mais similar do histórico;
  • hostscli – tenta corrigir uso do hostscli;
  • ifconfig_device_not_found – corrige nomes de dispositivos errados como wlan0 para wlp2s0;
  • java – remove extensão .java ao executar programas Java;
  • javac – adiciona .java ausente ao compilar arquivos Java;
  • lein_not_task – corrige tasks erradas do lein como lein rpl;
  • long_form_help – muda -h para --help quando a versão curta não é suportada
  • ln_no_hard_link – captura criação de hard link em diretórios, sugere link simbólico;
  • ln_s_order – corrige ordem dos argumentos do ln -s;
  • ls_all – adiciona -A ao ls quando a saída está vazia;
  • ls_lah – adiciona -lah ao ls;
  • man – muda seção do manual;
  • man_no_space – corrige comandos man sem espaços, por exemplo mandiff;
  • mercurial – corrige comandos errados do hg;
  • missing_space_before_subcommand – corrige comando sem espaço como npminstall;
  • mkdir_p – adiciona -p ao tentar criar diretório sem pai;
  • mvn_no_command – adiciona clean package ao mvn;
  • mvn_unknown_lifecycle_phase – corrige estágios do ciclo de vida digitados incorretamente no mvn;
  • npm_missing_script – corrige nome de script customizado em npm run-script
