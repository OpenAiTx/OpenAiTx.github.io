安装

在 macOS 或 Linux 上，你可以通过 [Homebrew][homebrew] 安装 The Fuck：

brew install thefuck

在 Ubuntu / Mint 上，使用以下命令安装 The Fuck： 

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

在 FreeBSD 上，使用以下命令安装 The Fuck： 

pkg install thefuck

在 ChromeOS 上，使用 chromebrew 安装 The Fuck： 

crew install thefuck

在 Arch 系统上，使用以下命令安装 The Fuck： 

sudo pacman -S thefuck

在其他系统上，可以通过 pip 安装 The Fuck：

pip install thefuck

或者，你也可以使用操作系统自带的包管理器（OS X、Ubuntu、Arch）。

# 建议你将以下命令添加到你的 .bash_profile、.bashrc、.zshrc 或其他启动脚本中：

eval $(thefuck --alias)

你可以自定义别名，比如用于周一：

eval $(thefuck --alias FUCK)

或添加到你的 shell 配置（Bash、Zsh、Fish、Powershell、tcsh）。

更改只在新 shell 会话中生效。要立即生效，请执行 source ~/.bashrc（或者你的 shell 配置文件如 .zshrc）。

要在无需确认的情况下直接执行修正命令，使用 --yeah 选项（简写为 -y，若特别沮丧可用 --hard）：

fuck --yeah

要递归修正命令直到成功，使用 -r 选项：

fuck -r

升级

pip3 install thefuck --upgrade

卸载

要移除 The Fuck，请逆向操作安装步骤：

  • 删除或注释掉 shell 配置文件中的 thefuck 别名行（Bash、Zsh、Fish、Powershell、tcsh 等）
  • 使用你的包管理器（brew、pip3、pkg、crew、pip）卸载程序

工作原理

The Fuck 会尝试将上一个命令与规则进行匹配。如果匹配成功，则使用匹配的规则生成新的命令并执行。默认启用以下规则：

  • adb_unknown_command – 修正拼写错误的 adb 命令，如 adb logcta;
  • ag_literal – 按建议为 ag 命令添加 -Q;
  • aws_cli – 修正拼写错误的 aws 命令，如 aws dynamdb scan;
  • az_cli – 修正拼写错误的 az 命令，如 az providers;
  • cargo – 运行 cargo build 替代 cargo;
  • cargo_no_command – 修正错误命令，如 cargo buid;
  • cat_dir – 当你试图 cat 目录时，将其替换为 ls;
  • cd_correction – 拼写检查并修正失败的 cd 命令;
  • cd_cs – 将 cs 改为 cd;
  • cd_mkdir – 在 cd 进入之前创建目录;
  • cd_parent – 将 cd.. 改为 cd ..;
  • chmod_x – 添加可执行权限;
  • choco_install – 为 chocolatey 包附加常见后缀;
  • composer_not_command – 修正 composer 命令名称;
  • conda_mistype – 修正 conda 命令;
  • cp_create_destination – 当你试图 cp 或 mv 到不存在的目录时创建新目录;
  • cp_omitting_directory – 当你复制目录时为 cp 添加 -a;
  • cpp11 – 为 g++ 或 clang++ 添加缺失的 -std=c++11;
  • dirty_untar – 修正 tar x 命令导致的解压到当前目录;
  • dirty_unzip – 修正 unzip 命令导致的解压到当前目录;
  • django_south_ghost – 针对 ghost django south migration 失败时添加 --delete-ghost-migrations;
  • django_south_merge – 针对不一致的 django south migration 添加 --merge;
  • docker_login – 执行 docker login 并重复上一个命令;
  • docker_not_command – 修正错误的 docker 命令，如 docker tags;
  • docker_image_being_used_by_container ‐ 在移除镜像前移除正在使用该镜像的容器;
  • dry – 修正如 git git push 这样的重复命令;
  • fab_command_not_found – 修正拼写错误的 fabric 命令;
  • fix_alt_space – 将 Alt+Space 替换为空格字符;
  • fix_file – 用你的 $EDITOR 打开出错的文件;
  • gem_unknown_command – 修正错误的 gem 命令;
git_add – 修正 "pathspec 'foo' did not match any file(s) known to git."*;
  • git_add_force – 当路径被 .gitignore 时，为 git add ... 添加 --force;
  • git_bisect_usage – 在二分调试时修正 git bisect strt、git bisect goood、git bisect rset 等命令;
  • git_branch_delete – 将 git branch -d 改为 git branch -D;
  • git_branch_delete_checked_out – 当尝试删除已签出的分支时，将 git branch -d 改为 git checkout master && git branch -D;
  • git_branch_exists – 当创建已存在分支时，提供 git branch -d foo、git branch -D foo 或 git checkout foo;
  • git_branch_list – 捕获 git branch list 替代 git branch 并移除已创建分支;
  • git_branch_0flag – 修正如 git branch 0v、git branch 0r 等命令并移除已创建分支;
  • git_checkout – 修正分支名或创建新分支;
  • git_clone_git_clone – 将 git clone git clone ... 替换为 git clone ...
  • git_clone_missing – 为像 git 仓库链接的 URL 添加 git clone；
  • git_commit_add – 在上次提交失败（无暂存内容）时，提供 git commit -a ... 或 git commit -p ...;
  • git_commit_amend – 在上次提交后，提供 git commit --amend;
  • git_commit_reset – 在上次提交后，提供 git reset HEAD~;
  • git_diff_no_index – 针对未跟踪文件的上一条 git diff 添加 --no-index;
  • git_diff_staged – 针对输出异常的上一条 git diff 添加 --staged;
  • git_fix_stash – 修正 git stash 命令（子命令拼写错误或缺少 save）;
  • git_flag_after_filename – 修正 fatal: bad flag '...' after filename
  • git_help_aliased – 修正 git help 命令，将 替换为别名对应命令;
  • git_hook_bypass – 在 git am、git commit 或 git push 命令前添加 --no-verify 标志;
  • git_lfs_mistype – 修正拼写错误的 git lfs 命令;
  • git_main_master – 修正 main 和 master 分支名混淆问题;
  • git_merge – 为分支名添加远程;

  • git_not_command – 修正错误的 git 命令，如 git brnch;
  • git_pull – 在执行上一条 git pull 前设置 upstream;
  • git_pull_clone – 当仓库不存在时，使用 clone 替代 pull;
  • git_pull_uncommitted_changes – 拉取前暂存更改，拉取后恢复;
  • git_push – 为失败的 git push 添加 --set-upstream origin $branch;
  • git_push_different_branch_names – 当本地分支名与远程分支名不一致时修正 push;
  • git_push_pull – 当 push 被拒绝时运行 git pull;
  • git_push_without_commits – 在新建项目时，如果你只执行了 git add . 却忘记提交，创建初始提交;
  • git_rebase_no_changes – 当无更改时，替代 git rebase --continue 为 git rebase --skip;
  • git_remote_delete – 将 git remote delete remote_name 替换为 git remote remove remote_name;
  • git_rm_local_modifications – 当你试图删除本地修改文件时，添加 -f 或 --cached;
  • git_rm_recursive – 当你试图删除目录时，添加 -r;
  • git_rm_staged – 当你试图删除带暂存更改的文件时，添加 -f 或 --cached
  • git_rebase_merge_dir – 当正在 rebase 时，提供 git rebase (--continue | --abort | --skip) 或移除 .git/rebase-merge 目录;
  • git_remote_seturl_add – 当 git remote set_url 针对不存在的远程时，运行 git remote add;
  • git_stash – rebase 或切换分支前暂存本地更改;
  • git_stash_pop – 在 pop stash 前暂存本地更改，然后重置;
  • git_tag_force – 当标签已存在时，为 git tag 添加 --force;
  • git_two_dashes – 为如 git commit -amend 或 git rebase -continue 的命令添加缺失的连字符;

  • gradle_wrapper – 将 gradle 替换为 ./gradlew;
  • grep_arguments_order – 修正 grep 参数顺序，如 grep -lir . test;
  • grep_recursive – 当你试图 grep 目录时添加 -r;
  • grunt_task_not_found – 修正拼写错误的 grunt 命令;
  • gulp_not_task – 修正拼写错误的 gulp 任务;
  • has_exists_script – 当脚本/二进制存在时前置 ./;
  • heroku_multiple_apps – 为 heroku 命令（如 heroku pg）添加 --app ;
  • heroku_not_command – 修正错误的 heroku 命令，如 heroku log;
  • history – 尝试用历史中最相似的命令替换当前命令;
  • hostscli – 尝试修正 hostscli 用法;
  • ifconfig_device_not_found – 修正错误的设备名，如 wlan0 改为 wlp2s0;
