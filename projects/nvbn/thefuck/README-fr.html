##### Retour au Sommaire

Installation

Sur macOS ou Linux, vous pouvez installer The Fuck via [Homebrew][homebrew] :

brew install thefuck

Sur Ubuntu / Mint, installez The Fuck avec les commandes suivantes : 

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

Sur FreeBSD, installez The Fuck avec les commandes suivantes : 

pkg install thefuck

Sur ChromeOS, installez The Fuck en utilisant chromebrew avec la commande suivante : 

crew install thefuck

Sur les systèmes basés sur Arch, installez The Fuck avec la commande suivante : 

sudo pacman -S thefuck

Sur d’autres systèmes, installez The Fuck avec pip :

pip install thefuck

Vous pouvez également utiliser un gestionnaire de paquets de votre OS (OS X, Ubuntu, Arch).

# Il est recommandé d’ajouter cette commande dans votre .bash_profile, .bashrc, .zshrc ou autre script de démarrage :

eval $(thefuck --alias)

Vous pouvez utiliser l’alias que vous voulez, par exemple pour le lundi :

eval $(thefuck --alias FUCK)

Ou dans votre configuration de shell (Bash, Zsh, Fish, Powershell, tcsh).

Les changements ne sont disponibles que dans une nouvelle session de shell. Pour rendre les changements disponibles immédiatement, exécutez source ~/.bashrc (ou le fichier de config de votre shell comme .zshrc).

Pour exécuter les commandes corrigées sans confirmation, utilisez l’option --yeah (ou simplement -y pour faire court, ou --hard si vous êtes particulièrement frustré) :

fuck --yeah

Pour corriger les commandes de façon récursive jusqu’à réussir, utilisez l’option -r :

fuck -r

##### Retour au Sommaire

Mise à jour

pip3 install thefuck --upgrade

Remarque : La fonctionnalité d’alias a changé dans la version 1.34 de The Fuck

Désinstallation

Pour supprimer The Fuck, inversez le processus d’installation :

  • supprimez ou commentez la ligne d’alias thefuck de votre fichier de configuration de shell Bash, Zsh, Fish, Powershell, tcsh, ...
  • utilisez votre gestionnaire de paquets (brew, pip3, pkg, crew, pip) pour désinstaller les binaires

Comment ça marche

The Fuck tente de faire correspondre la commande précédente avec une règle. Si une correspondance est trouvée, une nouvelle commande est créée à l’aide de la règle correspondante et exécutée. Les règles suivantes sont activées par défaut :

  • adb_unknown_command – corrige les commandes mal orthographiées comme adb logcta ;
  • ag_literal – ajoute -Q à ag si suggéré ;
  • aws_cli – corrige les commandes mal orthographiées comme aws dynamdb scan ;
  • az_cli – corrige les commandes mal orthographiées comme az providers ;
  • cargo – exécute cargo build au lieu de cargo ;
  • cargo_no_command – corrige les commandes erronées comme cargo buid ;
  • cat_dir – remplace cat par ls si vous essayez de cat un répertoire ;
  • cd_correction – corrige l’orthographe des commandes cd échouées ;
  • cd_cs – change cs en cd ;
  • cd_mkdir – crée les répertoires avant d’y accéder avec cd ;
  • cd_parent – change cd.. en cd .. ;
  • chmod_x – ajoute le bit d’exécution ;
  • choco_install – ajoute des suffixes courants pour les paquets chocolatey ;
  • composer_not_command – corrige le nom de la commande composer ;
  • conda_mistype – corrige les commandes conda ;
  • cp_create_destination – crée un nouveau répertoire lorsque vous tentez de cp ou mv vers un répertoire inexistant
  • cp_omitting_directory – ajoute -a quand vous copiez un répertoire avec cp ;
  • cpp11 – ajoute le -std=c++11 manquant à g++ ou clang++ ;
  • dirty_untar – corrige la commande tar x qui a extrait dans le répertoire courant ;
  • dirty_unzip – corrige la commande unzip qui a extrait dans le répertoire courant ;
  • django_south_ghost – ajoute --delete-ghost-migrations à l’échec de migration ghost django south ;
  • django_south_merge – ajoute --merge à une migration django south incohérente ;
  • docker_login – exécute un docker login puis répète la commande précédente ;
  • docker_not_command – corrige les mauvaises commandes docker comme docker tags ;
  • docker_image_being_used_by_container – supprime le conteneur qui utilise l’image avant de supprimer l’image ;
  • dry – corrige les répétitions comme git git push ;
  • fab_command_not_found – corrige les commandes fabric mal orthographiées ;
  • fix_alt_space – remplace Alt+Espace par un espace classique ;
  • fix_file – ouvre un fichier avec une erreur dans votre $EDITOR ;
  • gem_unknown_command – corrige les mauvaises commandes gem ;
git_add – corrige "pathspec 'foo' did not match any file(s) known to git."* ;
  • git_add_force – ajoute --force à git add ... quand les chemins sont .gitignore’d ;

  • git_branch_list – intercepte git branch list au lieu de git branch et supprime la branche créée ;
  • git_branch_0flag – corrige des commandes telles que git branch 0v et git branch 0r en supprimant la branche créée ;
  • git_checkout – corrige le nom de la branche ou en crée une nouvelle ;
  • git_clone_git_clone – remplace git clone git clone ... par git clone ...
  • git_clone_missing – ajoute git clone à des URLs ressemblant à des dépôts git.
  • git_commit_add – propose git commit -a ... ou git commit -p ... après un commit précédent échoué car rien n’était préparé ;
  • git_commit_amend – propose git commit --amend après le commit précédent ;
  • git_commit_reset – propose git reset HEAD~ après le commit précédent ;

  • git_diff_staged – ajoute --staged au git diff précédent avec une sortie inattendue ;

  • gradle_no_task – corrige les tâches gradle introuvables ou ambiguës ;
