The Fuck es una aplicación magnífica, inspirada por un tweet de @liamosaur, que corrige errores en comandos anteriores de la consola.

¿The Fuck es demasiado lento? ¡Prueba el modo instantáneo experimental!

##### Volver al Contenido

Instalación

En macOS o Linux, puedes instalar The Fuck vía [Homebrew][homebrew]:

brew install thefuck

En Ubuntu / Mint, instala The Fuck con los siguientes comandos: 

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

En FreeBSD, instala The Fuck con los siguientes comandos: 

pkg install thefuck

En ChromeOS, instala The Fuck usando chromebrew con el siguiente comando: 

crew install thefuck

En sistemas basados en Arch, instala The Fuck con el siguiente comando: 

sudo pacman -S thefuck

En otros sistemas, instala The Fuck usando pip:

pip install thefuck

Alternativamente, puedes usar un gestor de paquetes del SO (OS X, Ubuntu, Arch).

# Se recomienda que coloques este comando en tu .bash_profile, .bashrc, .zshrc u otro script de inicio:

eval $(thefuck --alias)

Puedes usar el alias que quieras, por ejemplo para los lunes:

eval $(thefuck --alias FUCK)

O en la configuración de tu shell (Bash, Zsh, Fish, Powershell, tcsh).

Los cambios solo estarán disponibles en una nueva sesión de shell. Para hacer los cambios disponibles inmediatamente, ejecuta source ~/.bashrc (o el archivo de configuración de tu shell como .zshrc).

Para ejecutar comandos corregidos sin confirmación, usa la opción --yeah (o simplemente -y para abreviar, o --hard si estás especialmente frustrado):

fuck --yeah

Para corregir comandos recursivamente hasta tener éxito, usa la opción -r:

fuck -r

Actualización

pip3 install thefuck --upgrade

Nota: La funcionalidad de alias cambió en la versión 1.34 de The Fuck

Desinstalación

Para eliminar The Fuck, revierte el proceso de instalación:

  • borra o comenta la línea del alias thefuck de la configuración de tu shell Bash, Zsh, Fish, Powershell, tcsh, ...
  • utiliza tu gestor de paquetes (brew, pip3, pkg, crew, pip) para desinstalar los binarios

Cómo funciona

The Fuck intenta hacer coincidir el comando anterior con una regla. Si encuentra una coincidencia, crea un nuevo comando usando la regla coincidente y lo ejecuta. Las siguientes reglas están habilitadas por defecto:

  • adb_unknown_command – corrige comandos mal escritos como adb logcta;
  • ag_literal – añade -Q a ag cuando se sugiere;
  • aws_cli – corrige comandos mal escritos como aws dynamdb scan;
  • az_cli – corrige comandos mal escritos como az providers;
  • cargo – ejecuta cargo build en lugar de cargo;
  • cargo_no_command – corrige comandos erróneos como cargo buid;
  • cat_dir – reemplaza cat por ls cuando intentas cat en un directorio;
  • cd_correction – revisa y corrige comandos cd fallidos;
  • cd_cs – cambia cs por cd;
  • cd_mkdir – crea directorios antes de entrar con cd;
  • cd_parent – cambia cd.. por cd ..;
  • chmod_x – añade el bit de ejecución;
  • choco_install – añade sufijos comunes para paquetes de chocolatey;
  • composer_not_command – corrige el nombre del comando de composer;
  • conda_mistype – corrige comandos de conda;
  • cp_create_destination – crea un nuevo directorio cuando intentas cp o mv a uno inexistente;
  • cp_omitting_directory – añade -a cuando haces cp a un directorio;
  • cpp11 – añade el faltante -std=c++11 a g++ o clang++;
  • dirty_untar – corrige el comando tar x que descomprime en el directorio actual;
  • dirty_unzip – corrige el comando unzip que descomprime en el directorio actual;
  • django_south_ghost – añade --delete-ghost-migrations a migraciones fantasma fallidas de django south;
  • django_south_merge – añade --merge a migraciones inconsistentes de django south;
  • docker_login – ejecuta un docker login y repite el comando anterior;
  • docker_not_command – corrige comandos erróneos de docker como docker tags;
  • docker_image_being_used_by_container – elimina el contenedor que está usando la imagen antes de eliminar la imagen;
  • dry – corrige repeticiones como git git push;
  • fab_command_not_found – corrige comandos mal escritos de fabric;
  • fix_alt_space – reemplaza Alt+Espacio por un espacio;
  • fix_file – abre un archivo con error en tu $EDITOR;
  • gem_unknown_command – corrige comandos erróneos de gem;
git_add – corrige "pathspec 'foo' no coincide con ningún archivo conocido por git."*;
  • git_add_force – añade --force a git add ... cuando las rutas están en .gitignore;
  • git_bisect_usage – corrige git bisect strt, git bisect goood, git bisect rset, etc. durante bisect;
  • git_branch_delete – cambia git branch -d por git branch -D;
  • git_branch_delete_checked_out – cambia git branch -d por git checkout master && git branch -D al intentar borrar una rama activa;
  • git_branch_exists – sugiere git branch -d foo, git branch -D foo o git checkout foo cuando se crea una rama que ya existe;
  • git_branch_list – detecta git branch list en lugar de git branch y elimina la rama creada;
  • git_branch_0flag – corrige comandos como git branch 0v y git branch 0r eliminando la rama creada;
  • git_checkout – corrige el nombre de la rama o crea una nueva rama;
  • git_clone_git_clone – reemplaza git clone git clone ... por git clone ...
  • git_clone_missing – añade git clone a URLs que parecen enlazar a un repositorio git.
  • git_commit_add – sugiere git commit -a ... o git commit -p ... después de un commit fallido por no haber archivos en stage;
  • git_commit_amend – sugiere git commit --amend después de un commit anterior;
  • git_commit_reset – sugiere git reset HEAD~ después de un commit anterior;
  • git_diff_no_index – añade --no-index al git diff anterior en archivos no rastreados;
  • git_diff_staged – añade --staged al git diff anterior con salida inesperada;
  • git_fix_stash – corrige comandos git stash (subcomando mal escrito o falta save);
  • git_flag_after_filename – corrige fatal: bad flag '...' after filename
  • git_help_aliased – corrige comandos git help reemplazando por el comando alias;
  • git_hook_bypass – añade --no-verify antes de git am, git commit o git push;
  • git_lfs_mistype – corrige comandos mal escritos de git lfs ;
  • git_main_master – corrige nombre de rama incorrecto entre main y master
  • git_merge – añade remoto a nombres de rama;
  • git_merge_unrelated – añade --allow-unrelated-histories cuando es necesario;
  • git_not_command – corrige comandos erróneos de git como git brnch;
  • git_pull – establece upstream antes de ejecutar git pull anterior;
  • git_pull_clone – clona en lugar de hacer pull cuando el repo no existe;
  • git_pull_uncommitted_changes – guarda cambios antes de hacer pull y los restaura después;
  • git_push – añade --set-upstream origin $branch al git push fallido;
  • git_push_different_branch_names – corrige push cuando el nombre de la rama local no coincide con el remoto;
  • git_push_pull – ejecuta git pull cuando el push fue rechazado;
  • git_push_without_commits – crea un commit inicial si olvidaste y solo hiciste git add . al crear un nuevo proyecto;
  • git_rebase_no_changes – ejecuta git rebase --skip en lugar de git rebase --continue cuando no hay cambios;
  • git_remote_delete – reemplaza git remote delete remote_name por git remote remove remote_name;
  • git_rm_local_modifications – añade -f o --cached cuando intentas rm un archivo modificado localmente;
  • git_rm_recursive – añade -r cuando intentas rm un directorio;
  • git_rm_staged – añade -f o --cached cuando intentas rm un archivo con cambios en stage;
  • git_rebase_merge_dir – sugiere git rebase (--continue | --abort | --skip) o eliminar el directorio .git/rebase-merge cuando hay un rebase en progreso;
  • git_remote_seturl_add – ejecuta git remote add cuando haces git remote set_url sobre un remoto inexistente;
  • git_stash – guarda tus modificaciones locales antes de rebasear o cambiar de rama;
  • git_stash_pop – añade tus modificaciones locales antes de aplicar stash, luego restablece;
  • git_tag_force – añade --force a git tag cuando la etiqueta ya existe;
  • git_two_dashes – añade un guion faltante a comandos como git commit -amend o git rebase -continue;
  • go_run – añade la extensión .go al compilar/ejecutar programas Go;
  • go_unknown_command – corrige comandos erróneos de go, por ejemplo go bulid;
  • gradle_no_task – corrige tareas de gradle no encontradas o ambiguas;
  • gradle_wrapper – reemplaza gradle por ./gradlew;
  • grep_arguments_order – corrige el orden de argumentos de grep para situaciones como grep -lir . test;
  • grep_recursive – añade -r cuando intentas grep en un directorio;
  • grunt_task_not_found – corrige comandos mal escritos de grunt;
  • gulp_not_task – corrige tareas mal escritas de gulp;
  • has_exists_script – antepone ./ cuando el script/binario existe;
  • heroku_multiple_apps – añade --app a comandos de heroku como heroku pg;
  • heroku_not_command – corrige comandos erróneos de heroku como heroku log;
  • history – intenta reemplazar el comando con el más similar del historial;
  • hostscli – intenta corregir el uso de hostscli;
  • ifconfig_device_not_found – corrige nombres de dispositivos erróneos como wlan0 a wlp2s0;
  • java – elimina la extensión .java al ejecutar programas Java;
  • javac – añade la extensión .java faltante al compilar archivos Java;
  • lein_not_task – corrige tareas erróneas de lein como lein rpl;
  • long_form_help – cambia -h por --help cuando la versión corta no es soportada;
  • ln_no_hard_link – detecta creación de enlaces duros en directorios, sugiere enlace simbólico;
  • ln_s_order – corrige el orden de argumentos de ln -s;
  • ls_all – añade -A a ls cuando la salida está vacía;
  • ls_lah – añade -lah a ls;
  • man – cambia la sección del manual;
  • man_no_space – corrige comandos man sin espacio, por ejemplo mandiff;
  • mercurial – corrige comandos erróneos de hg;
  • missing_space_before_subcommand – corrige comandos sin espacio, como npminstall;
  • mkdir_p – añade -p cuando intentas crear un directorio sin padre;
  • mvn_no_command – añade clean package a mvn;
  • mvn_unknown_lifecycle_phase – corrige fases del ciclo de vida mal escritas con mvn;
  • npm_missing_script – corrige el nombre del script personalizado en npm run-script
