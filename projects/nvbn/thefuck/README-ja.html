The Fuck は、@liamosaurの tweet に触発された素晴らしいアプリで、直前のコンソールコマンドのエラーを修正します。

The Fuck が遅すぎると感じますか？ 実験的なインスタントモードをお試しください！

必要要件

  • python (3.5以上)
  • pip
  • python-dev

##### 目次に戻る

インストール

macOSまたはLinuxでは、[Homebrew][homebrew] を使って The Fuck をインストールできます：

brew install thefuck

Ubuntu / Mint では、以下のコマンドで The Fuck をインストールします： 

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

FreeBSD では、以下のコマンドで The Fuck をインストールします： 

pkg install thefuck

ChromeOS では、chromebrew を使用して以下のコマンドでインストールします： 

crew install thefuck

Arch系システムでは、以下のコマンドでインストールします： 

sudo pacman -S thefuck

その他のシステムでは、pip を使って The Fuck をインストールしてください：

pip install thefuck

また、OSのパッケージマネージャー（OS X、Ubuntu、Arch）を利用することもできます。

# このコマンドを .bash_profile、.bashrc、.zshrc などのスタートアップスクリプトに記載することを推奨します：

eval $(thefuck --alias)

任意のエイリアス名を利用可能です。例：月曜日用

eval $(thefuck --alias FUCK)

または、シェル設定（Bash、Zsh、Fish、Powershell、tcsh）に追加してください。

設定の変更は新しいシェルセッションでのみ反映されます。すぐに反映させるには source ~/.bashrc（または .zshrc など）を実行してください。

確認なしで修正コマンドを実行したい場合は、--yeah オプション（または短縮の -y、特に苛立っている場合は --hard）を使います：

fuck --yeah

成功するまでコマンドを再帰的に修正したい場合は、-r オプションを使います：

fuck -r

アップデート

pip3 install thefuck --upgrade

注意: v1.34 でエイリアス機能が変更されました

アンインストール

The Fuck を削除するには、インストール手順を逆に実行してください：

  • Bash、Zsh、Fish、Powershell、tcsh などのシェル設定から thefuck のエイリアス行を削除またはコメントアウト
  • パッケージマネージャー（brew, pip3, pkg, crew, pipなど）でバイナリをアンインストール

仕組み

  • aws_cli – aws dynamdb scan のようなスペルミスを修正
  • az_cli – az providers のようなスペルミスを修正
  • cargo – cargo の代わりに cargo build を実行
  • cargo_no_command – cargo buid のような間違いを修正
  • cat_dir – ディレクトリに cat しようとした場合 ls に置換
  • cd_correction – cdコマンドのスペルチェックと修正
  • cd_cs – cs を cd に変更
  • cd_mkdir – ディレクトリがなければ作成してからcd
  • cd_parent – cd.. を cd .. に修正
  • chmod_x – 実行権限を追加
  • choco_install – chocolateyパッケージの一般的なサフィックスを追加
  • composer_not_command – composerコマンド名の修正
  • conda_mistype – condaコマンドの修正
  • cp_create_destination – 存在しないディレクトリに cp や mv しようとした場合にディレクトリを作成
  • cp_omitting_directory – ディレクトリを cp する際に -a を追加
  • cpp11 – g++ や clang++ に足りない -std=c++11 を追加
  • dirty_untar – 現在のディレクトリで展開してしまった tar x コマンドを修正
  • dirty_unzip – 現在のディレクトリで展開してしまった unzip コマンドを修正
  • django_south_ghost – ゴーストによる失敗に --delete-ghost-migrations を追加
