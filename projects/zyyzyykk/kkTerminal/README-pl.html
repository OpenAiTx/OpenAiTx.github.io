🐳 Wdrażanie przez Docker

  • Pobierz obraz:

docker pull zyyzyykk/kkterminal

  • Utwórz i uruchom kontener z mapowaniem portu: -p port:3000

docker run -d --name kkterminal -p 3000:3000 zyyzyykk/kkterminal

  • Własny napis artystyczny: -e TITLE="ArtWord"

docker run -d --name kkterminal -p 3000:3000 -e TITLE="kkbpro" zyyzyykk/kkterminal

  • Dostęp w przeglądarce: http://server-ip:3000/

💡 Opis funkcji

  • kkTerminal to terminal do połączeń Web SSH. Kliknij ikonę terminala w lewym górnym rogu i wybierz ustawienia połączenia, aby ustanowić połączenie SSH

  • Obsługa integracji na stronach internetowych przez znacznik iframe

  • Obsługa dostosowywania konfiguracji terminala przez parametry URL

  • Obsługa internacjonalizacji oraz przełączania języka między chińskim a angielskim

  • Obsługa lokalnego wdrożenia na komputerze PC, automatycznie otwiera okno przeglądarki po uruchomieniu

  • Obsługa adaptacyjnego rozmiaru okna i chińskiego wprowadzania znaków

  • Obsługa własnych ustawień preferencji, użytkownik może wybrać kolor tła/tekstu, rozmiar czcionki, styl wyświetlania kursora, włączyć TCode dla terminala itp.

  • Obsługa restartu: terminal automatycznie się zrestartuje po zmianie ustawień połączenia SSH lub własnych preferencji, lub można go ręcznie zrestartować po rozłączeniu SSH

  • Obsługa kopiowania i wklejania:

  • Kopiowanie: Jak w terminalu Git, zaznaczenie tekstu automatycznie go kopiuje

  • Wklejanie: Jak w terminalu Cmd, kliknięcie prawym przyciskiem myszy wkleja (wymaga zezwolenia przeglądarki)

  • Obsługa zarządzania plikami, otwórz moduł zarządzania plikami, aby przeglądać, rozpakowywać, wysyłać i pobierać pliki/foldery

  • Obsługa skrótów klawiszowych, takich jak wybieranie wielu/wszystkich plików, kopiowanie i wklejanie, wycinanie, przełączanie zaznaczenia, otwieranie itp.

  • Obsługa przeglądania i edycji plików, modyfikuj pliki i zapisuj je na zdalnym serwerze za pomocą ctrl+s

  • Obsługa TCode (Terminal Code), który umożliwia zautomatyzowane przepływy pracy podobne do skryptów Shell poprzez własny TCode

  • Obsługa rejestracji operacji i synchronizacji w chmurze

  • Obsługa funkcji Cooperate, Monitor i Docker
