Qwen3 MoE 融合版

HF Transformers 中的 Qwen3 MoE 模型（以及所有其他 MoE 模型）以速度緩慢著稱，因為它使用了for 迴圈來存取專家模型。此倉庫的目標是使用單一擁有 24GB VRAM 的 GPU 微調 Qwen3-30B-A3B 並達成高吞吐量。此實作與 HF Transformers 生態系統相容，例如 LoRA、bitsandbytes 4-bit 量化及 Unsloth。使用說明請參考example_train_30b_a3b_unsloth.py。

融合線性層

關鍵部分是實作 moe_fused_linear 函數： 

output[b, o] = sum_i weight[selected_experts[b], o, i] * input[b, i]
已有幾個不錯的實作，例如 triton-kernels、llama.cpp、vLLM。torch._grouped_mm 也正在實作中。我們需要依據專家將 input 進行排序，以提升 weight 的記憶體連續性。

本倉庫的實作主要基於 Unsloth 中的 MoE 核心，其又基於 Triton 的 grouped GEMM。我加入了 strides、遮罩以及針對小型或「細長」矩陣的自動調參配置，這些對 LoRA 是必要的。

我希望保持程式碼的可讀性與易於理解。我只使用了 Triton 最成熟的功能，如 load 與 store，而非 TMA 或 swizzle 等較複雜的功能。

LoRA

融合線性層的 LoRA 是先在每個專家中的線性層建立 LoRA，然後沿著專家維度堆疊。對於形狀為 (num_experts, out_features, in_features) 的權重張量，兩個 LoRA 權重分別為 lora_A: (num_experts, lora_rank, in_features), lora_B: (num_experts, out_features, lora_rank)。因此，先前訓練好的 LoRA 可以無損地轉換成融合格式。

qwen3_moe_fused/convert.py 中的函式可以在融合與非融合格式間轉換模型或 LoRA。當你在融合格式下訓練好 LoRA 後，可以轉換回非融合格式，再轉換成其他格式，如 GGUF。

授權

qwen3_moe_fused/grouped_gemm/ 中的檔案是修改自 Unsloth MoE 核心，因此採用 AGPLv3 授權，詳見 說明。若要更穩健且高效的整合，可以使用 MIT 授權的 triton-kernels 作為替代。

本倉庫其餘部分，包括修改自 Transformers、PEFT 與 bitsandbytes 的檔案，均採用 Apache-2.0 授權。
