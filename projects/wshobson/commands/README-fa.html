🤖 جریان‌های کاری

هماهنگی چند زیرعامل برای وظایف پیچیده:

توسعه ویژگی

  • feature-development - زیرعامل‌های بک‌اند، فرانت‌اند، تست و استقرار ویژگی‌های کامل را می‌سازند
  • full-review - چندین زیرعامل بررسی، تحلیل جامع کد را ارائه می‌دهند
  • smart-fix - مسائل را تحلیل و به زیرعامل متخصص مناسب واگذار می‌کند

فرآیندهای توسعه

  • git-workflow - پیاده‌سازی جریان‌های کاری مؤثر گیت با استراتژی شاخه‌بندی و قالب PR
  • improve-agent - بهبود عملکرد زیرعامل‌ها از طریق بهینه‌سازی پرامپت
  • legacy-modernize - مدرن‌سازی کدهای قدیمی با استفاده از زیرعامل‌های تخصصی
  • ml-pipeline - ساخت پایپ‌لاین‌های ML با زیرعامل‌های مهندسی داده و یادگیری ماشین
  • multi-platform - ساخت اپلیکیشن‌های چندسکویی با هماهنگی زیرعامل‌ها
  • workflow-automate - خودکارسازی CI/CD، مانیتورینگ و جریان‌های استقرار

جریان‌های کاری هماهنگ با زیرعامل

  • full-stack-feature - ویژگی‌های چندسکویی با زیرعامل‌های بک‌اند، فرانت‌اند و موبایل
  • security-hardening - پیاده‌سازی با رویکرد امنیتی و زیرعامل‌های تخصصی
  • data-driven-feature - ویژگی‌های مبتنی بر ML با زیرعامل‌های علم داده
  • performance-optimization - بهینه‌سازی سرتاسری با زیرعامل‌های عملکرد
  • incident-response - رفع رخدادهای تولید با زیرعامل‌های عملیات

🔧 ابزارها (دستورات تک‌منظوره)

هوش مصنوعی و یادگیری ماشین

  • ai-assistant - ساخت دستیار هوشمند و چت‌بات آماده تولید
  • ai-review - بررسی تخصصی برای کدهای AI/ML
  • langchain-agent - ساخت عوامل LangChain/LangGraph با الگوهای مدرن
  • ml-pipeline - ساخت پایپ‌لاین ML سرتاسری با MLOps
  • prompt-optimize - بهینه‌سازی پرامپت‌های AI برای عملکرد و کیفیت

معماری و کیفیت کد

  • code-explain - تولید توضیحات دقیق برای کد پیچیده
  • code-migrate - مهاجرت کد بین زبان‌ها، فریم‌ورک‌ها یا نسخه‌ها
  • refactor-clean - بازآرایی کد برای نگهداری‌پذیری و عملکرد بهتر
  • tech-debt - تحلیل و اولویت‌بندی بدهی فنی

داده و پایگاه داده

  • data-pipeline - طراحی معماری پایپ‌لاین داده مقیاس‌پذیر
  • data-validation - پیاده‌سازی سیستم‌های جامع اعتبارسنجی داده
  • db-migrate - استراتژی‌های پیشرفته مهاجرت پایگاه داده

DevOps و زیرساخت

  • deploy-checklist - تولید پیکربندی و چک‌لیست استقرار
  • docker-optimize - استراتژی‌های پیشرفته بهینه‌سازی کانتینر
  • k8s-manifest - استقرار Kubernetes در سطح تولید
  • monitor-setup - راه‌اندازی مانیتورینگ و مشاهده‌پذیری جامع
  • slo-implement - پیاده‌سازی اهداف سطح سرویس (SLO)
  • workflow-automate - خودکارسازی جریان‌های توسعه و عملیات

توسعه و تست

  • api-mock - ایجاد ماک API واقعی برای توسعه و تست
  • api-scaffold - تولید اندپوینت‌های API آماده تولید با پیاده‌سازی کامل
  • test-harness - ساخت مجموعه تست جامع با شناسایی فریم‌ورک

امنیت و انطباق

  • accessibility-audit - تست جامع دسترس‌پذیری و رفع مشکلات
  • compliance-check - اطمینان از تطابق با مقررات (GDPR، HIPAA، SOC2)
  • security-scan - اسکن امنیتی جامع با رفع خودکار

اشکال‌زدایی و تحلیل

  • debug-trace - استراتژی‌های پیشرفته اشکال‌زدایی و ردیابی
  • error-analysis - تحلیل عمیق الگوهای خطا و راهکارها
  • error-trace - ردیابی و تشخیص خطاهای تولید
  • issue - ایجاد issue ساختاریافته برای GitHub/GitLab

وابستگی و پیکربندی

  • config-validate - اعتبارسنجی و مدیریت پیکربندی برنامه
  • deps-audit - بررسی وابستگی‌ها از نظر امنیت و مجوز
  • deps-upgrade - ارتقاء ایمن وابستگی‌های پروژه

مستندسازی و همکاری

  • doc-generate - تولید مستندات جامع
  • git-workflow - پیاده‌سازی جریان‌های مؤثر گیت
  • pr-enhance - بهبود Pull Request با بررسی کیفیت

بهینه‌سازی هزینه

  • cost-optimize - بهینه‌سازی هزینه‌های زیرساخت و ابر

راه‌اندازی و پذیرش اعضا

  • onboard - راه‌اندازی محیط توسعه برای اعضای جدید تیم

ابزار زیرعامل

  • multi-agent-review - بازبینی کد با دیدگاه‌های مختلف و زیرعامل‌های تخصصی
  • smart-debug - اشکال‌زدایی عمیق با زیرعامل اشکال‌زدا و عملکرد
  • multi-agent-optimize - بهینه‌سازی فول‌استک با چندین زیرعامل
  • context-save - ذخیره زمینه پروژه با زیرعامل مدیریت زمینه
  • context-restore - بازیابی زمینه ذخیره شده برای تداوم

دستورات پیشرفته

امنیت و توسعه عملیات (DevOps)

#### /security-scan

اسکن امنیتی جامع با اصلاح خودکار مشکلات.

  • اسکن چند ابزاری: Bandit، Safety، Trivy، Semgrep، ESLint Security، Snyk
  • رفع خودکار: آسیب‌پذیری‌های رایج به طور خودکار اصلاح می‌شوند
  • یکپارچه‌سازی با CI/CD: گیت‌های امنیتی برای GitHub Actions و GitLab CI
  • اسکن کانتینر: تحلیل آسیب‌پذیری تصاویر
  • شناسایی اسرار: یکپارچه‌سازی GitLeaks و TruffleHog

#### /docker-optimize

استراتژی‌های پیشرفته بهینه‌سازی کانتینر.

  • بهینه‌سازی هوشمند: تحلیل و ارائه پیشنهادات بهبود
  • ساخت چندمرحله‌ای: Dockerfileهای بهینه‌شده ویژه فریم‌ورک‌ها
  • ابزارهای مدرن: BuildKit، Bun، UV برای ساخت سریع‌تر
  • سخت‌سازی امنیتی: تصاویر Distroless، کاربران غیر ریشه‌ای
  • یکپارچه با دستورات دیگر: کار با خروجی‌های /api-scaffold

#### /k8s-manifest

استقرار Kubernetes در سطح تولید.

  • الگوهای پیشرفته: استانداردهای امنیتی Pod، سیاست‌های شبکه، OPA
  • آماده GitOps: پیکربندی FluxCD و ArgoCD
  • قابلیت مشاهده: Prometheus ServiceMonitors، OpenTelemetry
  • مقیاس‌پذیری خودکار: پیکربندی HPA، VPA و مقیاس‌ساز خوشه
  • مش سرویس: یکپارچه‌سازی Istio/Linkerd

فرانت‌اند و داده

#### /db-migrate

استراتژی‌های پیشرفته مهاجرت پایگاه داده.

  • چند پایگاه داده‌ای: PostgreSQL، MySQL، MongoDB، DynamoDB
  • بدون توقف: استقرار blue-green، مهاجرت غلتان
  • منبع رویداد: یکپارچه‌سازی Kafka/Kinesis برای CDC
  • چندسکویی: مدیریت ماندگاری چندزبانه
  • پایش: بررسی سلامت مهاجرت و بازگشت به عقب

🔀 Workflowها و ابزارهای Subagent

  • حل مسئله: تحلیل و رفع مشکلات به صورت تطبیقی

🛠️ ابزارهای تخصصی

  • راه‌اندازی زیرساخت: پیکربندی محیط‌ها
  • تولید کد: ایجاد پیاده‌سازی‌های خاص
  • تحلیل: تولید گزارش بدون رفع مشکلات
  • وظایف دامنه‌ای: عملیات بسیار تخصصی

مثال‌ها:
  • "سیستم احراز هویت کاربر را پیاده‌سازی کن" → /feature-development
  • "رفع مشکلات عملکردی در سراسر استک" → /smart-fix
  • "مدرن‌سازی مونولیت قدیمی" → /legacy-modernize

🔧 ابزارها را زمانی استفاده کنید که:

  • نیاز به تخصص خاص - وظیفه‌ای واضح و متمرکز در یک حوزه
  • کنترل دقیق مورد نیاز است - می‌خواهید جزئیات پیاده‌سازی را تعیین کنید
  • بخشی از جریان کاری دستی - ادغام با فرآیندهای موجود
  • نیاز به تخصص عمیق - پیاده‌سازی در سطح متخصص
  • ساخت بر پایه کار قبلی - بهبود یا اصلاح خروجی‌های قبلی

مثال‌ها:
  • "مانیفست‌های Kubernetes بساز" → /k8s-manifest
  • "اسکن آسیب‌پذیری‌های امنیتی" → /security-scan
  • "تولید مستندات API" → /doc-generate
