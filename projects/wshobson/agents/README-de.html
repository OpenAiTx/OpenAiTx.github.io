Claude Code Subagenten-Sammlung

Eine umfassende Sammlung spezialisierter KI-Subagenten für Claude Code, entwickelt zur Verbesserung von Entwicklungs-Workflows mit domänenspezifischem Fachwissen.

Übersicht

Dieses Repository enthält 57 spezialisierte Subagenten, die die Fähigkeiten von Claude Code erweitern. Jeder Subagent ist Experte in einem bestimmten Bereich, wird automatisch je nach Kontext aktiviert oder kann bei Bedarf explizit aufgerufen werden. Alle Agenten sind mit spezifischen Claude-Modellen je nach Aufgabenkomplexität für optimale Leistung und Kosteneffizienz konfiguriert.

Installation

Diese Subagenten sind automatisch verfügbar, wenn sie im Verzeichnis ~/.claude/agents/ abgelegt werden.

cd ~/.claude
git clone https://github.com/wshobson/agents.git

Verwendung

Automatische Ausführung

Claude Code delegiert automatisch an den entsprechenden Subagenten basierend auf dem Aufgaben-Kontext und der Beschreibung des Subagenten.

"Use the code-reviewer to check my recent changes"
"Have the security-auditor scan for vulnerabilities"
"Get the performance-engineer to optimize this bottleneck"

Subagentenformat

Jeder Subagent folgt dieser Struktur: 

---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools

System prompt defining the subagent's role and capabilities

Modellkonfiguration

Ab Claude Code v1.0.64 können Subagenten angeben, welches Claude-Modell sie verwenden sollen. Dies ermöglicht eine kosteneffiziente Aufgabenverteilung basierend auf der Komplexität:

  • Niedrige Komplexität (Haiku): Einfache Aufgaben wie grundlegende Datenanalyse, Dokumentationserstellung und Standardantworten
  • Mittlere Komplexität (Sonnet): Entwicklungsaufgaben, Code-Review, Testing und übliche Ingenieurarbeit
  • Hohe Komplexität (Opus): Kritische Aufgaben wie Sicherheitsüberprüfung, Architektur-Review, Incident Response und KI/ML-Engineering

Verfügbare Modelle (mit vereinfachten Namen ab Claude Code v1.0.64):
  • haiku - Schnell und kosteneffizient für einfache Aufgaben
  • sonnet - Ausgewogene Leistung für die meisten Entwicklungsarbeiten
  • opus - Am leistungsfähigsten für komplexe Analysen und kritische Aufgaben

Wenn kein Modell angegeben ist, verwendet der Subagent das Standardmodell des Systems.

Agenten-Orchestrierungsmuster

Claude Code koordiniert Agenten automatisch mit diesen gängigen Mustern:

Sequentielle Workflows

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

Bedingte Verzweigung

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

Überprüfung & Validierung

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

Best Practices

🎯 Aufgabenverteilung

  • Lassen Sie Claude Code automatisch delegieren – Der Hauptagent analysiert den Kontext und wählt optimale Agenten aus
  • Seien Sie spezifisch bei Anforderungen – Geben Sie Einschränkungen, Technologiestack und Qualitätsanforderungen an
  • Vertrauen Sie auf Expertenwissen der Agenten – Jeder Agent ist für seinen Bereich optimiert

🔄 Multi-Agenten-Workflows

  • Beginnen Sie mit Anfragen auf hoher Ebene – Lassen Sie Agenten komplexe mehrstufige Aufgaben koordinieren
  • Stellen Sie Kontext zwischen Agenten bereit – Sorgen Sie dafür, dass Agenten die nötigen Hintergrundinformationen haben
  • Überprüfen Sie Integrationspunkte – Prüfen Sie, wie die Ergebnisse verschiedener Agenten zusammenarbeiten

🎛️ Explizite Steuerung

  • Nutzen Sie gezielte Aufrufe für spezifische Anforderungen – Wenn Sie die Sichtweise eines bestimmten Experten wünschen
  • Kombinieren Sie mehrere Agenten strategisch – Verschiedene Spezialisten können gegenseitig ihre Arbeit validieren
  • Fordern Sie spezifische Prüfungsabläufe an – „Lassen Sie security-auditor das API-Design des backend-architect überprüfen“

📈 Optimierung

  • Überwachen Sie die Effektivität der Agenten – Lernen Sie, welche Agenten für Ihre Anwendungsfälle am besten geeignet sind
  • Iterieren Sie bei komplexen Aufgaben – Nutzen Sie das Feedback der Agenten, um Anforderungen zu verfeinern
  • Stärken der Agenten nutzen – Passen Sie die Aufgabenkomplexität an die Fähigkeiten der Agenten an

Beitrag leisten

So fügen Sie einen neuen Subagenten hinzu:

  • Erstellen Sie eine neue .md-Datei nach obigem Format
  • Verwenden Sie kleingeschriebene, durch Bindestriche getrennte Namen
  • Schreiben Sie klare Beschreibungen, wann der Subagent verwendet werden soll
  • Fügen Sie spezifische Anweisungen in die System-Prompt ein

Fehlerbehebung

Häufige Probleme

Agent wird nicht automatisch aufgerufen:

  • Stellen Sie sicher, dass Ihre Anfrage eindeutig die Domäne angibt (z. B. „Performance-Problem“ → performance-engineer)
  • Seien Sie spezifisch beim Aufgabentyp (z. B. „Code prüfen“ → code-reviewer)

Unerwartete Agentenauswahl:
  • Geben Sie mehr Kontext zu Ihrem Tech-Stack und Anforderungen an
  • Verwenden Sie expliziten Aufruf, wenn Sie einen bestimmten Agenten benötigen

Mehrere Agenten geben widersprüchliche Ratschläge:
  • Das ist normal – verschiedene Spezialisten haben unterschiedliche Prioritäten
  • Fordern Sie eine Klärung an: „Vereinbaren Sie die Empfehlungen von security-auditor und performance-engineer“

Agent scheint Kontext zu fehlen:
  • Geben Sie Hintergrundinformationen in Ihrer Anfrage an
  • Verweisen Sie auf vorherige Gespräche oder etablierte Muster

Hilfe erhalten

Wenn Agenten nicht wie erwartet funktionieren:

  • Prüfen Sie die Agentenbeschreibungen in den jeweiligen Dateien
  • Verwenden Sie präzisere Formulierungen in Ihren Anfragen
  • Testen Sie explizit bestimmte Agenten durch gezielten Aufruf
  • Geben Sie mehr Kontext zu Ihrem Projekt und Ihren Zielen an
