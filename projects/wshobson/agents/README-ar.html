مجموعة وكلاء كلود كود الفرعيين

مجموعة شاملة من وكلاء الذكاء الاصطناعي الفرعيين المتخصصين لـ Claude Code، مصممة لتعزيز سير عمل التطوير بخبرات متخصصة في المجالات المختلفة.

نظرة عامة

يحتوي هذا المستودع على 57 وكيلاً فرعياً متخصصاً يوسّع من إمكانيات Claude Code. كل وكيل فرعي هو خبير في مجال محدد، ويتم استدعاؤه تلقائياً بناءً على السياق أو بشكل صريح عند الحاجة. جميع الوكلاء مهيؤون بنماذج Claude محددة حسب تعقيد المهمة لتحقيق الأداء الأمثل والتكلفة الفعالة.

الوكلاء الفرعيون المتاحون

التطوير والهندسة المعمارية

  • backend-architect - تصميم واجهات RESTful، حدود الخدمات المصغرة، ومخططات قواعد البيانات
  • frontend-developer - بناء مكونات React، تنفيذ التصاميم المتجاوبة، وإدارة حالة العميل
  • ui-ux-designer - إنشاء تصاميم الواجهات، المخططات السلكية، وأنظمة التصميم
  • mobile-developer - تطوير تطبيقات React Native أو Flutter مع تكاملات أصلية
  • graphql-architect - تصميم مخططات GraphQL، الحلول، والاتحاد
  • architect-reviewer - مراجعة تغييرات الكود لضمان الاتساق المعماري ونماذج التصميم

المتخصصون في اللغات البرمجية

  • python-pro - كتابة كود Python متقن مع ميزات متقدمة وتحسينات
  • golang-pro - كتابة كود Go متقن باستخدام goroutines، القنوات، والواجهات
  • rust-pro - كتابة كود Rust متقن بأنماط الملكية، الأعمار، وتنفيذ السمات
  • c-pro - كتابة كود C فعال مع إدارة ذاكرة صحيحة واستدعاءات النظام
  • cpp-pro - كتابة كود C++ متقن بميزات حديثة، RAII، مؤشرات ذكية، وخوارزميات STL
  • javascript-pro - إتقان JavaScript الحديث مع ES6+، الأنماط غير المتزامنة، وواجهات Node.js
  • typescript-pro - إتقان TypeScript مع أنواع متقدمة، قوالب، وأمان صارم للأنواع
  • php-pro - كتابة كود PHP متقن مع ميزات حديثة وتحسينات في الأداء
  • java-pro - إتقان Java الحديثة مع الجداول، التزامن، وتحسين JVM
  • elixir-pro - كتابة كود Elixir متقن بأنماط OTP، البرمجة الوظيفية، وأطر عمل Phoenix
  • csharp-pro - كتابة كود C# حديث مع ميزات متقدمة وتحسين .NET
  • unity-developer - بناء ألعاب Unity مع سكريبتات محسنة وضبط الأداء
  • minecraft-bukkit-pro - إتقان تطوير إضافات خوادم Minecraft باستخدام Bukkit وSpigot وPaper APIs
  • ios-developer - تطوير تطبيقات iOS أصلية باستخدام Swift/SwiftUI
  • sql-pro - كتابة استعلامات SQL معقدة، تحسين خطط التنفيذ، وتصميم مخططات منطبقة

البنية التحتية والعمليات

  • devops-troubleshooter - تصحيح مشكلات الإنتاج، تحليل السجلات، وإصلاح فشل النشر
  • deployment-engineer - إعداد خطوط CI/CD، حاويات Docker، وعمليات النشر السحابية
  • cloud-architect - تصميم بنية AWS/Azure/GCP وتحسين تكاليف السحابة
  • database-optimizer - تحسين استعلامات SQL، تصميم فهارس فعالة، وإدارة ترحيل قواعد البيانات
  • database-admin - إدارة عمليات قواعد البيانات، النسخ الاحتياطي، التكرار، والمراقبة
  • terraform-specialist - كتابة وحدات Terraform متقدمة، إدارة ملفات الحالة، وتطبيق أفضل ممارسات IaC
  • incident-responder - التعامل مع حوادث الإنتاج بسرعة ودقة
  • network-engineer - تصحيح اتصال الشبكة، إعداد موازني التحميل، وتحليل أنماط المرور
  • dx-optimizer - متخصص تجربة المطورين لتحسين الأدوات والإعداد وسير العمل

الجودة والأمان

  • code-reviewer - مراجعة كود احترافية مع تركيز عميق على أمان التكوين وموثوقية الإنتاج
  • security-auditor - مراجعة الكود لاكتشاف الثغرات وضمان التوافق مع OWASP
  • test-automator - أنشئ مجموعات اختبار شاملة تشمل اختبارات الوحدة، التكامل، والنهاية إلى النهاية
  • performance-engineer - تحليل أداء التطبيقات، تحسين نقاط الاختناق، وتنفيذ استراتيجيات التخزين المؤقت
  • debugger - متخصص في تصحيح الأخطاء، فشل الاختبارات، والسلوك غير المتوقع
  • error-detective - البحث في السجلات وقواعد الكود عن أنماط الأخطاء، تتبعات المكدس، والشذوذ
  • search-specialist - باحث ويب خبير يستخدم تقنيات البحث المتقدمة والتركيب

البيانات والذكاء الاصطناعي

  • data-scientist - خبير تحليل البيانات لاستعلامات SQL، عمليات BigQuery، والرؤى البيانية
  • data-engineer - بناء خطوط أنابيب ETL، مستودعات البيانات، وهياكل البث
  • ai-engineer - بناء تطبيقات LLM، أنظمة RAG، وخطوط الأنابيب الخاصة بالمحفزات
  • ml-engineer - تنفيذ خطوط أنابيب التعلم الآلي، تقديم النماذج، وهندسة الميزات
  • mlops-engineer - بناء خطوط أنابيب ML، تتبع التجارب، وسجلات النماذج
  • prompt-engineer - تحسين المحفزات لأنظمة LLM والذكاء الاصطناعي

المجالات المتخصصة

  • api-documenter - إنشاء مواصفات OpenAPI/Swagger وكتابة الوثائق التقنية للمطورين
  • payment-integration - دمج Stripe، PayPal، ومعالجات الدفع
  • quant-analyst - بناء نماذج مالية، اختبار استراتيجيات التداول، وتحليل بيانات السوق
  • risk-manager - مراقبة مخاطر المحفظة، مضاعفات المخاطر، وحدود المراكز
  • legacy-modernizer - إعادة هيكلة قواعد الكود القديمة وتنفيذ التحديث التدريجي
  • context-manager - إدارة السياق عبر وكلاء متعددين ومهام طويلة الأمد

التوثيق

  • docs-architect - إنشاء وثائق تقنية شاملة من قواعد الكود الحالية
  • reference-builder - إنشاء مراجع تقنية شاملة ووثائق API
  • tutorial-engineer - إنشاء شروحات تعليمية خطوة بخطوة ومحتوى تدريبي من الكود

الأعمال والتسويق

  • business-analyst - تحليل المقاييس، إنشاء التقارير، وتتبع مؤشرات الأداء الرئيسية
  • content-marketer - كتابة منشورات المدونات، محتوى وسائل التواصل الاجتماعي، ونشرات البريد الإلكتروني
  • sales-automator - صياغة رسائل بريد إلكتروني أولية، متابعات، وقوالب العروض
  • customer-support - معالجة تذاكر الدعم، الردود على الأسئلة الشائعة، ورسائل العملاء
  • legal-advisor - صياغة سياسات الخصوصية، شروط الخدمة، إخلاءات المسؤولية، والإشعارات القانونية

سير العمل متعدد الوكلاء

تعمل هذه الوكلاء الفرعيون معًا بسلاسة، ولتنظيمات أكثر تعقيدًا، يمكنك استخدام مجموعة Claude Code Commands التي توفر 52 أمرًا مسبق الصنع تستفيد من هذه الوكلاء الفرعيين في سير عمل متقدم.

# Feature development workflow
"Implement user authentication feature"

Automatically uses: backend-architect → frontend-developer → test-automator → security-auditor
Performance optimization workflow  

"Optimize the checkout process performance"

Automatically uses: performance-engineer → database-optimizer → frontend-developer
Production incident workflow

"Debug high memory usage in production"

Automatically uses: incident-responder → devops-troubleshooter → error-detective → performance-engineer
Network connectivity workflow

"Fix intermittent API timeouts"

Automatically uses: network-engineer → devops-troubleshooter → performance-engineer
Database maintenance workflow

"Set up disaster recovery for production database"

Automatically uses: database-admin → database-optimizer → incident-responder
ML pipeline workflow

"Build end-to-end ML pipeline with monitoring"

Automatically uses: mlops-engineer → ml-engineer → data-engineer → performance-engineer
Product launch workflow

"Launch new feature with marketing campaign"

Automatically uses: business-analyst → content-marketer → sales-automator → customer-support

سير العمل المتقدم باستخدام أوامر السلاش

للحصول على تنسيق أكثر تعقيدًا لوكلاء فرعيين متعددين، استخدم مستودع الأوامر المرافق:

# Complex feature development (8+ subagents)
/full-stack-feature Build user dashboard with real-time analytics
Production incident response (5+ subagents)

/security-hardening Implement OAuth2 with zero-trust architecture

تنسيق الوكيل الفرعي

يتبع كل وكيل فرعي هذا الهيكل: 

---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools

System prompt defining the subagent's role and capabilities

إعداد النموذج

اعتبارًا من Claude Code v1.0.64، يمكن للوكلاء الفرعيين تحديد نموذج Claude الذي يجب عليهم استخدامه. يسمح ذلك بتفويض المهام بشكل فعال من حيث التكلفة استنادًا إلى التعقيد:

  • تعقيد منخفض (Haiku): مهام بسيطة مثل تحليل البيانات الأساسي، إنشاء الوثائق، والاستجابات القياسية
  • تعقيد متوسط (Sonnet): مهام التطوير، مراجعة الكود، الاختبار، والعمل الهندسي القياسي
  • تعقيد عالٍ (Opus): مهام حرجة مثل تدقيق الأمان، مراجعة البنية، الاستجابة للحوادث، وهندسة الذكاء الاصطناعي/تعلم الآلة

النماذج المتوفرة (باستخدام التسميات المبسطة اعتبارًا من Claude Code v1.0.64):
  • haiku - سريع وفعّال من حيث التكلفة للمهام البسيطة
  • sonnet - أداء متوازن لمعظم أعمال التطوير
  • opus - الأكثر قدرة على التحليل المعقد والمهام الحرجة

إذا لم يتم تحديد نموذج، سيستخدم الوكيل الفرعي النموذج الافتراضي للنظام.

أنماط تنظيم الوكلاء

يقوم Claude Code تلقائيًا بتنسيق الوكلاء باستخدام هذه الأنماط الشائعة:

سير العمل التسلسلي

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

التفرع الشرطي

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

المراجعة والتحقق

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation
