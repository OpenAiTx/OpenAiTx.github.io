Коллекция подагентов Claude Code

Обширная коллекция специализированных ИИ-подагентов для Claude Code, предназначенная для повышения эффективности рабочих процессов разработки за счет экспертных знаний в отдельных областях.

Обзор

Этот репозиторий содержит 58 специализированных подагентов, расширяющих возможности Claude Code. Каждый подагент является экспертом в определенной области, автоматически вызывается по контексту или может быть вызван вручную при необходимости. Все агенты настроены на конкретные модели Claude в зависимости от сложности задачи для оптимальной производительности и экономичности.

Доступные подагенты

Разработка и архитектура

  • backend-architect – Проектирование RESTful API, границ микросервисов и схем баз данных
  • frontend-developer – Разработка компонентов React, реализация адаптивных макетов и управление состоянием на клиенте
  • ui-ux-designer – Создание дизайнов интерфейсов, wireframes и дизайн-систем
  • mobile-developer – Разработка приложений на React Native или Flutter с нативными интеграциями
  • graphql-architect – Проектирование схем GraphQL, резолверов и федерации
  • architect-reviewer – Рецензирование изменений кода на архитектурную согласованность и шаблоны

Специалисты по языкам программирования

Установка

Эти субагенты автоматически доступны, если размещены в каталоге ~/.claude/agents/.

cd ~/.claude
git clone https://github.com/wshobson/agents.git

Использование

Автоматический вызов

Claude Code автоматически делегирует задачу соответствующему субагенту в зависимости от контекста задачи и описания субагента.

Явный вызов

Упомяните субагента по имени в вашем запросе: 
"Use the code-reviewer to check my recent changes"
"Have the security-auditor scan for vulnerabilities"
"Get the performance-engineer to optimize this bottleneck"

Формат субагента

Каждый субагент следует этой структуре: 

---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools

System prompt defining the subagent's role and capabilities

Конфигурация модели

Начиная с Claude Code v1.0.64, субагенты могут указывать, какую модель Claude им следует использовать. Это позволяет эффективно делегировать задачи с учетом их сложности и стоимости:

  • Низкая сложность (Haiku): Простые задачи, такие как базовый анализ данных, генерация документации и стандартные ответы
  • Средняя сложность (Sonnet): Задачи разработки, ревью кода, тестирование и обычная инженерная работа
  • Высокая сложность (Opus): Критические задачи, такие как аудит безопасности, ревью архитектуры, реагирование на инциденты и инженерия ИИ/МО

Доступные модели (упрощенные названия с Claude Code v1.0.64):
  • haiku – Быстрая и экономичная для простых задач
  • sonnet – Сбалансированная производительность для большинства задач разработки
  • opus – Наиболее способная для сложного анализа и критических задач

Если модель не указана, субагент использует модель по умолчанию системы.

Шаблоны оркестрации агентов

Claude Code автоматически координирует агентов, используя эти распространенные шаблоны:

Последовательные рабочие процессы

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

Условное ветвление

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

Проверка и подтверждение

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

  • search-specialist: Глубокий веб-поиск, конкурентный анализ, проверка фактов

📚 Документация

  • api-documenter: Спецификации OpenAPI/Swagger, документация API
  • docs-architect: Комплексная техническая документация, архитектурные руководства, системные мануалы
  • reference-builder: Подробные справочники API, руководства по конфигурации, документация параметров
  • tutorial-engineer: Пошаговые туториалы, образовательные маршруты, учебный контент

  • content-marketer: SEO-контент, блоги, социальные сети, email-кампании

Лучшие практики

🎯 Делегирование задач

  • Позвольте Claude Code делегировать автоматически - Основной агент анализирует контекст и выбирает оптимальных агентов
  • Будьте конкретны в требованиях - Указывайте ограничения, стек технологий и требования к качеству
  • Доверьтесь экспертности агентов - Каждый агент оптимизирован для своей области

🔄 Многоагентные рабочие процессы

  • Начинайте с высокоуровневых запросов - Пусть агенты координируют сложные многоступенчатые задачи
  • Передавайте контекст между агентами - Убедитесь, что агенты имеют необходимую информацию
  • Проверяйте точки интеграции - Оценивайте, как результаты разных агентов работают вместе

🎛️ Явный контроль

  • Используйте явный вызов для специфических задач - Когда нужен взгляд конкретного эксперта
  • Стратегически комбинируйте нескольких агентов - Разные специалисты могут проверять работу друг друга
  • Запрашивайте определённые схемы проверки - "Пусть security-auditor проверит дизайн API от backend-architect"

📈 Оптимизация

  • Отслеживайте эффективность агентов - Выясните, какие агенты лучше подходят для ваших задач
  • Итерация сложных задач — используйте обратную связь агента для уточнения требований
  • Используйте сильные стороны агента — сопоставляйте сложность задачи с возможностями агента

Вклад

Чтобы добавить нового субагента:

  • Создайте новый файл .md, следуя приведенному выше формату
  • Используйте имена в нижнем регистре, разделённые дефисами
  • Напишите чёткое описание, когда следует использовать субагента
  • Включите конкретные инструкции в системный промпт

Устранение неполадок

Распространённые проблемы

Агент не вызывается автоматически:

  • Убедитесь, что ваш запрос явно указывает домен (например, "проблема с производительностью" → performance-engineer)
  • Будьте конкретны относительно типа задачи (например, "рецензировать код" → code-reviewer)

Выбран неожиданный агент:
  • Предоставьте больше информации о вашем техническом стеке и требованиях
  • Используйте явное вызовы, если вам нужен конкретный агент

Несколько агентов дают противоречивые советы:
  • Это нормально — разные специалисты могут иметь разные приоритеты
  • Попросите разъяснения: "Согласуйте рекомендации от security-auditor и performance-engineer"

Агенту не хватает контекста:
  • Предоставьте фоновую информацию в запросе
  • Сошлитесь на предыдущие беседы или устоявшиеся шаблоны

Получение помощи

Если агенты работают не так, как ожидалось:

  • Проверьте описания агентов в их отдельных файлах
  • Попробуйте более специфичные формулировки в ваших запросах
  • Используйте явный вызов для проверки конкретных агентов
  • Предоставьте больше информации о вашем проекте и целях
