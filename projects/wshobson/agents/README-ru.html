Коллекция субагентов Claude Code

Обширная коллекция специализированных ИИ-субагентов для Claude Code, разработанная для повышения эффективности процессов разработки за счёт экспертных знаний в определённых областях.

Обзор

Данный репозиторий содержит 57 специализированных субагентов, расширяющих возможности Claude Code. Каждый субагент является экспертом в определённой области, автоматически вызывается в зависимости от контекста или может быть вызван вручную при необходимости. Все агенты настроены на использование конкретных моделей Claude в зависимости от сложности задачи для оптимальной производительности и экономии ресурсов.

Установка

Эти субагенты автоматически доступны при размещении в каталоге ~/.claude/agents/.

cd ~/.claude
git clone https://github.com/wshobson/agents.git

Использование

Автоматический вызов

Claude Code автоматически делегирует задачу соответствующему субагенту в зависимости от контекста задачи и описания субагента.

Явный вызов

Упомяните субагента по имени в вашем запросе: 
"Use the code-reviewer to check my recent changes"
"Have the security-auditor scan for vulnerabilities"
"Get the performance-engineer to optimize this bottleneck"

Формат субагента

Каждый субагент следует этой структуре: 

---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools

System prompt defining the subagent's role and capabilities

Конфигурация модели

Начиная с Claude Code v1.0.64, субагенты могут указывать, какую модель Claude им следует использовать. Это позволяет эффективно делегировать задачи с учетом их сложности и стоимости:

  • Низкая сложность (Haiku): Простые задачи, такие как базовый анализ данных, генерация документации и стандартные ответы
  • Средняя сложность (Sonnet): Задачи разработки, ревью кода, тестирование и обычная инженерная работа
  • Высокая сложность (Opus): Критические задачи, такие как аудит безопасности, ревью архитектуры, реагирование на инциденты и инженерия ИИ/МО

Доступные модели (упрощенные названия с Claude Code v1.0.64):
  • haiku – Быстрая и экономичная для простых задач
  • sonnet – Сбалансированная производительность для большинства задач разработки
  • opus – Наиболее способная для сложного анализа и критических задач

Если модель не указана, субагент использует модель по умолчанию системы.

Шаблоны оркестрации агентов

Claude Code автоматически координирует агентов, используя эти распространенные шаблоны:

Последовательные рабочие процессы

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

Условное ветвление

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

Проверка и подтверждение

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

  • tutorial-engineer: Пошаговые руководства, учебные маршруты, образовательный контент

💼 Бизнес и стратегия

  • business-analyst: KPI, модели доходов, прогнозы роста, метрики для инвесторов
  • risk-manager: Риски портфеля, хеджирующие стратегии, R-множители, размер позиции
  • content-marketer: SEO-контент, блог-посты, социальные сети, email-кампании
  • sales-automator: Холодные письма, фоллоу-апы, предложения, развитие лидов
  • customer-support: Обработка заявок, FAQ, справочная документация, устранение проблем
  • legal-advisor - Составление политик конфиденциальности, условий использования, отказов от ответственности и юридических уведомлений

Лучшие практики

🎯 Делегирование задач

  • Пусть Claude Code делегирует автоматически – главный агент анализирует контекст и выбирает оптимальных агентов
  • Будьте конкретны в требованиях – указывайте ограничения, стек технологий и требования к качеству
  • Доверяйте экспертности агентов – каждый агент оптимизирован под свою область

🔄 Мультиагентские рабочие процессы

  • Начинайте с запросов высокого уровня – пусть агенты координируют сложные многошаговые задачи
  • Передавайте контекст между агентами – обеспечьте агентов необходимой исходной информацией
  • Проверяйте точки интеграции – оценивайте, как результаты разных агентов работают вместе

🎛️ Явный контроль

  • Используйте явное вызовы при специфических нуждах – если нужен взгляд конкретного эксперта
  • Стратегически комбинируйте нескольких агентов – разные специалисты могут проверять работу друг друга
  • Запрашивайте конкретные схемы проверки – «Пусть security-auditor проверит дизайн API от backend-architect»

📈 Оптимизация

  • Отслеживайте эффективность агентов – выясняйте, какие агенты лучше подходят для ваших задач
  • Итерируйте сложные задачи – используйте обратную связь агентов для уточнения требований
  • Используйте сильные стороны агентов – Соотносите сложность задачи с возможностями агента

Вклад

Чтобы добавить нового субагента:

  • Создайте новый .md файл по приведённому выше образцу
  • Используйте имена в нижнем регистре, разделённые дефисами
  • Напишите чёткое описание случаев использования субагента
  • Включите конкретные инструкции в системный промпт

Устранение неполадок

Распространённые проблемы

Агент не вызывается автоматически:

  • Убедитесь, что ваш запрос явно указывает домен (например, "проблема с производительностью" → performance-engineer)
  • Будьте конкретны в типе задачи (например, "провести ревью кода" → code-reviewer)

Неожиданный выбор агента:
  • Предоставьте больше информации о вашем технологическом стеке и требованиях
  • Используйте явный вызов, если нужен конкретный агент

Несколько агентов дают противоречивые рекомендации:
  • Это нормально — разные специалисты могут иметь разные приоритеты
  • Запросите уточнение: "Согласуйте рекомендации от security-auditor и performance-engineer"

Агенту не хватает контекста:
  • Предоставьте фоновую информацию в вашем запросе
  • Сошлитесь на предыдущие обсуждения или установленные шаблоны

Получение помощи

Если агенты работают не так, как ожидалось:

  • Проверьте описания агентов в их индивидуальных файлах
  • Используйте более конкретные формулировки в своих запросах
  • Протестируйте конкретных агентов с помощью явного вызова
  • Предоставьте больше информации о вашем проекте и целях
