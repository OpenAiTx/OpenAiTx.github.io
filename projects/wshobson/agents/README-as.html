মডেল কনফিগাৰেচন

Claude Code v1.0.64-ৰ পৰা, উপ-এজেন্টে নিৰ্দিষ্ট কৰিব পাৰে তেওঁলোকে কোনটো Claude মডেল ব্যৱহাৰ কৰিব। এইটো জটিলতাৰ আধাৰত খৰচ-প্ৰভাৱশালী টাস্ক ডেলেগেশ্যন সম্ভৱ কৰে:

  • নিম্ন জটিলতা (Haiku): সহজ টাস্ক যেনে মৌলিক ডাটা বিশ্লেষণ, ডকুমেন্টেশ্যন প্ৰস্তুতি, আৰু মানক উত্তৰ
  • মধ্যম জটিলতা (Sonnet): ডেভেলপমেন্ট টাস্ক, কোড পৰ্যালোচনা, পৰীক্ষণ, আৰু মানক ইঞ্জিনিয়াৰিং কাম
  • উচ্চ জটিলতা (Opus): গুৰত্বপূৰ্ণ টাস্ক যেনে সুৰক্ষা অডিটিং, স্থাপত্য পৰ্যালোচনা, ঘটনা প্ৰতিক্ৰিয়া, আৰু AI/ML ইঞ্জিনিয়াৰিং

উপলব্ধ মডেলসমূহ (Claude Code v1.0.64-ৰ মতে সহজ নাম ব্যৱহাৰ কৰি):
  • haiku - সহজ টাস্কৰ বাবে দ্ৰুত আৰু খৰচ-প্ৰভাৱশালী
  • sonnet - অধিকাংশ ডেভেলপমেন্ট কামৰ বাবে সু-সমতল কাৰ্যক্ষমতা
  • opus - জটিল বিশ্লেষণ আৰু গুৰত্বপূৰ্ণ টাস্কৰ বাবে আটাইতকৈ সক্ষম

যদি কোনো মডেল নিৰ্দিষ্ট কৰা নহয়, উপ-এজেন্টে ব্যৱস্থাৰ ডিফল্ট মডেল ব্যৱহাৰ কৰিব।

এজেন্ট অৰ্কেষ্ট্ৰেচন পেটাৰ্ন

Claude Code-এ স্বয়ংক্ৰিয়ভাৱে সাধাৰণ পেটাৰ্ন ব্যৱহাৰ কৰি এজেন্টসমূহ সমন্বয় কৰে:

ক্ৰমবদ্ধ ৱৰ্কফ্লো

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

শর্তাধীন শাখাবিভাজন

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

পৰ্যালোচনা আৰু প্ৰমাণীকৰণ

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

  • tutorial-engineer: ধাপে ধাপে টিউটোৰিয়েল, শিকনি পথ, শিক্ষামূলক সামগ্ৰী

💼 ব্যৱসায় & কৌশল

  • business-analyst: KPI, ৰাজহ মডেল, বৃদ্ধিৰ পূৰ্বানুমান, বিনিয়োগকাৰীৰ মেট্ৰিক্স
  • risk-manager: পোৰ্টফ'লিঅ'ৰ পৰিসৰ, হেড্জিং কৌশল, R-মাল্টিপ্ল, স্থিতি মাপ
  • content-marketer: SEO সামগ্ৰী, ব্লগ পোষ্ট, ছচিয়েল মিডিয়া, ইমেইল অভিযান
  • sales-automator: ঠাণ্ডা ইমেইল, অনুসৰণ, প্ৰস্তাৱ, গ্ৰাহক যত্ন
  • customer-support: সহায় টিকেট, FAQ, সহায় দলিল, সমস্যাৰ সমাধান
  • legal-advisor - গোপনীয়তা নীতি, সেৱাৰ চৰ্ত, অস্বীকাৰসূচক, আৰু আইনগত বিজ্ঞপ্তি খচৰা কৰা

উত্তম অনুশীলনসমূহ

🎯 কাৰ্যবণ্টন

  • Claude Code-এ স্বয়ংক্ৰিয়ভাৱে কাৰ্যবণ্টন কৰক - মুখ্য এজেন্টে পৰিপ্ৰেক্ষিত বিশ্লেষণ কৰি উত্তম এজেন্ট বাচে
  • দাবীৰ বিষয়ে স্পষ্ট হওক - সীমাবদ্ধতা, প্ৰযুক্তি স্তাক আৰু গুণগত দাবী অন্তৰ্ভুক্ত কৰক
  • এজেন্টৰ দক্ষতাত বিশ্বাস ৰাখক - প্ৰতিটো এজেন্ট নিজৰ ক্ষেত্ৰত উন্নত

🔄 বহু-এজেন্ট কৰ্মপ্রবাহ

  • উচ্চ-স্তৰৰ অনুৰোধৰে আৰম্ভ কৰক - এজেন্টে জটিল বহু-ধাপীয় কাৰ্য সমন্বয় কৰে
  • এজেন্টসমূহৰ মাজত পৰিপ্ৰেক্ষিত যোগান ধৰক - এজেন্টসমূহক প্ৰয়োজনীয় পটভূমি তথ্য দিয়ক
  • সংহতি বিন্দুসমূহ পৰ্যালোচনা কৰক - ভিন্ন এজেন্টৰ আউটপুট কেনেকৈ একেলগে কাম কৰে পৰীক্ষা কৰক

🎛️ স্পষ্ট নিয়ন্ত্ৰণ

  • বিশেষ প্ৰয়োজনৰ বাবে স্পষ্ট আহ্বান ব্যৱহাৰ কৰক - যেতিয়া আপোনাৰ বিশেষজ্ঞৰ দৃষ্টিভংগীৰ প্ৰয়োজন হয়
  • একাধিক এজেন্ট কৌশলগতভাৱে সংমিশ্ৰণ কৰক - ভিন্ন বিশেষজ্ঞে পৰস্পৰৰ কাম প্ৰমাণিত কৰিব পাৰে
  • নির্দিষ্ট পৰ্যালোচনা প্যাটাৰ্ন অনুৰোধ কৰক - "security-auditor-এ backend-architect-ৰ API ডিজাইন পৰ্যালোচনা কৰক"

📈 অপ্টিমাইজেশ্বন

  • এজেন্টৰ কার্যক্ষমতা পর্যবেক্ষণ কৰক - কোন এজেন্ট আপোনাৰ ব্যৱহাৰ ক্ষেত্ৰত আটাইতকৈ ভাল কাম কৰে জানক
  • জটিল কাৰ্যসমূহত পুনৰাবৃত্তি কৰক - এজেন্টৰ প্ৰতিক্ৰিয়া ব্যৱহাৰ কৰি আৱশ্যকতাসমূহ উন্নত কৰক
  • এজেন্টৰ শক্তি সদ্ব্যৱহাৰ কৰক - কাৰ্যৰ জটিলতা এজেন্টৰ সক্ষমতাৰ সৈতে মিল কৰক

অৱদান

এটা নতুন উপ-এজেন্ট যোগ কৰিবলৈ:

  • ওপৰৰ ফৰ্মেট অনুসৰি এটা নতুন .md ফাইল সৃষ্টি কৰক
  • সৰু আখৰ, হাইফেন-দ্বারা বিভাজিত নাম ব্যৱহাৰ কৰক
  • উপ-এজেন্ট কেতিয়া ব্যৱহাৰ কৰিব লাগে তাৰ বাবে স্পষ্ট বিৱৰণ লিখক
  • চিস্টেম প্ৰম্প্টত নিৰ্দিষ্ট নিৰ্দেশনা অন্তৰ্ভুক্ত কৰক

সাধাৰণ সমস্যা

এজেন্ট স্বয়ংক্ৰিয়ভাৱে আহ্বান নহয়:

  • আপোনাৰ অনুৰোধত স্পষ্টভাৱে ডোমেইন উল্লেখ কৰক (উদাঃ "performance issue" → performance-engineer)
  • কাৰ্যৰ ধৰণ স্পষ্টকৈ উল্লেখ কৰক (উদাঃ "review code" → code-reviewer)

অপ্ৰত্যাশিত এজেন্ট নিৰ্বাচন:
  • আপোনাৰ টেক ষ্টেক আৰু আৱশ্যকতাসমূহ সম্পৰ্কে অধিক পৰিপ্ৰেক্ষিত দিয়ক
  • নিৰ্দিষ্ট এজেন্টৰ বাবে স্পষ্ট আহ্বান ব্যৱহাৰ কৰক

একাধিক এজেন্টে বিভ্ৰান্তিকৰ পৰামৰ্শ প্ৰদান কৰিছে:
  • এইটো স্বাভাৱিক - বিভিন্ন বিশেষজ্ঞৰ ভিন্ন প্ৰাধান্য থাকিব পাৰে
  • স্পষ্টীকৰণ বিচাৰক: "security-auditor আৰু performance-engineer ৰ পৰামৰ্শ মিলাওক"

এজেন্টে পৰিপ্ৰেক্ষিতৰ অভাৱ দেখুৱাইছে:
  • আপোনাৰ অনুৰোধত পটভূমি তথ্য দিয়ক
  • পূৰ্বৰ কথোপকথন বা স্থাপিত পেটাৰ্ন উল্লেখ কৰক

সহায়তা লাভ কৰা

  • পৃথক ফাইলত এজেন্টৰ বিৱৰণ চাওক
  • আপোনাৰ অনুৰোধত অধিক নিৰ্দিষ্ট ভাষা ব্যৱহাৰ কৰক
  • নিৰ্দিষ্ট এজেন্ট পৰীক্ষা কৰিবলৈ স্পষ্ট আহ্বান ব্যৱহাৰ কৰক
  • আপোনাৰ প্ৰকল্প আৰু লক্ষ্যসমূহৰ বিষয়ে অধিক পৰিপ্ৰেক্ষিত দিয়ক
