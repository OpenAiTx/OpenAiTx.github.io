মডেল কনফিগাৰেচন

Claude Code v1.0.64-ৰ পৰা, উপ-এজেন্টে নিৰ্দিষ্ট কৰিব পাৰে তেওঁলোকে কোনটো Claude মডেল ব্যৱহাৰ কৰিব। এইটো জটিলতাৰ আধাৰত খৰচ-প্ৰভাৱশালী টাস্ক ডেলেগেশ্যন সম্ভৱ কৰে:

  • নিম্ন জটিলতা (Haiku): সহজ টাস্ক যেনে মৌলিক ডাটা বিশ্লেষণ, ডকুমেন্টেশ্যন প্ৰস্তুতি, আৰু মানক উত্তৰ
  • মধ্যম জটিলতা (Sonnet): ডেভেলপমেন্ট টাস্ক, কোড পৰ্যালোচনা, পৰীক্ষণ, আৰু মানক ইঞ্জিনিয়াৰিং কাম
  • উচ্চ জটিলতা (Opus): গুৰত্বপূৰ্ণ টাস্ক যেনে সুৰক্ষা অডিটিং, স্থাপত্য পৰ্যালোচনা, ঘটনা প্ৰতিক্ৰিয়া, আৰু AI/ML ইঞ্জিনিয়াৰিং

উপলব্ধ মডেলসমূহ (Claude Code v1.0.64-ৰ মতে সহজ নাম ব্যৱহাৰ কৰি):
  • haiku - সহজ টাস্কৰ বাবে দ্ৰুত আৰু খৰচ-প্ৰভাৱশালী
  • sonnet - অধিকাংশ ডেভেলপমেন্ট কামৰ বাবে সু-সমতল কাৰ্যক্ষমতা
  • opus - জটিল বিশ্লেষণ আৰু গুৰত্বপূৰ্ণ টাস্কৰ বাবে আটাইতকৈ সক্ষম

যদি কোনো মডেল নিৰ্দিষ্ট কৰা নহয়, উপ-এজেন্টে ব্যৱস্থাৰ ডিফল্ট মডেল ব্যৱহাৰ কৰিব।

এজেন্ট অৰ্কেষ্ট্ৰেচন পেটাৰ্ন

Claude Code-এ স্বয়ংক্ৰিয়ভাৱে সাধাৰণ পেটাৰ্ন ব্যৱহাৰ কৰি এজেন্টসমূহ সমন্বয় কৰে:

ক্ৰমবদ্ধ ৱৰ্কফ্লো

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

শর্তাধীন শাখাবিভাজন

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

পৰ্যালোচনা আৰু প্ৰমাণীকৰণ

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

  • security-auditor: ভল্নাৰেবিলিটি স্কেনিং, কম্প্লায়েন্স চেক
  • data-scientist: ডেটা বিশ্লেষণ, ইনছাইট, ৰিপৰ্টিং

  • mlops-engineer: এমএল ইনফ্ৰাষ্ট্ৰাকচাৰ, এক্সপেৰিমেন্ট ট্ৰেকিং, মডেল ৰেজিষ্ট্ৰী, পাইপলাইন স্বয়ংক্ৰীয়কৰণ

🧪 গুণগত নিশ্চয়তা

  • code-reviewer: ক'ডৰ গুণ, কনফিগাৰেচন নিৰাপত্তা, প্ৰডাকশ্যন বিশ্বাসযোগ্যতা
  • test-automator: টেষ্ট কৌশল, টেষ্ট ছুট সৃষ্টি
  • debugger: বাগ অনুসন্ধান, ত্ৰুটি সমাধান
  • error-detective: লগ বিশ্লেষণ, ত্ৰুটি পেটাৰ্ন চিনাক্তকৰণ, মূল কাৰণ বিশ্লেষণ
  • search-specialist: গভীৰ ৱেব গৱেষণা, প্ৰতিযোগিতামূলক বিশ্লেষণ, তথ্য যাচাই

📚 নথিপত্ৰ

  • api-documenter: OpenAPI/Swagger স্পেচ, API নথিপত্ৰ
  • docs-architect: বিস্তৃত প্ৰযুক্তিগত নথিপত্ৰ, স্থাপত্য গাইড, চিষ্টেম ম্যানুৱেল
  • reference-builder: বিস্তৃত API ৰেফাৰেঞ্চ, কনফিগাৰেচন গাইড, পেৰামিটাৰ নথিপত্ৰ
  • tutorial-engineer: ধাপ-ধাপে টিউটোৰিয়েল, শিক্ষাৰ পথ, শিক্ষামূলক সামগ্ৰী

💼 ব্যৱসায় আৰু কৌশল

  • business-analyst: KPI, ৰেভিনিউ মডেল, বৃদ্ধিৰ পূৰ্বানুমান, বিনিয়োগকাৰী মেট্ৰিক্স
  • risk-manager: পোৰ্টফলিঅ' ঝুঁকি, হেজিং কৌশল, R-মাল্টিপল, অৱস্থান মাপ
  • content-marketer: SEO সামগ্ৰী, ব্লগ পোষ্ট, ছচিয়েল মিডিয়া, ইমেইল অভিযান
  • sales-automator: ঠাণ্ডা ইমেইল, অনুসৰণ, প্ৰস্তাৱ, লীড পোষণ
  • customer-support: সহায় টিকিট, FAQs, সহায় নথিপত্ৰ, ত্ৰুটিমুক্তকৰণ
  • legal-advisor - গোপনীয়তা নীতি, সেৱাৰ চৰ্ত, অস্বীকৃতি, আৰু আইনি নোটিশ খচৰা কৰক

শ্ৰেষ্ঠ অনুশীলন

🎯 কৰ্ম নিৰ্দেশনা

  • ক্লড ক'ডক স্বয়ংক্ৰিয়ভাৱে কৰ্ম নিৰ্দেশনা দিয়ক - মুখ্য এজেন্টে প্ৰসংগ বিশ্লেষণ কৰে আৰু উৎকৃষ্ট এজেন্ট বাছি লয়
  • আৱশ্যকতাসমূহ স্পষ্টকৈ উল্লেখ কৰক - সীমাবদ্ধতা, প্ৰযুক্তি স্তূপ, আৰু গুণগত মানৰ আৱশ্যকতা অন্তর্ভুক্ত কৰক
  • এজেন্টৰ দক্ষতাত বিশ্বাস ৰাখক - প্ৰতিটো এজেন্ট তেওঁলোকৰ ক্ষেত্ৰত অনুকূলিত

🔄 বহু-এজেন্ট কার্যপদ্ধতি

  • উচ্চ-স্তৰৰ অনুৰোধৰে আৰম্ভ কৰক - এজেন্টে জটিল বহু-ধাপীয় কৰ্ম সমন্বয় কৰক
  • এজেন্টৰ মাজত প্ৰসংগ দিয়ক - এজেন্টে আৱশ্যকীয় পটভূমি তথ্য পায় নিশ্চিত কৰক
  • সমন্বয় বিন্দু পৰ্যালোচনা কৰক - বিভিন্ন এজেন্টৰ আউটপুট কিদৰে একেলগে কাম কৰে পৰীক্ষা কৰক

🎛️ স্পষ্ট নিয়ন্ত্ৰণ

  • বিশেষ প্ৰয়োজনত স্পষ্টভাৱে আহ্বান ব্যৱহাৰ কৰক - যেতিয়া আপুনি কোনো বিশেষ বিশেষজ্ঞৰ দৃষ্টিভংগী বিচাৰে
  • একাধিক এজেন্ট কৌশলগতভাৱে সংমিশ্ৰণ কৰক - বিভিন্ন বিশেষজ্ঞে ইজনে সিজনৰ কাম যাচাই কৰিব পাৰে
  • বিশেষ পৰ্যালোচনা পেটাৰ্ন অনুৰোধ কৰক - "নিরাপত্তা-অডিটৰে বেকএণ্ড-আর্কিটেক্টৰ API ডিজাইন পৰ্যালোচনা কৰক"

📈 অপ্টিমাইজেচন

  • এজেন্টৰ কার্যক্ষমতা মনিটৰ কৰক - আপোনাৰ ব্যৱহাৰ ক্ষেত্ৰসমূহৰ বাবে কোন এজেন্ট ভাল কাম কৰে শিকক
  • জটিল কাৰ্যসমূহত পুনৰাবৃত্তি কৰক - এজেন্টৰ ফিডব্যাক ব্যৱহাৰ কৰি দাবীসমূহ উন্নত কৰক
  • এজেন্টৰ শক্তিসমূহ সদ্ব্যৱহাৰ কৰক - কাৰ্যৰ জটিলতা এজেন্টৰ সক্ষমতাৰ সৈতে মিলাওক

অৱদান

এটা নতুন সাবএজেন্ট যোগ কৰিবলৈ:

  • ওপৰৰ ফৰ্মেট অনুসৰি এটা নতুন .md ফাইল সৃষ্টি কৰক
  • সৰু আখৰ আৰু হাইফেন-দ্বারা বিভাজিত নাম ব্যৱহাৰ কৰক
  • সাবএজেন্ট কেতিয়া ব্যৱহাৰ কৰিব লাগে তাৰ স্পষ্ট বৰ্ণনা লিখক
  • চিস্টেম প্ৰম্পটত নিৰ্দিষ্ট নিৰ্দেশনা অন্তর্ভুক্ত কৰক

সমস্যাৰ সমাধান

সাধাৰণ সমস্যা

এজেন্ট স্বচালিতভাৱে আহ্বান নহয়:

  • আপোনাৰ অনুৰোধত স্পষ্টকৈ ডোমেইন উল্লেখ কৰক (যেনে, "performance issue" → performance-engineer)
  • কাৰ্যৰ ধৰণ স্পষ্টকৈ উল্লেখ কৰক (যেনে, "review code" → code-reviewer)

অপ্ৰত্যাশিত এজেন্ট নিৰ্বাচন:
  • আপোনাৰ টেক ষ্টেক আৰু দাবীসমূহৰ বিষয়ে অধিক প্ৰসংগ দিয়ক
  • যদি আপোনাৰ বিশেষ এজেন্টৰ প্ৰয়োজন, স্পষ্ট আহ্বান ব্যৱহাৰ কৰক

একাধিক এজেন্টে বিপৰীত পৰামর্শ প্ৰদান কৰে:
  • এইটো স্বাভাবিক - বিভিন্ন বিশেষজ্ঞৰ ভিন্ন প্ৰাধান্য থাকিব পাৰে
  • স্পষ্টীকৰণ বিচাৰক: "security-auditor আৰু performance-engineer ৰ পৰামর্শ মিলাওক"

এজেন্টে প্ৰসংগৰ অভাৱ অনুভৱ কৰে:
  • আপোনাৰ অনুৰোধত পটভূমি তথ্য দিয়ক
  • পূৰ্বৰ আলোচনাবোৰ বা প্ৰতিষ্ঠিত ধাৰণাবোৰ উল্লেখ কৰক

সহায়তা পাবলৈ

যদি এজেন্টবোৰ আশা অনুসৰি কাম নকৰে:

  • তেওঁলোকৰ নিজ নিজ ফাইলত এজেন্টৰ বৰ্ণনা চাওক
  • আপোনাৰ অনুৰোধত অধিক নিৰ্দিষ্ট ভাষা চেষ্টা কৰক
  • নিৰ্দিষ্ট এজেন্ট পৰীক্ষা কৰিবলৈ স্পষ্ট আহ্বান ব্যৱহাৰ কৰক
  • আপোনাৰ প্ৰকল্প আৰু লক্ষ্যসমূহৰ বিষয়ে অধিক প্ৰসংগ দিয়ক
