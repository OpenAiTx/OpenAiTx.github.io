সামৰি

এই ৰিপ'জিট'ৰিটোত ৫৭ টা বিশেষ চাব-এজেন্ট আছে যি Claude Code-ৰ সক্ষমতা বৃদ্ধি কৰে। প্ৰতিটো চাব-এজেন্ট এটা নিৰ্দিষ্ট ক্ষেত্ৰৰ বিশেষজ্ঞ, প্ৰাসংগিক অৱস্থাত স্বয়ংক্ৰিয়ভাৱে আহ্বান কৰা হয় বা প্ৰয়োজনত স্পষ্টকৈ মাতি ওলাই আনে। সকলো এজেন্ট নির্দিষ্ট Claude মডেলৰ সৈতে সংৰূপিত, কাৰ্যৰ জটিলতাৰ ওপৰত আধাৰিত, যাতে উৎকৃষ্ট কার্যক্ষমতা আৰু ব্যয়-দক্ষতা নিশ্চিত হয়।

মডেল কনফিগাৰেচন

Claude Code v1.0.64-ৰ পৰা, উপ-এজেন্টে নিৰ্দিষ্ট কৰিব পাৰে তেওঁলোকে কোনটো Claude মডেল ব্যৱহাৰ কৰিব। এইটো জটিলতাৰ আধাৰত খৰচ-প্ৰভাৱশালী টাস্ক ডেলেগেশ্যন সম্ভৱ কৰে:

  • নিম্ন জটিলতা (Haiku): সহজ টাস্ক যেনে মৌলিক ডাটা বিশ্লেষণ, ডকুমেন্টেশ্যন প্ৰস্তুতি, আৰু মানক উত্তৰ
  • মধ্যম জটিলতা (Sonnet): ডেভেলপমেন্ট টাস্ক, কোড পৰ্যালোচনা, পৰীক্ষণ, আৰু মানক ইঞ্জিনিয়াৰিং কাম
  • উচ্চ জটিলতা (Opus): গুৰত্বপূৰ্ণ টাস্ক যেনে সুৰক্ষা অডিটিং, স্থাপত্য পৰ্যালোচনা, ঘটনা প্ৰতিক্ৰিয়া, আৰু AI/ML ইঞ্জিনিয়াৰিং

উপলব্ধ মডেলসমূহ (Claude Code v1.0.64-ৰ মতে সহজ নাম ব্যৱহাৰ কৰি):
  • haiku - সহজ টাস্কৰ বাবে দ্ৰুত আৰু খৰচ-প্ৰভাৱশালী
  • sonnet - অধিকাংশ ডেভেলপমেন্ট কামৰ বাবে সু-সমতল কাৰ্যক্ষমতা
  • opus - জটিল বিশ্লেষণ আৰু গুৰত্বপূৰ্ণ টাস্কৰ বাবে আটাইতকৈ সক্ষম

যদি কোনো মডেল নিৰ্দিষ্ট কৰা নহয়, উপ-এজেন্টে ব্যৱস্থাৰ ডিফল্ট মডেল ব্যৱহাৰ কৰিব।

এজেন্ট অৰ্কেষ্ট্ৰেচন পেটাৰ্ন

Claude Code-এ স্বয়ংক্ৰিয়ভাৱে সাধাৰণ পেটাৰ্ন ব্যৱহাৰ কৰি এজেন্টসমূহ সমন্বয় কৰে:

ক্ৰমবদ্ধ ৱৰ্কফ্লো

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

শর্তাধীন শাখাবিভাজন

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

পৰ্যালোচনা আৰু প্ৰমাণীকৰণ

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

  • কাষ্টমাৰ-ছাপৰ্ট: ছাপৰ্ট টিকেট, FAQ, সহায় ডকুমেন্টেচন, সমস্যা সমাধান

শ্ৰেষ্ঠ অভ্যাস

🎯 কাৰ্য বিভাজন

  • Claude Code-এ স্বয়ংক্ৰিয়ভাৱে দায়িত্ব প্ৰদান কৰক - মূল এজেন্টে প্ৰসঙ্গ বিশ্লেষণ কৰি উৎকৃষ্ট এজেন্ট নিৰ্বাচন কৰে
  • আপোনাৰ প্ৰয়োজনসমূহ স্পষ্টভাৱে উল্লেখ কৰক - সীমাবদ্ধতা, প্ৰযুক্তি স্তাক আৰু গুণগত প্ৰয়োজন উল্লেখ কৰক
  • এজেন্টৰ দক্ষতাত বিশ্বাস কৰক - প্ৰতিটো এজেন্ট তেওঁলোকৰ ক্ষেত্ৰত অনুকূলিত

🔄 বহু-এজেন্ট ৱাৰ্কফ্ল'

  • উচ্চ-স্তৰৰ অনুৰোধৰ পৰা আৰম্ভ কৰক - এজেন্টে জটিল বহু-ধাপীয়া কাৰ্য সমন্বয় কৰে
  • এজেন্টসমূহৰ মাজত প্ৰসঙ্গ প্রদান কৰক - এজেন্টসমূহক প্ৰয়োজনীয় পটভূমি তথ্য নিশ্চিত কৰক
  • ইন্টেগ্ৰেশ্যন পইণ্টসমূহ পৰ্যালোচনা কৰক - বিভিন্ন এজেন্টৰ উত্পাদন একেলগে কেনেকৈ কাম কৰে পৰীক্ষা কৰক

🎛️ স্পষ্ট নিয়ন্ত্ৰণ

  • বিশেষ প্ৰয়োজনত স্পষ্টভাৱে আহ্বান কৰক - যেতিয়া আপোনাৰ বিশেষজ্ঞৰ দৃষ্টিভংগীৰ প্ৰয়োজন
  • বহু এজেন্ট কৌশলগতভাৱে সংমিশ্ৰণ কৰক - বিভিন্ন বিশেষজ্ঞে পৰস্পৰৰ কাম যাচাই কৰিব পাৰে
  • নির্দিষ্ট পৰ্যালোচনাৰ অনুৰোধ কৰক - "security-auditor-এ backend-architect-ৰ API ডিজাইন পৰ্যালোচনা কৰক"

📈 অপ্টিমাইজেচন

  • এজেন্টৰ কার্যক্ষমতা মনিটৰ কৰক - কোন এজেন্ট আপোনাৰ ব্যৱহাৰৰ ক্ষেত্ৰত শ্ৰেষ্ঠ জানক
  • জটিল কাৰ্যসমূহ পুনৰাবৃত্তি কৰক - এজেন্টৰ প্ৰতিক্ৰিয়াৰ দ্বাৰা প্ৰয়োজনসমূহ উন্নত কৰক
  • এজেন্টৰ শক্তি ব্যৱহাৰ কৰক - কামৰ জটিলতা এজেন্টৰ সক্ষমতাৰ সৈতে মিলাওক

  • সৰু অক্ষৰ, হাইফেন-দ্বাৰা বিভক্ত নাম ব্যৱহাৰ কৰক
  • কেতিয়া ছাব-এজেন্ট ব্যৱহাৰ কৰিব লাগে তাৰ বাবে স্পষ্ট বিৱৰণ লিখক
  • ছিষ্টেম প্ৰম্প্টত নিৰ্দিষ্ট নিৰ্দেশনা অন্তর্ভুক্ত কৰক

  • আপোনাৰ অনুৰোধত স্পষ্টকৈ ড’মেইন উল্লেখ কৰক (যেনে, "performance issue" → performance-engineer)
  • কামৰ প্ৰকাৰ স্পষ্টকৈ উল্লেখ কৰক (যেনে, "review code" → code-reviewer)

  • আপোনাৰ টেক ষ্টেক আৰু আৱশ্যকতাৰ বিষয়ে অধিক পৰিপ্ৰেক্ষিত দিয়ক
  • নিৰ্দিষ্ট এজেন্ট প্ৰয়োজন হ’লে স্পষ্ট আহ্বান ব্যৱহাৰ কৰক

  • স্পষ্টীকৰণ বিচাৰক: "security-auditor আৰু performance-engineer ৰ পৰামৰ্শ মিলাওক"

এজেন্টে যেনে পৰিপ্ৰেক্ষিত নাপায়:
  • আপোনাৰ অনুৰোধত পটভূমি তথ্য দিয়ক
  • পূৰ্বৰ কথোপকথন বা স্থাপিত পেটাৰ্ন উল্লেখ কৰক
