Raccolta di Subagenti Claude Code

Una raccolta completa di subagenti AI specializzati per Claude Code, progettata per migliorare i flussi di lavoro di sviluppo con competenze specifiche di dominio.

Panoramica

Questo repository contiene 57 subagenti specializzati che estendono le capacità di Claude Code. Ogni subagente è un esperto in un dominio specifico, richiamato automaticamente in base al contesto o esplicitamente quando necessario. Tutti gli agenti sono configurati con modelli Claude specifici in base alla complessità del compito per ottenere prestazioni ottimali ed efficienza nei costi.

Installazione

Questi subagenti sono automaticamente disponibili se inseriti nella directory ~/.claude/agents/.

cd ~/.claude
git clone https://github.com/wshobson/agents.git

Utilizzo

Invocazione Automatica

Claude Code delega automaticamente al sottoagente appropriato in base al contesto del compito e alla descrizione del sottoagente.

Invocazione Esplicita

Menziona il sottoagente per nome nella tua richiesta: 
"Use the code-reviewer to check my recent changes"
"Have the security-auditor scan for vulnerabilities"
"Get the performance-engineer to optimize this bottleneck"

Formato Subagente

Ogni subagente segue questa struttura: 

---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools

System prompt defining the subagent's role and capabilities

Configurazione del Modello

A partire da Claude Code v1.0.64, i subagent possono specificare quale modello Claude utilizzare. Questo consente una delega dei compiti conveniente in base alla complessità:

  • Bassa Complessità (Haiku): Attività semplici come analisi dati di base, generazione di documentazione e risposte standard
  • Media Complessità (Sonnet): Attività di sviluppo, revisione del codice, test e lavoro ingegneristico standard
  • Alta Complessità (Opus): Attività critiche come auditing di sicurezza, revisione dell’architettura, risposta agli incidenti e ingegneria AI/ML

Modelli disponibili (nomenclatura semplificata con Claude Code v1.0.64):
  • haiku - Rapido ed economico per attività semplici
  • sonnet - Prestazioni bilanciate per la maggior parte dei lavori di sviluppo
  • opus - Il più capace per analisi complesse e compiti critici

Se non viene specificato alcun modello, il subagent utilizzerà il modello predefinito del sistema.

Pattern di Orchestrazione degli Agenti

Claude Code coordina automaticamente gli agent utilizzando questi pattern comuni:

Flussi di lavoro sequenziali

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Esecuzione parallela

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

Diramazione Condizionale

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

Revisione e Validazione

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

Best Practice

🎯 Delegazione dei Compiti

  • Lascia che Claude Code deleghi automaticamente - L'agente principale analizza il contesto e seleziona gli agenti ottimali
  • Sii specifico sui requisiti - Includi vincoli, stack tecnologici e requisiti di qualità
  • Fidati dell'esperienza degli agenti - Ogni agente è ottimizzato per il proprio dominio

🔄 Workflow Multi-Agente

  • Inizia con richieste ad alto livello - Lascia che gli agenti coordinino compiti complessi e multi-step
  • Fornisci contesto tra gli agenti - Assicurati che gli agenti abbiano le informazioni di base necessarie
  • Revisiona i punti di integrazione - Controlla come si combinano i risultati dei diversi agenti

🎛️ Controllo Esplicito

  • Usa invocazione esplicita per esigenze specifiche - Quando desideri il punto di vista di un esperto particolare
  • Combina più agenti strategicamente - Diversi specialisti possono validare il lavoro reciproco
  • Richiedi schemi di revisione specifici - "Fai revisionare il design API dell'architetto backend dal security-auditor"

📈 Ottimizzazione

  • Monitora l'efficacia degli agenti - Scopri quali agenti funzionano meglio per i tuoi casi d'uso
  • Itera su compiti complessi - Usa il feedback degli agenti per affinare i requisiti
  • Sfrutta i punti di forza degli agenti - Abbina la complessità del compito alle capacità dell'agente

Risoluzione dei problemi

Problemi comuni

L’agente non viene invocato automaticamente:

  • Assicurati che la tua richiesta indichi chiaramente il dominio (es. "problema di prestazioni" → performance-engineer)
  • Sii specifico riguardo al tipo di compito (es. "revisiona il codice" → code-reviewer)

Selezione imprevista dell’agente:
  • Fornisci più contesto sul tuo stack tecnologico e sulle tue esigenze
  • Usa l’invocazione esplicita se hai bisogno di un agente specifico

Più agenti producono consigli contrastanti:
  • È normale: diversi specialisti possono avere priorità differenti
  • Chiedi chiarimenti: "Riconcilia le raccomandazioni di security-auditor e performance-engineer"

L’agente sembra non avere contesto:
  • Fornisci informazioni di background nella tua richiesta
  • Fai riferimento a conversazioni precedenti o a schemi consolidati

Ottenere aiuto

Se gli agenti non funzionano come previsto:

  • Controlla le descrizioni degli agenti nei loro file individuali
  • Prova a usare un linguaggio più specifico nelle tue richieste
  • Usa l’invocazione esplicita per testare agenti specifici
  • Fornisci più contesto sul tuo progetto e sui tuoi obiettivi
