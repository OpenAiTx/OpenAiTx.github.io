Coleção de Subagentes Claude Code

Uma coleção abrangente de subagentes de IA especializados para o Claude Code, projetados para aprimorar fluxos de trabalho de desenvolvimento com conhecimento específico de domínio.

Visão Geral

Este repositório contém 61 subagentes especializados que expandem as capacidades do Claude Code. Cada subagente é especialista em um domínio específico, sendo automaticamente invocado conforme o contexto ou chamado explicitamente quando necessário. Todos os agentes são configurados com modelos Claude específicos, baseando-se na complexidade da tarefa, para desempenho ideal e eficiência de custos.

Instalação

Esses subagentes ficam automaticamente disponíveis quando colocados no diretório ~/.claude/agents/.

cd ~/.claude
git clone https://github.com/wshobson/agents.git

Uso

Invocação Automática

O Claude Code irá delegar automaticamente para o subagente apropriado com base no contexto da tarefa e na descrição do subagente.

Invocação Explícita

Mencione o subagente pelo nome em sua solicitação: 
"Use the code-reviewer to check my recent changes"
"Have the security-auditor scan for vulnerabilities"
"Get the performance-engineer to optimize this bottleneck"

Formato do Subagente

Cada subagente segue esta estrutura: 

---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools

System prompt defining the subagent's role and capabilities

Configuração do Modelo

A partir do Claude Code v1.0.64, subagentes podem especificar qual modelo Claude devem utilizar. Isso permite uma delegação de tarefas mais econômica baseada na complexidade:

  • Baixa Complexidade (Haiku): Tarefas simples como análise básica de dados, geração de documentação e respostas padrão
  • Média Complexidade (Sonnet): Tarefas de desenvolvimento, revisão de código, testes e trabalhos de engenharia padrão
  • Alta Complexidade (Opus): Tarefas críticas como auditoria de segurança, revisão de arquitetura, resposta a incidentes e engenharia de IA/ML

Modelos disponíveis (usando nomenclatura simplificada conforme Claude Code v1.0.64):
  • haiku - Rápido e econômico para tarefas simples
  • sonnet - Desempenho equilibrado para a maioria dos trabalhos de desenvolvimento
  • opus - Mais capaz para análises complexas e tarefas críticas

Se nenhum modelo for especificado, o subagente usará o modelo padrão do sistema.

Padrões de Orquestração de Agentes

O Claude Code coordena automaticamente agentes usando estes padrões comuns:

Fluxos de Trabalho Sequenciais

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

Ramificação Condicional

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

Revisão & Validação

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

  • Itere em tarefas complexas - Use o feedback dos agentes para refinar os requisitos

  • Inclua instruções específicas no prompt do sistema

Solução de Problemas

Problemas Comuns

Agente não sendo invocado automaticamente:

  • Certifique-se de que sua solicitação indica claramente o domínio (ex: "problema de desempenho" → performance-engineer)
  • Seja específico sobre o tipo de tarefa (ex: "revisar código" → code-reviewer)

Seleção inesperada de agente:
  • Forneça mais contexto sobre sua stack tecnológica e requisitos
  • Use invocação explícita se precisar de um agente específico

Vários agentes produzindo conselhos conflitantes:
  • Isso é normal - diferentes especialistas podem ter prioridades diferentes
  • Peça esclarecimento: "Reconciliar as recomendações do security-auditor e do performance-engineer"

Agente parece não ter contexto:
  • Forneça informações de fundo em sua solicitação
  • Faça referência a conversas anteriores ou padrões estabelecidos

Obtendo Ajuda

Se os agentes não estiverem funcionando como esperado:

  • Verifique as descrições dos agentes em seus arquivos individuais
  • Tente usar linguagem mais específica em suas solicitações
  • Use invocação explícita para testar agentes específicos
  • Forneça mais contexto sobre seu projeto e objetivos
