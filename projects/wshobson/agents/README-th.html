ชุดรวม Claude Code Subagents

ชุดรวมซับเอเจนต์ AI เฉพาะทางสำหรับ Claude Code ที่ออกแบบมาเพื่อเสริมประสิทธิภาพการพัฒนาด้วยความเชี่ยวชาญเฉพาะด้าน

ภาพรวม

รีโพซิทอรีนี้ประกอบด้วยซับเอเจนต์เฉพาะทาง 76 ตัว ที่ช่วยขยายขีดความสามารถของ Claude Code ซับเอเจนต์แต่ละตัวเป็นผู้เชี่ยวชาญในแต่ละสาขา ถูกเรียกใช้อัตโนมัติตามบริบท หรือเรียกใช้โดยตรงเมื่อจำเป็น ทุกเอเจนต์ถูกตั้งค่าด้วยโมเดล Claude ที่เหมาะสมกับความซับซ้อนของงาน เพื่อประสิทธิภาพและความคุ้มค่าที่ดีที่สุด

ซับเอเจนต์ที่มีให้บริการ

การพัฒนาและสถาปัตยกรรม

  • backend-architect - ออกแบบ RESTful APIs, ขอบเขตไมโครเซอร์วิส และสคีมาฐานข้อมูล
  • frontend-developer - สร้างคอมโพเนนต์ React, จัดเลย์เอาต์ให้ตอบสนอง และจัดการสถานะฝั่งไคลเอนต์
  • ui-ux-designer - สร้างดีไซน์อินเทอร์เฟซ, วาด wireframe และออกแบบระบบดีไซน์
  • ui-visual-validator - ตรวจสอบว่าการปรับแต่ง UI ได้ผลลัพธ์ตามเป้าหมายด้วยการวิเคราะห์ภาพหน้าจออย่างละเอียด
  • mobile-developer - พัฒนาแอป React Native หรือ Flutter พร้อมการผสานกับระบบพื้นฐาน
  • graphql-architect - ออกแบบ GraphQL schema, resolver และ federation
  • architect-reviewer - ตรวจสอบการเปลี่ยนแปลงโค้ดให้สอดคล้องกับหลักสถาปัตยกรรมและรูปแบบมาตรฐาน

  • ruby-pro - เขียนโค้ด Ruby ตามหลัก idiomatic ด้วย metaprogramming, รูปแบบ Rails, การพัฒนา gem และเฟรมเวิร์คทดสอบ
  • golang-pro - เขียนโค้ด Go ตามหลัก idiomatic พร้อม goroutine, channel และ interface
  • rust-pro - เขียนโค้ด Rust ตามหลัก idiomatic ด้วยรูปแบบ ownership, lifetime และการใช้งาน trait
  • c-pro - เขียนโค้ด C ที่มีประสิทธิภาพพร้อมการจัดการหน่วยความจำและ system call อย่างถูกต้อง
  • cpp-pro - เขียนโค้ด C++ ตามหลัก idiomatic ด้วยฟีเจอร์สมัยใหม่, RAII, smart pointer และอัลกอริทึม STL
  • javascript-pro - เชี่ยวชาญ JavaScript สมัยใหม่ด้วย ES6+, รูปแบบ async และ API ของ Node.js
  • typescript-pro - เชี่ยวชาญ TypeScript ด้วยประเภทขั้นสูง, generic และความปลอดภัยของชนิดข้อมูลแบบเข้มงวด
  • php-pro - เขียนโค้ด PHP ตามหลัก idiomatic พร้อมฟีเจอร์สมัยใหม่และการปรับแต่งประสิทธิภาพ
  • java-pro - เชี่ยวชาญ Java สมัยใหม่ด้วย stream, การประมวลผลพร้อมกัน และการปรับแต่ง JVM
  • elixir-pro - เขียนโค้ด Elixir ตามหลัก idiomatic ด้วยรูปแบบ OTP, การเขียนโปรแกรมเชิงฟังก์ชัน และเฟรมเวิร์ค Phoenix
  • csharp-pro - เขียนโค้ด C# สมัยใหม่พร้อมฟีเจอร์ขั้นสูงและการปรับแต่ง .NET
  • scala-pro - เชี่ยวชาญการพัฒนา Scala ระดับองค์กรด้วยการเขียนโปรแกรมเชิงฟังก์ชัน, ระบบกระจาย และการประมวลผลข้อมูลขนาดใหญ่
  • flutter-expert - เชี่ยวชาญการพัฒนา Flutter ด้วย Dart, widget และการผสานกับแพลตฟอร์มต่าง ๆ
  • unity-developer - สร้างเกม Unity พร้อมสคริปต์ที่ปรับแต่งประสิทธิภาพ
  • minecraft-bukkit-pro - เชี่ยวชาญการพัฒนา Minecraft server plugin ด้วย Bukkit, Spigot และ Paper API
  • ios-developer - พัฒนาแอป iOS แบบ native ด้วย Swift/SwiftUI
  • sql-pro - เขียนคำสั่ง SQL ที่ซับซ้อน, ปรับแต่งแผนการดำเนินการ และออกแบบ schema ที่ normalized

โครงสร้างพื้นฐานและปฏิบัติการ

  • devops-troubleshooter - แก้ไขปัญหาในระบบจริง, วิเคราะห์ log และแก้ไขข้อผิดพลาดในการ deploy
  • deployment-engineer - กำหนดค่า CI/CD pipeline, Docker container และการ deploy ขึ้นคลาวด์
  • cloud-architect - ออกแบบโครงสร้าง AWS/Azure/GCP และปรับแต่งต้นทุนคลาวด์
  • hybrid-cloud-architect - ออกแบบโครงสร้างระบบคลาวด์แบบ hybrid ทั้ง AWS/Azure/GCP และ OpenStack ในองค์กร
  • kubernetes-architect - ออกแบบโครงสร้างพื้นฐานแบบ cloud-native ด้วย Kubernetes และหลัก GitOps บน AWS/Azure/GCP และระบบ hybrid
  • database-optimizer - ปรับแต่งคำสั่ง SQL, ออกแบบ index ที่มีประสิทธิภาพ และจัดการ migration ฐานข้อมูล
  • database-admin - ดูแลการทำงานของฐานข้อมูล, การสำรองข้อมูล, การทำ replication และการมอนิเตอร์
  • terraform-specialist - เขียนโมดูล Terraform ขั้นสูง, จัดการไฟล์สถานะ และดำเนินตามแนวทาง IaC ที่ดีที่สุด
  • incident-responder - รับมือกับเหตุการณ์ในระบบจริงอย่างเร่งด่วนและแม่นยำ
  • network-engineer - แก้ไขปัญหาการเชื่อมต่อเครือข่าย, กำหนดค่าตัวกระจายโหลด, และวิเคราะห์รูปแบบการรับส่งข้อมูล
  • dx-optimizer - ผู้เชี่ยวชาญด้านประสบการณ์นักพัฒนา ปรับปรุงเครื่องมือ การติดตั้ง และกระบวนการทำงาน

คุณภาพ & ความปลอดภัย

  • code-reviewer - ตรวจสอบโค้ดอย่างเชี่ยวชาญ เน้นความปลอดภัยและความน่าเชื่อถือในการผลิต
  • security-auditor - ตรวจสอบโค้ดหาช่องโหว่และรับรองมาตรฐาน OWASP
  • test-automator - สร้างชุดทดสอบครอบคลุม ทั้ง unit, integration, และ e2e
  • performance-engineer - วิเคราะห์ประสิทธิภาพ, ปรับปรุงจุดคอขวด, และวางกลยุทธ์แคช
  • debugger - ผู้เชี่ยวชาญในการดีบักข้อผิดพลาด, การทดสอบล้มเหลว, และพฤติกรรมไม่คาดคิด
  • error-detective - ค้นหาข้อผิดพลาดใน log และ codebase, วิเคราะห์ stack trace และความผิดปกติ
  • search-specialist - ผู้เชี่ยวชาญด้านการค้นคว้าเว็บด้วยเทคนิคการค้นหาขั้นสูงและการสังเคราะห์ข้อมูล

ข้อมูล & AI

  • data-scientist - ผู้เชี่ยวชาญด้านการวิเคราะห์ข้อมูลสำหรับ SQL, BigQuery และข้อมูลเชิงลึก
  • data-engineer - สร้าง ETL pipeline, data warehouse และสถาปัตยกรรมสตรีมมิ่ง
  • ai-engineer - สร้างแอปพลิเคชัน LLM, ระบบ RAG และ pipeline สำหรับ prompt
  • ml-engineer - สร้าง ML pipeline, การให้บริการโมเดล และวิศวกรรมคุณลักษณะ
  • mlops-engineer - สร้าง ML pipeline, ติดตามการทดลอง และ registry โมเดล
  • prompt-engineer - ปรับแต่ง prompt สำหรับ LLM และระบบ AI

โดเมนเฉพาะทาง

  • api-documenter - สร้าง OpenAPI/Swagger spec และเขียนเอกสารสำหรับนักพัฒนา
  • payment-integration - เชื่อมต่อ Stripe, PayPal และตัวประมวลผลการชำระเงิน
  • quant-analyst - สร้างโมเดลการเงิน, ทดสอบกลยุทธ์เทรด และวิเคราะห์ข้อมูลตลาด
  • risk-manager - ตรวจสอบความเสี่ยงของพอร์ต, R-multiples, และขีดจำกัดการถือครอง
  • legacy-modernizer - ปรับปรุงโค้ดเก่าและนำเทคโนโลยีใหม่มาใช้แบบค่อยเป็นค่อยไป
  • context-manager - จัดการ context ระหว่าง agent หลายตัวและงานระยะยาว

เอกสารประกอบ

  • docs-architect - สร้างเอกสารเทคนิคอย่างครบถ้วนจาก codebase ที่มีอยู่
  • mermaid-expert - สร้างไดอะแกรม Mermaid สำหรับ flowchart, ลำดับ, ERD และสถาปัตยกรรม
  • reference-builder - สร้างเอกสารอ้างอิงทางเทคนิคอย่างละเอียดและ API documentation
  • tutorial-engineer - สร้างบทเรียนทีละขั้นและเนื้อหาการสอนจากโค้ด

ธุรกิจ & การตลาด

  • business-analyst - วิเคราะห์ตัวชี้วัด, สร้างรายงาน, และติดตาม KPI
  • content-marketer - เขียนบทความ, เนื้อหาสื่อสังคม และจดหมายข่าวอีเมล
  • hp-pro - คู่คิดด้านการจ้างงาน, onboarding/offboarding, ลางาน, ผลงาน, นโยบาย และความสัมพันธ์พนักงาน
  • sales-automator - ร่างอีเมลหาลูกค้า, อีเมลติดตามผล และเทมเพลตข้อเสนอ
  • customer-support - ดูแลตั๋วสนับสนุน, ตอบคำถาม FAQ และอีเมลลูกค้า
  • legal-advisor - ร่างนโยบายความเป็นส่วนตัว ข้อกำหนดการให้บริการ ข้อจำกัดความรับผิดชอบ และประกาศทางกฎหมาย

SEO & การเพิ่มประสิทธิภาพเนื้อหา

  • seo-content-auditor - วิเคราะห์เนื้อหาที่ให้มาเพื่อคุณภาพ สัญญาณ E-E-A-T และแนวทางปฏิบัติที่ดีที่สุดด้าน SEO
  • seo-meta-optimizer - สร้างชื่อเมตา คำอธิบาย และข้อเสนอแนะ URL ที่เหมาะสมที่สุด
  • seo-keyword-strategist - วิเคราะห์การใช้คีย์เวิร์ด คำนวณความหนาแน่น เสนอแนวทางการใช้คำที่เกี่ยวข้อง
  • seo-structure-architect - ปรับโครงสร้างเนื้อหา ลำดับส่วนหัว และการกำหนด schema markup
  • seo-snippet-hunter - จัดรูปแบบเนื้อหาเพื่อให้เหมาะสมกับ featured snippet และฟีเจอร์ใน SERP
  • seo-content-refresher - ระบุองค์ประกอบที่ล้าสมัยและเสนอการอัปเดตเนื้อหา
  • seo-cannibalization-detector - วิเคราะห์หลายหน้าเพื่อหาการซ้ำซ้อนและความขัดแย้งของคีย์เวิร์ด
  • seo-authority-builder - วิเคราะห์เนื้อหาเพื่อสัญญาณ E-E-A-T และตัวชี้วัดความน่าเชื่อถือ
  • seo-content-writer - เขียนเนื้อหาที่เหมาะกับ SEO ตามคีย์เวิร์ดและข้อกำหนด
  • seo-content-planner - สร้างโครงร่างเนื้อหา คลัสเตอร์หัวข้อ และปฏิทินเนื้อหา

รูปแบบซับเอเจนต์

แต่ละซับเอเจนต์จะมีโครงสร้างดังนี้: 

---
name: subagent-name

System prompt defining the subagent's role and capabilities

การกำหนดค่าของโมเดล

ณ Claude Code v1.0.64, ซับเอเจนต์สามารถระบุได้ว่าควรใช้โมเดล Claude รุ่นใด ซึ่งช่วยให้สามารถมอบหมายงานตามความซับซ้อนอย่างประหยัดต้นทุน:

  • ความซับซ้อนต่ำ (Haiku): งานง่าย ๆ เช่น การวิเคราะห์ข้อมูลพื้นฐาน การสร้างเอกสาร และการตอบกลับมาตรฐาน
  • ความซับซ้อนปานกลาง (Sonnet): งานพัฒนา การรีวิวโค้ด การทดสอบ และงานวิศวกรรมมาตรฐาน
  • ความซับซ้อนสูง (Opus): งานสำคัญ เช่น การตรวจสอบความปลอดภัย การรีวิวสถาปัตยกรรม การตอบสนองเหตุการณ์ และวิศวกรรม AI/ML

โมเดลที่มีให้เลือก (ใช้ชื่อแบบย่อ ณ Claude Code v1.0.64):
  • haiku - รวดเร็วและประหยัดสำหรับงานง่าย ๆ
  • sonnet - สมดุลทั้งประสิทธิภาพสำหรับงานพัฒนาส่วนใหญ่
  • opus - มีความสามารถสูงสุดสำหรับการวิเคราะห์ที่ซับซ้อนและงานสำคัญ

หากไม่ได้ระบุโมเดล ซับเอเจนต์จะใช้โมเดลเริ่มต้นของระบบ

รูปแบบการประสานงานของเอเจนต์

Claude Code จะประสานงานเอเจนต์โดยอัตโนมัติด้วยรูปแบบทั่วไปเหล่านี้:

ลำดับเวิร์กโฟลว์

User Request → Agent A → Agent B → Agent C → Result
Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor

Parallel Execution

User Request → Agent A + Agent B (simultaneously) → Merge Results
Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations

การแยกแขนงตามเงื่อนไข

User Request → Analysis → Route to appropriate specialist
Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter

การทบทวนและการตรวจสอบ

Primary Agent → Review Agent → Final Result
Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation

เมื่อไรควรใช้เอเจนต์ตัวไหน

  • unity-developer: การพัฒนาเกม Unity, การเขียนสคริปต์ C#, การปรับประสิทธิภาพ

  • sql-pro: คำสั่งฐานข้อมูล, การออกแบบสคีมา, การปรับ query ให้มีประสิทธิภาพ

แนวปฏิบัติที่ดีที่สุด

🎯 การมอบหมายงาน

  • ให้ Claude Code มอบหมายงานอัตโนมัติ - ตัวแทนหลักวิเคราะห์บริบทและเลือกตัวแทนที่เหมาะสมที่สุด
  • ระบุข้อกำหนดให้ชัดเจน - รวมข้อจำกัด, เทคโนโลยีที่ใช้, และข้อกำหนดด้านคุณภาพ
  • เชื่อมั่นในความเชี่ยวชาญของตัวแทน - แต่ละตัวแทนได้รับการปรับแต่งให้เหมาะกับโดเมนของตน

🔄 เวิร์กโฟลว์หลายตัวแทน

  • เริ่มด้วยคำขอระดับสูง - ให้ตัวแทนประสานงานงานที่ซับซ้อนหลายขั้นตอน
  • ให้ข้อมูลระหว่างตัวแทน - ตรวจสอบให้แน่ใจว่าตัวแทนมีข้อมูลพื้นฐานที่จำเป็น
  • ทบทวนจุดบูรณาการ - ตรวจสอบว่าผลลัพธ์ของตัวแทนต่าง ๆ ทำงานร่วมกันอย่างไร

🎛️ การควบคุมอย่างชัดเจน

  • ใช้การเรียกใช้อย่างชัดเจนเมื่อมีความต้องการเฉพาะ - เมื่อคุณต้องการมุมมองจากผู้เชี่ยวชาญเฉพาะด้าน
  • รวมตัวแทนหลายรายอย่างมีกลยุทธ์ - ผู้เชี่ยวชาญต่างสายสามารถตรวจสอบงานซึ่งกันและกัน
  • ขอรูปแบบการตรวจสอบเฉพาะ - "ให้ security-auditor ตรวจสอบการออกแบบ API ของ backend-architect"

📈 การเพิ่มประสิทธิภาพ

  • ติดตามประสิทธิผลของเอเจนต์ - เรียนรู้ว่าเอเจนต์ใดเหมาะสมที่สุดกับกรณีการใช้งานของคุณ
  • ปรับปรุงงานที่ซับซ้อนอย่างต่อเนื่อง - ใช้ข้อเสนอแนะจากเอเจนต์เพื่อปรับแต่งข้อกำหนด
  • ใช้จุดแข็งของเอเจนต์ - จับคู่ความซับซ้อนของงานกับความสามารถของเอเจนต์

การมีส่วนร่วม

เพื่อเพิ่มซับเอเจนต์ใหม่:

  • สร้างไฟล์ .md ใหม่ตามรูปแบบข้างต้น
  • ใช้ชื่อที่เป็นตัวพิมพ์เล็กและคั่นด้วยขีดกลาง
  • เขียนคำอธิบายให้ชัดเจนว่าควรใช้ซับเอเจนต์เมื่อใด
  • รวมคำแนะนำเฉพาะไว้ใน system prompt

การแก้ไขปัญหา

ปัญหาทั่วไป

เอเจนต์ไม่ถูกเรียกใช้งานโดยอัตโนมัติ:

  • ตรวจสอบว่าคำขอของคุณระบุโดเมนอย่างชัดเจน (เช่น "ปัญหาด้านประสิทธิภาพ" → performance-engineer)
  • ระบุประเภทของงานอย่างชัดเจน (เช่น "ตรวจสอบโค้ด" → code-reviewer)

การเลือกเอเจนต์ที่ไม่คาดคิด:
  • ให้รายละเอียดเพิ่มเติมเกี่ยวกับเทคโนโลยีและข้อกำหนดของคุณ
  • ใช้การเรียกใช้งานแบบเจาะจงหากคุณต้องการเอเจนต์เฉพาะ

เอเจนต์หลายตัวให้คำแนะนำขัดแย้งกัน:
  • สิ่งนี้เป็นเรื่องปกติ - ผู้เชี่ยวชาญต่างสายอาจมีลำดับความสำคัญแตกต่างกัน
  • ขอคำชี้แจง: "ช่วยประสานข้อแนะนำจาก security-auditor และ performance-engineer"

เอเจนต์ดูเหมือนไม่มีบริบท:
  • ให้ข้อมูลพื้นหลังในคำขอของคุณ
  • อ้างอิงบทสนทนาหรือรูปแบบที่มีการกำหนดไว้ก่อนหน้า

การขอความช่วยเหลือ

หากเอเจนต์ไม่ทำงานตามที่คาดหวัง:

  • ตรวจสอบคำอธิบายของเอเจนต์ในแต่ละไฟล์
  • ลองใช้ภาษาที่เฉพาะเจาะจงมากขึ้นในคำขอของคุณ
  • ใช้การเรียกใช้งานแบบเจาะจงเพื่อทดสอบเอเจนต์แต่ละตัว
  • ให้ข้อมูลเพิ่มเติมเกี่ยวกับโปรเจกต์และเป้าหมายของคุณ
