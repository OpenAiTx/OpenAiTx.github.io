Der Cursor für Designer – ein Open-Source, visuell orientierter Code-Editor

Erstelle Websites, Prototypen und Designs mit KI in Next.js + TailwindCSS. Bearbeite direkt im Browser-DOM mit einem visuellen Editor. Designe in Echtzeit mit Code. Eine Open-Source-Alternative zu Bolt.new, Lovable, V0, Replit Agent, Figma Make, Webflow usw.

Onlook für Desktop (auch bekannt als Onlook Alpha)

Wir sind in einer frühen Vorschau für Onlook Web. Wenn du nach der herunterladbaren Desktop-Electron-App suchst, findest du sie jetzt unter Onlook Desktop.

Warum wechseln wir ins Web? Mehr zu unserer Entscheidung: Migrating from Electron to Web

Erste Schritte

Bald verfügbar als gehostete App oder zum lokalen Ausführen.

Verwendung

Onlook läuft mit jedem Next.js + TailwindCSS-Projekt, importiere dein Projekt in Onlook oder starte direkt im Editor von Grund auf neu.

Nutze den KI-Chat, um ein Projekt zu erstellen oder zu bearbeiten, an dem du arbeitest. Jederzeit kannst du mit einem Rechtsklick auf ein Element die genaue Position im Code öffnen.


Ziehe neue Divs hinein und ordne sie innerhalb ihrer Eltern-Container per Drag-and-Drop neu an.


Sieh dir den Code Seite an Seite mit deinem Site-Design an.


Nutze die Editor-Toolbar von Onlook, um Tailwind-Styles anzupassen, Objekte direkt zu manipulieren und Layouts zu testen.

Dokumentation

Für die vollständige Dokumentation besuche docs.onlook.com

Wie du beitragen kannst, erfährst du unter Contributing to Onlook in unserer Dokumentation.

Wie es funktioniert

  • Wenn du eine App erstellst, laden wir den Code in einen Web-Container
  • Der Container läuft und stellt den Code bereit
  • Unser Editor erhält den Vorschau-Link und zeigt ihn in einem iFrame an
  • Unser Editor liest und indiziert den Code aus dem Container
  • Wir instrumentieren den Code, um Elemente mit ihrer Stelle im Code zu verknüpfen
  • Wenn das Element bearbeitet wird, bearbeiten wir es zuerst im iFrame, dann im Code
  • Unser KI-Chat hat ebenfalls Zugriff auf den Code und Werkzeuge, um diesen zu verstehen und zu bearbeiten

Diese Architektur kann theoretisch auf jede Sprache oder jedes Framework skaliert werden, das DOM-Elemente deklarativ darstellt (z.B. jsx/tsx/html). Wir konzentrieren uns derzeit darauf, dass es gut mit Next.js und TailwindCSS funktioniert.

Für eine vollständige Einführung, siehe unsere Architektur-Dokumentation.
