Karpov Gateway

オールインワン API ゲートウェイ＋コンソール。Go バックエンド（Gin + gRPC）＋ Next.js フロントエンド（App Router + shadcn/ui）、認証情報プール、メール認証、Linux.do OAuth2 SSO、クォータ課金、TOTP 二段階認証を標準搭載。

✨ 機能

  • 統一 REST ゲートウェイ — 単一プロセスまたは複数サービスモード、gateway → auth/music/pool/quota/billing/worker gRPC バックエンドは独立してデプロイ可能
  • 認証情報プール（暗号化） — KEK + DEK 封筒暗号化、AES-256-GCM with AAD、POOL_KEK_HEX 主鍵派生
  • メール登録 + アクティベーション — SMTP コード送信／アクティベーションリンク、二重保護（認証コード＋メールリンク）、SMTP 未設定時は自動的に LogSender へ降格
  • OAuth2 サードパーティログイン — Linux.do SSO（PKCE S256 + HMAC 署名 state cookie）、ログイン／バインド／解除の三状態判定、token 暗号化保存
  • クォータ／課金 — PlanQuotaMiddleware が gateway 層で保護、business service 内部で plan＋scope による二重検証
  • TOTP 二段階認証 — pquerna/otp ＋ Redis リプレイ防止
  • 監査ログ — 構造化 JSON、type ごとにファイル分割、毎日ローテーション
  • CSRF + セッション — sid httpOnly cookie ＋ X-CSRF-Token ヘッダーによる二重検証
  • shadcn/ui コンソール — Next.js 16 App Router ＋ Radix ＋ Tailwind v4

🚀 クイックスタート

1. 前提条件

| ツール | バージョン | | ------------ | ---------- | | Go | 1.24以上 | | Node.js | 20.11以上 | | pnpm | 9以上 | | Docker | 24以上 | | PostgreSQL | 14以上 | | Redis | 7以上 |

2. 起動依存関係（Postgres + Redis + pgAdmin）

cd deploy/compose
cp .env.example .env       # 改强密码！
docker compose up -d

3. Gateway バックエンドの起動

cd gateway
cp .env.example .env       # 与 deploy/compose/.env 的 PG/Redis 密码保持一致
go mod download
go run ./cmd/qqmusic-gateway

默认 :8080 (HTTP) + :9000 (gRPC)


首次启动会在 stderr 打印一个 superadmin 账号 + 临时密码

4. Webコンソールの起動

cd web
cp .env.example .env.local
pnpm install
pnpm dev

http://localhost:3000

🐳 本番環境へのデプロイ (Docker Compose 単一ホスト)

cd deploy/compose-prod
cp .env.example .env

编辑 .env：填入真域名 / 强密码 / SMTP 凭据 / OAuth client

nano .env
生成 KEK

openssl rand -hex 32     # → 写到 POOL_KEK_HEX=
docker compose up -d --build
docker compose logs -f gateway

注意事項：

  • NEXT_PUBLIC_APP_URL を変更したら必ず docker compose build --no-cache web を実行してください。これはクライアントバンドルにインラインで組み込まれるためです。
  • リバースプロキシ（Nginx/Caddy）を使用する場合は必ず TRUST_PROXY=true を設定してください。そうしないと X-Forwarded-Proto が機能しません。
  • KEK が変更されると、古いクレデンシャルストア / 古い OAuth トークンはすべて読み取れなくなります

🔐 OAuth2 (Linux.do) 接続

  • https://connect.linux.do でアプリを申請してください
  • コールバックURLに以下を入力してください：https://your-domain/v1/auth/oauth/linuxdo/callback
  • 取得した client_id / client_secret を .env に記入してください：
   OAUTH_LINUXDO_ENABLED=true
   OAUTH_LINUXDO_CLIENT_ID=...
   OAUTH_LINUXDO_CLIENT_SECRET=...
   OAUTH_LINUXDO_MIN_TRUST_LEVEL=1
   OAUTH_PUBLIC_BASE=https://your-domain
   OAUTH_FRONTEND_BASE=https://your-domain
   ``

  • Restart the gateway, and the login/register page will automatically show "Linux.do One-Click Login"

📦 重要な依存関係

バックエンド：

  • gin-gonic/gin — HTTP フレームワーク
  • grpc-ecosystem — gRPC + gateway
  • jackc/pgx — PostgreSQL ドライバー
  • redis/go-redis — Redis クライアント
  • golang.org/x/oauth2 — OAuth2 / PKCE
  • pquerna/otp — TOTP

フロントエンド：

  • Next.js 16 (App Router + Turbopack)
  • shadcn/ui + Radix
  • Tailwind CSS v4
  • next-auth-csrf エッジセーフ CSRF
